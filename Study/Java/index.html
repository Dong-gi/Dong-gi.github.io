접근 제한자

1. public : 전체 영역에서 접근 가능
2. private : 해당 클래스 내부에서만 접근 가능
3. default = 생략 : 같은 패키지 내부에서만 접근 가능
4. protected : 같은 패키지 or 하위 클래스에서만 접근 가능
5. final : 재정의 or 수정 불가
6. abstract : 인스턴스화 불가. 추상 클래스만 추상 메서드를 가질 수 있다.
7. static : 클래스 로드시 하나만 생성

부모의 것보다 접근 범위를 넓힐 수는 있지만, 축소할 수는 없다.
1. Outer Class : abstract, final, public, default
2. Inner Class : abstract, final, private, protected, public, static, default
3. Outer Interface : abstract, public, default
4. Inner Interface : abstract, private, protected, public, static, default
5. Constructor : private, protected, public, default
6. Method : All
7. Variable : final, private, protected, public, static, default

5. super : 부모 클래스를 가리키는 키워드. super()를 통해 부모 메서드 호출
6. 다형성 설명 생략. 자바는 오버라이드를 명시하지 않아도 재정의한 메서드를 호출해준다.(Dynamic method dispatch)

Object

1. protected Object clone() : 객체의 복사본을 생성하여 반환한다.(얕은 복사)
2. boolean equals(Object obj) : 현재 객체가 obj와 같은지 검사한다.
3. void finalize() : 객체 폐기. 보통 GC가 알아서 처리해주지만, non-Java resource를 해제할 때 사용.
GC가 언제, 정말로 호출할지 알 수 없으므로, finalize()와 별개의 메서드를 정의하는 것이 낫다.
4. Class<?> getClass() : 객체의 Class를 반환
5. int hashCode() : 객체의 hash code 반환 // HashSet, HashMap 등에서 이용
6. void notify() : 해당 객체에 대해 대기하는 스레드를 깨운다.
7. void notifyAll() : 해당 객체에 대해 대기하는 모든 스레드를 깨운다.
8. String toString() : 객체의 문자열 표현 반환
9. void wait() : 해당 객체에 대해 스레드를 대기.
10. void wait(long timeout) : 깨워질 때까지 최대 timeout ms동안 대기.
11. void wait(long timeout, int nanos) : timeout ms + nanos ns

패키지

1. 패키지는 네임스페이스와 비슷하게 클래스들을 구분해준다.
2. 일반적으로 패키지 이름은 도메인의 역순으로 구성하고, 소문자만을 이용한다.
3. 패키지 내부에 패키지를 가질 수 있다. package donggi.package1.sub1;
4. import 패키지.(타입 or *); 이름 충돌이 발생할 때는 패키지 이름을 명시한다.
5. 정적 멤버 임포트 : import static package . className . ( staticMemberName | * );

예외

1. Throwable <- Exception(Checked Exception) <- RuntimeException(Unchecked Exception)
2. Error : 프로그램 내부에서 해결할 수 없어 종료됨.
3. Unchecked Exception은 따로 처리하지 않아도 프로그램이 계속 실행됨.
4. Chained Exceptions : 하나의 예외로 인해 다른 예외들이 발생할 수 있다. JDK 1.4부터 근원적인 예외를 알아올 수 있는 방법이 추가되었다.
Throwable(Throwable causeExc), Throwable(String msg, Throwable causeExc) : 보다 근본적인 원인을 인자로 하여 새로운 예외 생성
Throwable initCause(Throwable causeExc) : 예외의 근원을 설정하며, 생성자를 통해 호출되거나 이미 호출된 경우 다시 호출 불가
Throwable getCause() : 현재 예외의 근원을 얻는다. 없으면 null

스레드

1. 가변길이 매개변수를 갖는 메서드 : (int... nums)와 같이 여러 개의 인자를 배열로 받는다.

6. 동기화 : synchronized(객체를 이용한 잠금), volatile(변수에 대한 잠금)
