실습 1

1. XAML로 컨트롤을 작성하는 것과 코드에서 생성하는 것은 결과적으로 동일하다.
Blend에서 디자인을 따로 할 수 있다. 분업-협업 가능.
2. 참조 -> 참조 추가에서 .NET 어셈블리 or 제작한 프로젝트 참조 추가
3. NuGet 패키지 관리에서 다른 사용자들이 배포하는 프레임워크를 설치할 수 있다.
4. 네임스페이스 접두어 xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
5. Window : Initialized -> Activated -> Loaded -> ContentRendered -> 실행 -> Closing -> Unloaded -> Deactivated -> Closed
Activated, Deactivated는 키보드/마우스 포커스를 얻거나 잃는 이벤트를 의미한다.

레이아웃


디자인을 할 생각은 없기 때문에, 종류만 알아두고 넘어간다.
1. Canvas : 캔버스 내부의 상대 좌표를 이용해 자식 요소들을 배치한다.
2. DockPanel : 자식 요소들을 수평적, 수직적, 상대적으로 배치한다.
3. Grid : 영역을 행과 열로 나누어 요소들을 배치한다. 나눠진 그리드 하나를 여러 요소가 공유할 수도, 한 요소가 여러 그리드를 차지할 수도 있다.
4. StackPanel : 수직/수평적으로 정렬한 요소들을 하나씩 보여준다.
5. VirtualizingPanel : Panel에 대한 추상 클래스.
6. WrapPanel : 자식 요소들을 좌에서 우로 차례대로 배치하다, 패널의 경계를 넘어서면 다음 줄(바로 아래)로 넘겨 배치한다.

이벤트

1. Event Tunneling : 부모 컨트롤로부터 자식 컨트롤로 Preview 접두어가 붙은 이벤트가 전달됨.
2. Event Bubbling : 이벤트 센더는 이벤트를 처리하고 이를 다시 부모로 전달
3. 1,2와 같은 이벤트를 RoutedEvent라고 하며, 따라서 이벤트 핸들러의 파라미터로 RoutedEventArgs가 넘겨진다.

컨트롤 3

1. ListBox : 항목들을 리스트로 보여주고 한 개 이상을 동시 선택 가능
ListBox :: Items :: Item, Count, IsReadOnly, Add(), Clear(), IndexOf() 등
2. ComboBox : 항목들 중 하나만 선택 가능. DropDownStyle 속성 등
3. ScrollBar : VScrollBar, HScrollBar
LargeChange, SmallChange, Maximum, Minimum, Value 속성 등

컨트롤 4

1. Timer : 주기적인 동작 혹인 일시적인 동작 정의 가능. Enabled 속성, Tick 이벤트 등
2. ListView : 아이콘과 항목을 함께 표시. View, AutoArrange, Sorting, MultiSelect 속성 등

4. MsgBox(메시지, [, MsgBoxStyle] [, 제목]) : 사용자 반응(MsgBoxResult) 리턴
5. InputBox(메시지, [, 제목] [, 기본값]) : 사용자 입력 문자열 리턴
