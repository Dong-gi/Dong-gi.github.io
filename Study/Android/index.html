시작

응용 프로그램을 구성하는 4개의 컴포넌트

1. 액티비티 : 인터페이스를 구성하는 기본 단위. 눈에 보이는 화면 하나. 여러 개의 뷰와 프래그먼트로 구성.
2. 서비스 : 백그라운드에서 실행
3. Broadcast Receiver(BR) : 시스템 신호 수신
4. Content Provider(CP) : 다른 응용프로그램을 위해 자신의 데이터 제공.

서로 독립적으로 동작하는 컴포넌트 간 통신을 Intent가 담당.
Content Provider는 ContentResolver를 사용하여 실행된다.
모든 컴포넌트들은 manifest file에 선언되어야 한다.(AndroidManifest.xml)

어댑터 뷰

화면에 동적으로 변경되는 컨텐츠를 채울 때는 어댑터 뷰에서 상속된 클래스를 이용하는 것이 좋다.
어댑터 뷰에는 ListView, Gallery, GridView 등이 있다.
Adapter : 어댑터 뷰에 데이터를 공급.
예. xml에 정의하지 않고 Activity 클래스를 ListActivity에서 상속받는 코드 package hu.pe.wiz.test; import android.app.ListActivity; import android.support.v7.app.AppCompatActivity; import android.os.Bundle; import android.view.View; import android.widget.ArrayAdapter; import android.widget.ListView; import android.widget.Toast; public class MainActivity extends ListActivity { @Override protected void onCreate(Bundle savedInstanceState) { super.onCreate(savedInstanceState); String[] values = {"a", "b", "c", "d", "e", "f", "a", "b", "c", "d", "e", "f"}; ArrayAdapter adapter = new ArrayAdapter(this, android.R.layout.simple_list_item_1, values); setListAdapter(adapter); } // 리스트가 선택되면 호출된다. @Override protected void onListItemClick(ListView l, View v, int position, long id) { String item = (String) getListAdapter().getItem(position); Toast.makeText(this, item + " selected", Toast.LENGTH_SHORT).show(); } } 리스트 뷰는 항목을 표시할 때, 어댑터 클래스의 getView()를 호출해 반환되는 뷰를 이용한다.
따라서 getView()를 재정의하여 필요한 동작을 수행하게 한다.
