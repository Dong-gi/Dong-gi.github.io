주석 1. 주석(Comment)은 코더를 위한 것으로, 컴파일러는 무시하는 문장들이다.
2. // : '//'부터 줄의 끝까지 무시한다.
3. /* */ : '/*'와 '*/' 사이의 모든 텍스트를 무시한다.
식별자 1. 식별자(Identifier)는 의미 있는 코드에 붙이는 이름을 의미한다. 의미 있는 코드에는 변수(Variable), 상수(Constant), 메서드, 클래스 등이 있다.
2. 자바 식별자는 '_', '$'의 사용을 허용한다.
3. 식별자 관례 : 상수를 제외한 식별자는 CamelCase 이용. 클래스는 대문자로 시작. 상수는 대문자로 이루어진 SNAKE_CASE 이용.
카멜 케이스 설명 스네이크 케이스 설명
자료형 1. 기본 자료형(Base Type)은 사용자가 정의하지 않은, 이미 자바 언어에 정의된 자료형을 의미한다.
2. 자료형은 메모리 상에서 변수가 존재하는 형식을 정의한다.
3. 자바의 8가지 기본 자료형 : boolean, char, byte, short, int, long, float, double
4. 식별자 없이 값 자체를 나타내는 코드를 리터럴(Literal)이라 부른다. 각 자료형에 대응하는 리터럴 표기는 코드를 참고. 예시에서 '+' 연산자가 기본 자료형 각각에 대해 문자열 연결을 해줌을 볼 수 있다. 하지만 자바는 연산자 재정의를 지원해주지는 않으므로 오해하지 말아야 한다.
5. 자바는 강한 타입 검사를 하며, 이는 자바스크립트와 같이 null, '', 0 등을 false로 간주하거나 하는 일이 결코 없음을 의미한다.
6. 참조 자료형(Reference Type)은 기본 자료형을 제외한 모든 자료형을 포함하며, 메모리의 주소를 통해 값에 접근한다. C/C++의 포인터와 같지만, 자바는 포인터 연산을 지원하지는 않는다.
다음 예시는 자바의 배열에 대해 보인다. 자바의 다차원 배열은 기본적으로 Jagged Array이다.
7. 기본 자료형 변수는 정적 할당, 참조 자료형 변수는 동적 할당된다. 동적 할당된 객체들은 자동으로 GC 대상이 된다.
정적 할당이란 컴파일러가 메모리 점유를 정확하게 예측할 수 있는 변수를 생성하는 방법으로, 선언과 동시에 스택 영역에 할당되어 코드 블록이 끝나는 순간 해제된다.
반면 동적 할당되는 객체들은 힙 영역에 생성되며, 그 참조가 유지되는 한 프로그램 종료시까지 유지된다.
연산자 1. 다음 예시는 비트 연산자와 0 나누기를 보여준다. 정수 0으로 나누는 것은 허용되지 않지만, 실수 0.으로 나누는 것은 허용된다. 다만, 0.으로 나눈 결과를 비교할 때는 예시와 같이 Double 클래스 메서드를 이용하는 것이 정확하다.
문장 1. if ~ else if ~ else : 조건식에는 boolean 값만 허용된다.
2. switch : 정수(char 포함), String, Enum이 허용된다.
3. for : label을 설정해 중첩된 루프들 중 어느 것이든 중지시킬 수 있다. Iterable하면 향상된 for 구문을 이용할 수 있다.
4. while, do ~ while; 카멜 케이스 설명 스네이크 케이스 설명

접근 제한자

1. public : 전체 영역에서 접근 가능
2. private : 해당 클래스 내부에서만 접근 가능
3. default = 생략 : 같은 패키지 내부에서만 접근 가능
4. protected : 같은 패키지 or 하위 클래스에서만 접근 가능
5. final : 재정의 or 수정 불가
6. abstract : 인스턴스화 불가. 추상 클래스만 추상 메서드를 가질 수 있다.
7. static : 클래스 로드시 하나만 생성

부모의 것보다 접근 범위를 넓힐 수는 있지만, 축소할 수는 없다.
1. Outer Class : abstract, final, public, default
2. Inner Class : abstract, final, private, protected, public, static, default
3. Outer Interface : abstract, public, default
4. Inner Interface : abstract, private, protected, public, static, default
5. Constructor : private, protected, public, default
6. Method : All
7. Variable : final, private, protected, public, static, default

클래스

1. 객체 지향 원리 : 캡슐화(코드와 데이터를 함께 묶고 외부의 접근/오용을 제한), 상속(extends), 다형성
2. 클래스는 하나의 클래스만 상속할 수 있다. // 인터페이스는 여러 개 가능
3. class initializer, instance initializer : 예제 코드 참조
4. this : 객체 자신을 가리키는 키워드. this()를 통해 다른 생성자 호출
5. super : 부모 클래스를 가리키는 키워드. super()를 통해 부모 메서드 호출
6. 다형성 설명 생략. 자바는 오버라이드를 명시하지 않아도 재정의한 메서드를 호출해준다.(Dynamic method dispatch)
7. instanceof 연산자로 호환 여부 확인 가능

Object

1. protected Object clone() : 객체의 복사본을 생성하여 반환한다.(얕은 복사)
2. boolean equals(Object obj) : 현재 객체가 obj와 같은지 검사한다.
3. void finalize() : 객체 폐기. 보통 GC가 알아서 처리해주지만, non-Java resource를 해제할 때 사용.
GC가 언제, 정말로 호출할지 알 수 없으므로, finalize()와 별개의 메서드를 정의하는 것이 낫다.
4. Class<?> getClass() : 객체의 Class를 반환
5. int hashCode() : 객체의 hash code 반환 // HashSet, HashMap 등에서 이용
6. void notify() : 해당 객체에 대해 대기하는 스레드를 깨운다.
7. void notifyAll() : 해당 객체에 대해 대기하는 모든 스레드를 깨운다.
8. String toString() : 객체의 문자열 표현 반환
9. void wait() : 해당 객체에 대해 스레드를 대기.
10. void wait(long timeout) : 깨워질 때까지 최대 timeout ms동안 대기.
11. void wait(long timeout, int nanos) : timeout ms + nanos ns

패키지

1. 패키지는 네임스페이스와 비슷하게 클래스들을 구분해준다.

예외

1. Throwable <- Exception(Checked Exception) <- RuntimeException(Unchecked Exception)
2. Error : 프로그램 내부에서 해결할 수 없어 종료됨.
3. Unchecked Exception은 따로 처리하지 않아도 프로그램이 계속 실행됨.
4. Chained Exceptions : 하나의 예외로 인해 다른 예외들이 발생할 수 있다. JDK 1.4부터 근원적인 예외를 알아올 수 있는 방법이 추가되었다.
Throwable(Throwable causeExc), Throwable(String msg, Throwable causeExc) : 보다 근본적인 원인을 인자로 하여 새로운 예외 생성
Throwable initCause(Throwable causeExc) : 예외의 근원을 설정하며, 생성자를 통해 호출되거나 이미 호출된 경우 다시 호출 불가
Throwable getCause() : 현재 예외의 근원을 얻는다. 없으면 null

Generic

1. 여러 타입을 기본 요소로 할 수 있는 범용 클래스/인터페이스 작성을 위한 문법.
2. A가 B의 서브타입일 때, A[]는 B[]의 서브타입이지만, G<A>는 G<B>의 서브타입이 아니다.
3. 메서드에서는 와일드카드 '?'를 이용하여 제네릭 타입을 매개변수로 받을 수 있다.
4. Erasure : 컴파일 시 제네릭 타입을 모두 지우고 바운드 타입으로(명시되지 않았다면 Object) 변경한다. javap를 통해 이를 확인할 수 있다. 자식 메서드의 Erasure가 부모와 다를 때 부모 메서드는 bridge 메서드가 되어 자식의 Erasure를 이용하는 메서드를 호출한다. 이 과정에서 리턴 타입만 다른 메서드들이 생성될 수 있지만 바이트 코드 단계기 때문에 에러가 발생되지 않고 잘 처리된다.
5. 제네릭을 통한 객체 생성 불가. 제네릭을 static 요소에서 이용하는 데 제한.

스레드

1. 가변길이 매개변수를 갖는 메서드 : (int... nums)와 같이 여러 개의 인자를 배열로 받는다.

6. 동기화 : synchronized(객체를 이용한 잠금), volatile(변수에 대한 잠금)
