주석 1. 주석(Comment)은 코더를 위한 것으로, 컴파일러는 무시하는 문장들이다.
2. // : '//'부터 줄의 끝까지 무시한다.
3. /* */ : '/*'와 '*/' 사이의 모든 텍스트를 무시한다.
식별자 1. 식별자(Identifier)는 의미 있는 코드에 붙이는 이름을 의미한다. 의미 있는 코드에는 변수(Variable), 상수(Constant), 메서드, 클래스 등이 있다.
2. 자바 식별자는 '_', '$'의 사용을 허용한다.
3. 식별자 관례 : 상수를 제외한 식별자는 CamelCase 이용. 클래스는 대문자로 시작. 상수는 대문자로 이루어진 SNAKE_CASE 이용.
카멜 케이스 설명 스네이크 케이스 설명
자료형 1. 기본 자료형(Base Type)은 사용자가 정의하지 않은, 이미 자바 언어에 정의된 자료형을 의미한다.
2. 자료형은 메모리 상에서 변수가 존재하는 형식을 정의한다.
3. 자바의 8가지 기본 자료형 : boolean, char, byte, short, int, long, float, double
4. 식별자 없이 값 자체를 나타내는 코드를 리터럴(Literal)이라 부른다. 각 자료형에 대응하는 리터럴 표기는 코드를 참고. 예시에서 '+' 연산자가 기본 자료형 각각에 대해 문자열 연결을 해줌을 볼 수 있다. 하지만 자바는 연산자 재정의를 지원해주지는 않으므로 오해하지 말아야 한다.
5. 자바는 강한 타입 검사를 하며, 이는 자바스크립트와 같이 null, '', 0 등을 false로 간주하거나 하는 일이 결코 없음을 의미한다.
6. 참조 자료형(Reference Type)은 기본 자료형을 제외한 모든 자료형을 포함하며, 메모리의 주소를 통해 값에 접근한다. C/C++의 포인터와 같지만, 자바는 포인터 연산을 지원하지는 않는다. 예시 코드는 자바의 배열에 대해 보인다. 자바의 다차원 배열은 기본적으로 Jagged Array이다.
7. 기본 자료형 변수는 정적 할당, 참조 자료형 변수는 동적 할당된다. 동적 할당된 객체들은 자동으로 GC 대상이 된다.
» 정적 할당 : 컴파일러가 메모리 점유를 정확하게 예측할 수 있는 (지역)변수를 생성하는 방법으로, 선언과 동시에 스택 영역에 할당되어 코드 블록이 끝나는 순간 해제된다.
» 동적 할당 : 동적으로 할당되는 객체들은 힙 영역에 생성되며, 그 참조가 유지되는 한 프로그램 종료시까지 유지된다.
연산자 1. 다음 예시는 비트 연산자와 0 나누기를 보여준다. 정수 0으로 나누는 것은 허용되지 않지만, 실수 0.으로 나누는 것은 허용된다. 다만, 0.으로 나눈 결과를 비교할 때는 예시와 같이 Double 클래스 메서드를 이용하는 것이 정확하다.
문장 1. if ~ else if ~ else : 조건식에는 boolean 값만 허용된다.
2. switch : 정수(char 포함), String, Enum이 허용된다.
3. for : label을 설정해 중첩된 루프들 중 어느 것이든 중지시킬 수 있다. Iterable하면 향상된 for 구문을 이용할 수 있다.
4. while, do ~ while;
접근 제한자 1. 접근 제한자는 해당 코드를 접근/수정할 수 있는 영역을 제한하는 예약어다.
» public : 어디서든지 접근 가능
» private : 해당 클래스 내부에서만 접근 가능
» 생략(default) : 같은 패키지 내부에서만 접근 가능
» protected : 같은 패키지 or 하위 클래스에서만 접근 가능
» final : 재정의 불가, 수정 불가
» abstract : 인스턴스화 불가. 추상 클래스만 추상 메서드를 정의할 수 있다.
» static : 클래스 로드시 하나만 생성. 클래스 이름을 통해 접근.
2. 부모의 것보다 접근 범위를 넓힐 수는 있지만, 축소할 수는 없다.
3. 요소별 가능한 접근 제한자 목록
» Outer Class : abstract, final, public, default
» Inner Class : abstract, final, private, protected, public, static, default
» Outer Interface : abstract, public, default
» Inner Interface : abstract, private, protected, public, static, default
» Constructor : private, protected, public, default
» Method : All
» Variable : final, private, protected, public, static, default
가변 인수 1. 가변 길이 매개변수 선언 : type... 배열명
2. 나열된 매개변수들이 하나의 배열로 넘겨질 뿐, 배열 자체를 넘길 수는 없다.
람다 1. 익명 메서드 Lambda : 인터페이스의 메서드를 구현한 익명 객체를 생성한다. 메서드를 하나만 정의한 FunctionalInterface만 가능하다.
2. 나열된 매개변수들이 하나의 배열로 넘겨질 뿐, 배열 자체를 넘길 수는 없다.

클래스 1. 자바 코드의 기본 단위는 클래스이며, 파일 이름과 클래스 이름이 일치해야 한다. 이 일치하는 클래스만이 파일 내부에서 public으로 정의될 수 있다.
2. 객체 지향 3원리
» 캡슐화 : 코드와 데이터를 함께 묶고 외부의 접근/오용을 제한
» 상속 : 상위 클래스의 속성들을 하위 클래스에서 그대로 이용할 수 있도록
» 다형성 : 상위 타입 참조 변수로 하위 타입 객체를 참조할 수 있고, 이를 통해 호출하는 메서드는 실제로 참조하는 객체의 타입에 따라 나중에 결정된다.
3. 자바는 C++와 같은 다중상속을 근본적으로 제한했지만, 인터페이스로 인해 원칙이 깨졌다.
4. class initializer, instance initializer : 예시 참조.
5. this : 객체 자신을 가리키는 키워드. this()를 통해 다른 생성자 호출, this.methodName()을 통해 다른 메서드 호출, ClassName.this를 통해 가려진 영역의 객체 참조.
6. super : 부모 클래스를 가리키는 키워드. super()를 통해 부모 메서드 호출, super.methodName()을 통해 다른 메서드 호출

8. instanceof 연산자로 호환 여부 확인 가능
Object 1. Object는 모든 참조 자료형의 부모로, 가장 기본적인 클래스다.
2. protected Object clone() : 객체의 복사본을 생성하여 반환한다.(얕은 복사)
3. boolean equals(Object obj) : 현재 객체가 obj와 같은지 검사한다.
4. void finalize() : 객체 폐기. 보통 GC가 알아서 처리해주지만, non-Java resource를 해제할 때 사용한다. GC가 언제, 정말로 호출할지 알 수 없으므로, finalize()와 별개의 메서드를 정의하는 것이 낫다.

6. int hashCode() : 객체의 hash code 반환 // HashSet, HashMap 등에서 이용

8. void notifyAll() : 해당 객체에 대해 대기하는 모든 스레드를 깨운다.
9. String toString() : 객체의 문자열 표현 반환
10. void wait() : 해당 객체에 대해 스레드를 대기.
11. void wait(long timeout) : 깨워질 때까지 최대 timeout ms동안 대기.
12. void wait(long timeout, int nanos) : timeout ms + nanos ns
