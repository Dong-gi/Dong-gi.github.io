태그와 속성
태그 1. 문서 요소의 타입을 나타내는 이름으로, 사용자가 임의로 만들 수도 있다.
2. 태그는 <과 > 사이에 이름을 적음으로써 표현되며, 여는 태그와 닫는 태그 사이에 내용이 포함될 수 있다.
3. 여는 태그는 <name>, 닫는 태그는 </name>로 표현한다. 아래는 HTML 문서의 기본 골격을 나타내는 태그들이다.
» <!DOCTYPE html> : 문서가 HTML5로 작성되었음을 의미한다.
» <html></html> : root HTML element
» <head></head> : 문서에 대한 meta information
» <title></title> : 제목표시줄에 표시될 제목
» <body></body> : page content
속성 1. 모든 HTML 태그는 속성을 가질 수 있다. 속성은 여는 태그 내부에 이름='값' 쌍으로 표현한다.
2. 속성 값을 표현하기 위해 '', "" 모두 이용 가능하다. 아래는 대표적인 속성들이다.
» html태그 lang속성 : 문서의 국가 언어 표기.
» p태그 title속성 : 단락에 대한 툴팁 표기.
» style 속성 : CSS 속성을 지시
» » inline CSS : 태그에 직접 스타일 속성을 적는다. 예. <img width='100px'>, <img style='width: 100px'>
» » internal CSS : style 태그를 이용해 속성을 적는다.
» » external CSS : 외부 CSS 파일을 불러온다. <link rel="stylesheet" href="">
» disabled속성 : 입력 태그 요소를 사용할 수 없음.
» id속성 : 태그에 고유한 id를 부여한다.
» class속성 : 태그에 특정 그룹의 class를 부여한다.
기본 태그
heading <h1></h1> ~ <h6></h6>
paragraph <p></p>
link 1. <a></a>
2. href 속성 : 이동할 위치나 주소
» 상대주소 : 현재 페이지가 위치한 디렉터리를 기준으로 하여 주소를 표기하는 것. 따라서 문서가 다른 디렉터리로 이동될 때 참조가 유지되지 않을 수 있다. './page.html'은 현재 디렉터리의 'page.html'을 가리키며, 'page.html'과 동일한 문서를 가리킨다.
» 절대주소 : 서버의 루트 디렉터리를 기준으로 하여 주소를 표기하는 것. 루트 디렉터리를 가리키는 '/'으로 시작한다.
» # : 현재 페이지의 특정 위치(id)로 이동하기 위한 표기. '#here'은 현재 페이지의 HTML 요소 중 id가 'here'인 것을 가리킨다.
3. 링크 상태를 나타내는 속성 : link, visited, hover, active
4. target 속성 : _blank(새탭이나 새창에서 열기), _self, _parent, _top, framename
img 1. <img>
» src 속성 : 이미지 리소스의 위치나 주소
» alt 속성 : 이미지를 불러오지 못할 때, 혹은 시각장애인을 위한 대체 문자열
» width, height 속성 : 이미지 크기
line break <br> : HTML은 기본적으로 빈칸을 무시하므로, 새로운 줄에서 시작하기 위해 별도의 br태그를 이용한다.
수평선 <hr>
pre <pre></pre> : 별도 해석 없이 pre 태그 내용을 빈칸을 포함하여 글자 그대로 출력한다.
text 1. <b></b> : bold
2. <strong></strong> : bold & important

9. <sub></sub>, <sup></sup> : 아랫첨자, 윗첨자

3. tr은 th와 td들로 이루어진다.

list 1. unordered list : <ul></ul>
2. ordered list : <ol></ol>
» type 속성 : "1"(default), "A", "a", "I"(로마숫자), "i"

div : span 1. div : 가로 100%를 점유하는 block element 대표
2. span : 가로 공간이 여유롭다면 병렬로 표시되는 inline element 대표
script 1. <script></script> 사이에 JavaScript 코드 작성
2. <noscript></noscript> 사이에 지원하지 않는 브라우저에 대한 메시지 작성
3. <a href="javascript:;">처럼 속성으로 넣는 것도 가능
meta 1. 페이지에 대한 메타데이터를 명시한다.
» 인코딩 명시 : <meta charset="UTF-8">
» 검색엔진을 위한 키워드 설정 : <meta name="keywords" content="Word1, Word2, Word3">
» 페이지 설명 : <meta name="description" content="~~">
» 페이지 업데이트 설정 : <meta http-equiv="refresh" content="30"> (30초마다 갱신)
» 반응형 웹을 위한 viewport 설정 : <meta name="viewport" content="width=device-width, initial-scale=1.0">
base 1. 페이지에서 사용하는 모든 상대주소의 시작점을 설정한다.
예. <base href="https://www.w3schools.com/images/" target="_blank">

시맨틱 웹
Semantic HTML5 이전의 웹페이지들은 요소들을 손쉽게 배치하기 위해 table 태그를 주로 사용하여 디자인했다. 하지만 중첩된 table들로 이루어진 웹페이지는 검색 엔진을 비롯한 프로그램이 기계적으로 의미를 찾기 어려워 HTML5는 문서에 대한 의미를 태그가 포함하는 방향을 모색했다.
시맨틱 태그 1. header, nav, section, article, aside, footer, ...
2. 자세한 설명은 여기로
Semantic CSS 1. HTML5로 접어들면서 HTML 태그들은 골격만을 정의하고, 디자인은 JavaScript와 CSS로 보다 확실히 분리되었다. 디자인의 특징은 의미 범주가 같은 것들을(class) 비슷하게 표현한다는 것이며, 따라서 CSS 선택자들은 문서의 의미 정보를 내포하고 있다고 할 수 있다.
2. Bootstrap, W3C.CSS과 같은 인기 있는 CSS Framework들을 사용하면 체계적으로 정의된 클래스들 덕분에 수월하게 문서의 의미 정보를 뽑아낼 수 있다.
반응형 웹
Responsive 1. 스마트폰 대중화 이전의 일반 사용자 컴퓨팅 환경은 가로 768px 이상의 직사각형 모니터가 대부분이었다.
2. 스마트폰이 대중화 되면서 다양한 사용자 웹서핑 환경에 맞춰 알맞게 페이지를 보여줄 필요가 생겼다.
3. viewport로 대표되는 screen 속성을 이용해 페이지를 다시 로드할 필요 없이 디자인이 변경되는 반응형 웹 기술이 발전하게 되었다.
picture 1. 스크린에 따라 다른 이미지 소스를 이용하는 picture태그가 소개되었다.
<picture>
    <source srcset="img_smallflower.jpg" media="(max-width: 600px)">
    <source srcset="img_flowers.jpg" media="(max-width: 1500px)">
    <source srcset="flowers.jpg">
    <img src="img_smallflower.jpg" alt="Flowers">
</picture>
