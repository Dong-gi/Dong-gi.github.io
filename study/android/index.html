Basics

첫 프로젝트
프로젝트 생성
1. 기본 설정대로 프로젝트를 생성하면 아래의 4개 주요 파일을 볼 수 있다.
» MainActivity : 앱을 실행하면 이 클래스의 인스턴스가 실행되고 layout을 로드한다.
» activity_main.xml : 액티비티의 UI를 정의한다.
» AndroidManifest.xml : 앱의 기본적인 특성을 기술하고, 각 컴포넌트를 정의한다.
» build.gradle : 앱의 컴파일과 빌드에 관한 구성 설정.
UI 구성
1. 안드로이드 UI는 layout(ViewGroup 객체)들과 widget(View 객체)들의 계층으로 이루어진다.
» layout : 보이지 않는 container로, 자식 요소들을 어디에 위치시킬 지 조정한다.
» widget : 버튼, 텍스트 박스 등의 UI 요소
2. 기본으로 적용된 ConstraintLayout은 형제 요소들과 부모 레이아웃에 대한 거리를 통해 단층의 뷰를 쉽게 생성한다. 즉, 그리는 시간이 긴 nested layout을 방지한다.
3. Text box 추가 : EditText를 드래그-앤-드롭하여 blueprint에 추가한다. 요소를 선택하면 사각형의 크기 조절 핸들과 원형의 constraint anchor를 볼 수 있다. anchor를 드래그-앤-드롭하여 constraint(margin) 기준을 설정한다.
4. Button 추가 : Button을 드래그-앤-드롭하여 추가한다. 요소를 선택하고, Edit Baseline을 클릭하면 요소의 기준선으로 constraint를 설정할 수 있다.
5. UI 문자열 변경 : app » res » values » strings.xml을 수정하여 변경한다.
» 편집 창 우측 상단의 "Open editor"를 클릭하면 Translations Editor가 열린다. Add Key(+)버튼으로 간단하게 값을 추가할 수 있다.
6. 요소를 선택하면 우측에서 속성들을 변경할 수 있다.
» layout_width를 match_constraint로 변경하면 margin을 제외한 전체를 차지하게 된다.
7. 여러 요소들을 함께 선택하여 우클릭한 뒤 chain으로 연결하면 하나의 가로/세로 요소로 배치할 수 있다.
다른 액티비티 시작
1. Intent : 컴포넌트 간의 데이터 전달.
» public Intent(Context packageContext, Class<?> cls)
» intent.putExtra(String name, ? value)
2. startActivity(intent)
3. Project window » 우클릭 » New » Activity : 자동으로 매니페스트에도 추가된다.
4. 매니페스트에 논리적인 부모 액티비티를 정의하면 "app bar"에 네비게이션이 추가된다.
App Fundamentals
기본
1. Android 앱은 설치되고 나면 자체적인 보안 샌드박스 안에 존재한다.
2. Android는 멀티유저 Linux 시스템으로, 각 앱은 각기 다른 사용자와 같다.
3. 시스템은 각 앱에 고유한 ID(시스템만 사용 가능)를 할당하며, 앱 안의 모든 파일에 대해 할당된 ID의 사용자만 액세스할 수 있게 한다.
» 두 앱에 같은 ID를 공유하도록 만든 경우, 서로의 파일에 접근할 수 있다.
4. 각 프로세스는 자체 VM이 있어 다른 앱과 격리된 상태로 실행된다.
구성 요소
1. 액티비티 : UI가 있는 단일 화면. 액티비티는 한 번에 하나만 실행할 수 있다.
2. 서비스 : 백그라운드에서 실행되는 요소.
3. ContentProvider : 공유된 앱 데이터 집합을 관리.
4. BroadcastReceiver : 각 브로드캐스트는 Intent 객체로 전달된다.
5. Activity, Service, BroadcastReceiver는 Intent로 활성화되고, ContentProvider는 ContentResolver의 요청으로 활성화된다.
» Activity : startActivity() or startActivityForResult()
» Service : startService() or bindService()
» Broadcast : sendBroadcast(), sendOrderedBroadcast(), sendStickyBroadcast()
» ContentProvider : ContentResolver.query()
매니페스트
1. 구성 요소 선언, 요구 권한 식별, 최소 API 레벨 선언 등
2. <activity>, <service>, <receiver>, <provider>
» BroadcastReceiver는 동적으로 registerReceiver()를 통해 등록할 수 있다.
3. 다른 앱의 intent에 응답하기 위해 <intent-filter>요소를 해당 요소의 하위에 추가한다.
4. 요구사항 선언
카메라 필수 : <uses-feature android:name="android.hardware.camera.any" android:required="true" />
API 7이상 : <uses-sdk android:minSdkVersion="7" android:targetSdkVersion="19" />

UI 기본 속성
1. 크기 단위
» px(pixels) : 장치마다 밀도가 달라 제각기로 표현된다.
» dp(density-independent pixels) : 160dp는 밀도와 관계없이 1인치로 표시된다.
» pt(points) : 1/72 인치
» in(inches) : 인치
2. 배경색 : android:background
3. 가시성 : android:visibility
기본 View
1. TextView : 편집 불가능한 텍스트
» 속성 : android:text, textColor, textSize, typeface, lines
2. EditText : 편집 가능한 텍스트
» android:drawableBottom : 텍스트 하단에 표시되는 이미지 리소스
» android:editable : 편집 가능 여부
» android:singleLine : 단일 라인 여부
» android:inputType : 입력 종류
»» Text, textMultiLine, textPostalAddress, textEmailAddress, textPassword, number, numberSigned, numberDecimal, phone, datetime
» android:hint : 힌트 메시지
3. ImageView
» android:adjustViewBounds : 종횡비를 유지하기 위해 뷰의 가로/세로를 조정할 지 여부
» android:cropToPadding : 패딩 안에 맞춰 이미지를 자를지 여부
» android:scaleType : 뷰의 크기에 맞춰 이미지를 확대/축소할 방법
» android:src
» android:tint : 이미지 배경색
4. Button extends TextView
» android:onClick, drawableLeft
5. ToggleButton extends CompoundButton extends Button
Layouts
1. LinearLayout : 수직/수평 배치 : orientation, gravity{top, bottom, ...}, layout_weight
2. TableLayout : <TableRow>들로 구성. 다른 뷰로도 행을 구성 가능. : stretchColumns, shrinkColumns, collapseColumns
3. GridLayout : 격자 배치
4. RelativeLayout : 상대적 위치에 따른 배치 : layout_above, layout_below, ...
5. TabLayout : 탭을 이용하여 겹쳐진 자식 뷰 중 하나를 선택
6. AbsoluteLayout : 절대 위치로 배치
7. FrameLayout : 모든 자식들을 좌측 상단에 겹치게 배치. visibility속성으로 보이는 화면을 제어.
