1. abs(x) : 절댓값을 반환한다. 복소수의 경우 절댓값으로 magnitude(= sqrt(a^2+b^2))가 반환된다.

2. all(iterable) : iterable 모든 요소가 True면 True 반환. iterable이 빈 경우도 True 반환.

3. any(iterable) : iterable 하나의 요소라도 True면 True 반환. iterable이 빈 경우 False 반환.

4. ascii(object) : repr()처럼 객체의 문자열 표현을 반환한다. ASCII 문자가 아닌 경우 이스케이프된다.
>>> ascii("안녕 세상!")
"'\\uc548\\ub155 \\uc138\\uc0c1!'"

5. bin(x) : 정수를 "0b"로 시작하는 2진 문자열로 변환한다. 다른 객체의 경우 정수를 반환하는 __index__()를 정의하면 그것을 이용한다.
>>> bin(5), format(5, '#b'), format(5, 'b'), f'{5:#b}', f'{5:b}'
('0b101', '0b101', '101', '0b101', '101')

6. class bool([x]) : int의 서브클래스로, 더 상속할 수 없다. 클래스의 인스턴스는 True, False 둘 뿐이다.

7. class bytearray([source[, encoding[, errors]]])
» 변경할 수 있는 바이트 배열. 생성자에 인자가 없으면 크기가 0인 바이트 배열을 반환한다.
» source가 문자열인 경우 : encoding이 반드시 설정되어야 하며, str.encode()를 통해 변환된다.
» source가 정수인 경우 : 해당 바이트 크기 만큼의 배열이 0 초기화된다.
» source가 buffer 인터페이스와 호환되는 객체일 경우 : 객체의 읽기 전용 버퍼로 배열을 초기화한다.
» source가 iterable인 경우 : 반복자 모든 요소가 [0, 255] 사이의 정수여야 한다.

11. compile(source, filename, mode, flags=0, dont_inherit=False, optimize=-1)
» source를 code나 AST 객체로 컴파일한다. Code 객체는 exec()나 eval()을 통해 실행할 수 있다.
» source로는 문자열, 바이트 문자열, AST 객체가 가능하다.
» filename : 코드를 읽어온 파일. 파일로부터 읽은 것이 아니라면 '<string>'를 보통 대신 넘긴다.
» mode : 'exec'(여러 문장들 실행), 'eval'(하나의 표현식 계산), 'single'(한 줄 실행)
» optimize : 0(최적화 없음, __debug__ is true), 1(asserts 제거, __debug__ is false), 2(docstring도 제거)
>>> exec(compile("a = 1\nb = 2\nprint(a+b)", "<string>", "exec"))
3

3. dir([object])
» 인자가 없으면 현재 local scope의 이름들을 나열한다. 인자가 있으면 객체의 속성들을 알파벳순으로 나열한다.

1. str과 bytes모두 지원하지만, 찾는 패턴과 대상의 타입은 하나로 일치해야 한다.

» $ : 문자열 끝 또는 newline 직전이 일치하는지. MULTILINE 플래그가 있으면 각 newline 바로 앞을 검사

» + : RE가 1번 이상 반복.
» ? : RE가 0번 또는 1번.
» *?, +?, ?? : *, +, ?는 greedy 기법으로, 가장 큰 문자열 매칭을 찾는다. 이들 뒤에 ?를 붙임으로써 가장 짧은 문자열 매칭을 찾는다.
» {m} : RE가 정확히 m번 반복
» {m, n} : RE가 [m, n]번 반복. m이 없으면 0, n이 없으면 무한대
» {m, n}? : 가장 최소 반복의 매칭을 찾는다.
» \ : *, ? 등의 특수 문자를 이스케이프하거나 특수 구문을 만들기 위해 사용. 문자열에서도 \를 이스케이프하므로 자칫 \\\\처럼 4번 써야하는 수가 있으므로, raw 문자열을 이용하는 것이 편하다.
» [] : 문자 집합을 표현한다.
* '(', '+', '*', ')'는 []안에서 특별하지 않다.
* '-' 자체를 나타내려면 \-로 표기.
* \w, \S 등의 character class는 의미를 그대로 갖는다.
* ^으로 시작하면, ^ 다음의 집합에 대한 여집합 문자들로 매칭한다. 단, 맨 처음 ^를 제외한 ^는 단순한 리터럴이다.
* 리터럴 ']'를 집합에 명시하려면 \를 사용하거나 집합의 맨 처음에 적는다.
* 문자열 리터럴 표준 이스케이프 \a, \b, \f, \n, \r, \t, \u Since 3.3, \U Since 3.3, \v, \x, \\를 이용할 수 있다.
» | : A|B는 A나 B 하나만 만족하면 된다. 판정은 왼쪽부터하며, 하나라도 만족되면 이후는 판정하지 않는다.
» (...) : 내부에 있는 RE 자체. 매칭된 문자열을 나중에 가져올 수 있다.
» (?...) : 확장 표현. ? 뒤의 첫 글자가 의미와 구조를 결정한다. 다음이 지원되는 확장들.
* (?aiLmsux) : 'a', 'i', 'L', 'm', 's', 'u', 'x'중 하나 이상을 이용. 이 자체로는 빈 문자열을 매칭하며, re.compile()에 인자로 넘기는 것을 대신해 각각은 re.A, re.I, re.L, re.M, re.S, re.U, re.X 플래그와 관계된다.
* (?:...) : (...)와 같지만, 매칭된 문자열을 가져오거나 참조할 수 없다.
* (?imsx-imsx:...) Since 3.6 : 'i', 'm', 's', 'x'중 하나 이상을 사용. '-'와 이어지는 하나 이상의 문자들은 선택적이다. 이들 각각은 관계된 플래그를 설정하거나 제거한다.
* (?P<name>...) : 이 그룹에 의해 매칭된 문자열은 name으로 접근할 수 있다. name은 Python 식별자 규칙을 따라야 하며, 각 그룹에 순서가 매겨지므로 숫자로도 접근할 수 있다.
** 예를 들어 (?P<quote>['"]).*?(?P=quote)는 ''또는 ""로 감싸진 문자열을 매칭한다. quote가 가리키는 그룹은 다음 3가지 문맥에서 접근 가능하다.
** - 같은 패턴 내부에서, (?P=quote) 또는 \1
** - 객체 m에 대한 매칭, m.group('quote'), m.end('quote') 등
** - re.sub()의 repl 인자로서, \g<quote>, \g<1>, \1
* (?#...) : 주석. 무시된다.
* (?=...) : 바로 다음에 ...이 매칭되는 경우. 예를 들어 Isaac (?=Asimov)는 'Issac ' 뒤에 'Asimov'이 붙은 경우만 매칭된다.
* (?!...) : 바로 다음에 ...이 매칭되지 않는 경우.
* (?<=...) : ... 다음에 매칭되는 경우. 예를 들어 (?<=abc)def는 'abcdef'를 매칭한다.
** Since 3.5 group 참조 가능.
* (?<!...) :... 다음에 매칭되지 않는 경우.
* (?(id/name)yes-pattern|no-pattern) : id or name이 있는 그룹이 주어지면 yes-pattern을, 아니면 no-pattern을 매칭. 예를 들어 (<)?(\w+@\w+(?:\.\w+)+)(?(1)>|$)는 '<user@host.com>'과 'user@host.com'은 매칭하지만, '<user@host.com'과 'user@host.com>'는 그렇지 않다.
» \number : 해당 순번의 그룹에 해당하는 내용을 매칭한다. 그룹은 1번부터 99번까지만 허용된다. 예를 들어, (.+) \1은 'the the'나 '55 55'는 매칭하지만 'thethe'는 그렇지 않다.
» \A : 문자열의 시작만 매칭
» \b : Word의 시작이나 끝 빈 문자열을 매칭한다. \b는 \w와 \W 사이의 경계로 정의된다.
» \B : \b의 반대 상황에서 빈 문자열을 매칭한다. r'py\B'는 'python', 'py3'은 매칭하지만, 'py', 'py.', 'py!'는 아니다.
» \d : 유니코드 패턴에서는 유니코드 십진 숫자들을, 바이트 패턴에서는 [0-9]들을 매칭한다.
» \D : not \d
» \s : 공백 문자들을 매칭한다.
» \S : not \s
» \w : Word 문자들을 매칭한다.
» \W : not \W
» \Z : 문자열의 끝을 매칭한다.

1. ChainMap objects Since 3.3

* 여러 매핑을 한데 모아 갱신 가능한 view를 만든다.

* parents : maps의 첫번째 매핑을 제외한 것들로 새로운 ChainMap을 만들어 반환한다. ChainMap(*d.maps[1:])와 동일

» 예시 : https://docs.python.org/3/library/collections.html#chainmap-examples-and-recipes
* 사용 패턴
c = ChainMap() # Create root context
d = c.new_child() # Create nested child context
e = c.new_child() # Child of c, independent from d
e.maps[0] # Current context dictionary -- like Python's locals()
e.maps[-1] # Root context -- like Python's globals()
e.parents # Enclosing context chain -- like Python's nonlocals

d['x'] # Get first key in the chain of contexts
d['x'] = 1 # Set value in current context
del d['x'] # Delete from current context
list(d) # All nested values
k in d # Check all nested values
len(d) # Number of nested values
d.items() # All nested items
dict(d) # Flatten into a regular dictionary


2. Counter objects
» 편리하고 빠른 계수를 지원한다.
» class collections.Counter([iterable-or-mapping])
* Counter는 dict의 서브클래스다. Counter는 없는 키에 대하여 KeyError를 발생하지 않고, 새로운 키를 추가하고 0을 반환한다. 카운트를 0으로 설정하는 것으로 원소가 제거되지 않는다. 카운터 원소를 제거하려면 del 키워드를 사용.
c = Counter() # a new, empty counter
c = Counter('gallahad') # a new counter from an iterable
c = Counter({'red': 4, 'blue': 2}) # a new counter from a mapping
c = Counter(cats=4, dogs=8) # a new counter from keyword args
* elements() : 각 원소들을 각 카운트만큼 반복하는 iterator를 반환한다.
>>> c = Counter(a=4, b=2, c=0, d=-2)
>>> sorted(c.elements())
['a', 'a', 'a', 'a', 'b', 'b']
* most_common([n]) : 가장 많은 n개의 원소들을 그 갯수와 함께 반환한다.
>>> Counter('abracadabra').most_common(3)
[('a', 5), ('r', 2), ('b', 2)]
* subtract([iterable-or-mapping]) : 카운트를 뺀다.
* fromkeys(iterable) : not implemented
* update([iterable-or-mapping]) : 카운트를 더한다.

* 사용 패턴
sum(c.values()) # total of all counts
c.clear() # reset all counts
list(c) # list unique elements

dict(c) # convert to a regular dictionary
c.items() # convert to a list of (elem, cnt) pairs
Counter(dict(list_of_pairs)) # convert from a list of (elem, cnt) pairs
c.most_common()[:-n-1:-1] # n least common elements

5. namedtuple()
» collections.namedtuple(typename, field_names, *, verbose=False, rename=False, module=None)
* 튜플 각 위치에 의미를 부여하여 읽기 쉬게 한다. 기존 튜플이 이용되는 곳에 모두 사용될 수 있고, 인덱스 접근 대신 이름 접근이 가능하다.
* 이름이 typename인 tuple의 서브클래스를 반환한다. 인스턴스들은 유용한 docstring과 __repr__()을 갖는다.
* filed_names는 문자열 시퀀스거나, 'x y'또는 'x, y'처럼 분리한 단일 문자열일 수 있다.
* 필드명은 Python 식별자 모두 사용할 수 있지만, '_'로 시작할 수 없다.
* rename이 True면 유효하지 않은 필드명은 '_1'처럼 인덱스가 이용된다.
* verbose가 True면 클래스 정의가 출력된다. 이는 구식 옵션으로, _source 속성을 출력하는 것이 더 간단하다.
* Named tuple은 인스턴스 딕셔너리가 없어, 보통의 tuple보다 더 많은 메모리를 사용하지 않는다.
* Since 3.6 : verbose, rename은 키워드로만 사용 가능, module 인자 추가
* csv, sqlite3 모듈의 결과를 저장하기에 적합.
** 예.
EmployeeRecord = namedtuple('EmployeeRecord', 'name, age, title, department, paygrade')
import csv
for emp in map(EmployeeRecord._make, csv.reader(open("employees.csv", "rb"))):
    print(emp.name, emp.title)
» classmethod somenamedtuple._make(iterable) : iterable로 튜플 생성
» somenamedtuple._asdict() : OrderedDict 반환
» somenamedtuple._replace(**kwargs) : 값을 변경한 새 튜플 반환. 따라서 기본값 튜플 하나를 이용해 다른 튜플들을 생성하는 것도 좋은 방법이다.
» somenamedtuple._source : namedtuple Python 코드. 파일에 저장하거나 다른데서 exec() 가능
» somenamedtuple._fields : 필드명들의 튜플 반환. 예. Pixel = namedtuple('Pixel', Point._fields + Color._fields)
* getattr()로 필드 접근 가능.
* dictionary를 언패킹(**)하면 생성자에 이용 가능. 예. Point(**{'x':1, 'y':2})
* 생성된 namedtuple은 Python 정규 클래스므로, 함수나 서브클래스를 쉽게 추가할 수 있다.
** 예. class Point(namedtuple('Point', 'x, y')): ...
