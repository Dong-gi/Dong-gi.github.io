FP : Functional Programming

아래의 정의는 『스칼라로 배우는 함수형 프로그래밍』(폴 키우사노, 루나르 비아르드나손, 류광 역, 제이펍, 2015)을 따른다.
1. side effect 없는 pure function들로만 프로그램을 구축하는 것.
2. pure function : 출력이 오직 입력으로만 결정되며, 입력이 불변하면 외부 상태와 관계없이 출력도 불변
3. referentially transparent : 어떤 표현식을 그 평가 결과로 바꾸어도 의미가 변하지 않는다면 이는 참조에 투명.
4. f(x)가 참조에 투명한 모든 x에 대해 참조 투명하면 f는 pure.

문법 1

1. doc주석을 포함하여, 주석 표기는 Java와 같다.
2. 모든 스칼라 코드는 object 또는 class에 정의된다. object는 싱글톤 객체를 생성하며, Java의 익명 클래스 인스턴스를 만드는 것과 유사하다.
3. 스칼라의 모든 것은 객체다.
4. 스칼라에는 static 키워드가 없으며, 정적 멤버를 표현하기 위해 object를 이용한다.
5. 메서드 우변 = {}의 평가값 자체가 리턴되므로 return 키워드가 필요없다.

문법 2

1. 메서드의 반환 형식을 추론할 수 있는 경우 정의에 반환 형식을 생략할 수 있다.
2. Java의 main()에 해당하는 정의 : def main(args: Array[String]): Unit = {}
3. Unit은 Java의 void와 같다.
4. 인자를 하나만 받는 모든 메서드는 괄호를 생략해 중위 표기할 수 있다.
a + b == a.(b), o.func(a) == o func a
