  4. 선택자
    구분문법예시
    전체 선택*$('*').length

    ID 선택#id
    다중 선택,$('h1, h2, h3, h4, h5, h6')

    속성값 포함[attrName*="subString"]$('input[value*="hello"]')
    속성값 시작[attrName^="subString"]$('input[value^="hello"]')
    속성값 종료[attrName$="subString"]$('input[value$="hello"]')
    속성값 단어로 포함[attrName~="word"]$('input[value~="hello"]')
    Pseudo-class 선택:pseudo-class$('a:hover')
    필터 선택:odd|even|first|last$('li:even')
    함수 필터:contains("subStr")|eq(n)|gt(n)|lt(n)|has("tagName")|not("selector")|nth-child(2n+3)

이벤트 모델
  1. DOM 이벤트 흐름 : 이벤트 캡처링 → 타깃 → 이벤트 버블링
    • 이벤트 캡처링 : 최상위 노드로부터 이벤트가 전달되는 과정
      • 캡처링 단계에서 리스너의 호출 기본값은 false. 동작을 위해 addEventListener()의 옵션 매개변수를 지정해야 한다
    • 이벤트 버블링 : 최하위 노드로부터 이벤트가 전달되는 과정
  2. 이벤트 강제 발생
    3. $(node).trigger('eventName');
    $.event.trigger(); // 사용자 정의 이벤트 강제 발생
  3. 이벤트 핸들러 추가/제거
    4. JavaScript 이벤트 핸들러 설정 메서드
    1. EventTarget : Element, Document, Window, XMLHttpRequest
    2. EventTarget.addEventListener() : 이벤트 핸들러 추가
    3. EventTarget.dispatchEvent() : 처리할 이벤트 추가
    4. EventTarget.removeEventListener() : 이벤트 핸들러 제거
    jQuery 이벤트 핸들러 설정 함수
    1. .on(events[, selector][, data], handler)
      2. selector가 null이거나 제공되지 않은 경우, 선택된 노드들에 대해 이벤트 핸들러 추가. selector가 제공된 경우 선택자에 매칭되는 후손 노드(미래에 추가되는 것들도 포함)들에 핸들러가 연결된다.
    2. .one(events[, selector][, data], handler)
      3. 핸들러가 한 번만 이벤트를 처리한다.
