데이터베이스 클러스터 생성

  • 파일 시스템 관점에서 데이터베이스 클러스터는 모든 데이터가 저장되는 단일 디렉터리다.
  • 일반적인 위치 : /usr/local/pgsql/data, /var/lib/pgsql/data
  • 위치 지정
    1. initdb 명령에 -D 옵션을 사용한다
      2. $ initdb -D /tmp/pgsql/data
    2. PGDATA 환경 변수를 설정한다
    3. $ pg_ctl -D /tmp/pgsql/data initdb
  • 인코딩 지정
    • $ initdb -D /usr/local/pgsql/data -E utf8 --no-locale

서버 시작

  • 포그라운드에서 실행 : -D 옵션이 없으면 PGDATA 환경 변수를 이용한다. 이도 없으면 구동 실패
    • $ postgres -D /usr/local/pgsql/data
  • 백그라운드에서 실행
    • $ postgres -D /usr/local/pgsql/data >logfile 2>&1 & $ pg_ctl start -l logfile
  • 서버 자동 시작 스크립트가 contrib/start-scripts에 제공된다
  • 포트 지정
    • $ postgres -p 56789

서버 종료

$ pg_ctl [-w] [-D datadir] [-m [s[mart]|f[ast]|i[mmediate]]] stop
  • SIGTERM : smart

    • 기존 세션의 정상적인 종료를 허용하지만, 새로운 연결은 허용하지 않는다.

  • SIGINT : fast(default)

    • 새로운 연결을 허용하지 않고, 모든 서버 프로세스에 SIGTERM을 전송한다. 이로써 트랜잭션이 중단되고 즉시 종료된다.

  • SIGQUIT : immediate

    • 모든 자식 프로세스에 SIGQUIT을 전달하고 대기한다. 5초 이내에 종료되지 않으면 SIGKILL로 즉시 종료하고, 자신도 정상적인 중지 절차없이 중지한다. 다음 서버 시작 시 복구 작업이 진행된다.

데이터베이스 클러스터 업데이트

PostgreSQL의 버전은 major.minor로 구성된다. 10부터는 major 버전이 단일 숫자지만, 그 외는 9.5, 9.6과 같이 두 영역이 major 버전이다. minor 버전 차이는 클러스터 구조에 영향을 미치지 않기 때문에 실행 파일만 바꾸면 업그레이드가 완료된다

  • pg_dumpall을 통한 업그레이드
    1. 덤프
      2. $ pg_dumpall > outputfile
    2. 서버 종료
      3. pg_ctl stop
    3. 업그레이드를 위해 기존 클러스터 디렉터리 이동
      4. mv /usr/local/pgsql /usr/local/pgsql.old
    4. 새 버전 설치
    5. 이전 pg_hba.conf, postgresql.conf 수정 내용 복원
    6. 서버 시작
      7. $ /usr/local/pgsql/bin/postgres -D /usr/local/pgsql/data
    7. 복원
      8. $ /usr/local/pgsql/bin/psql -d postgres -f outputfile
  • 새 서버를 다른 디렉터리에 설치하고, 이전 서버와 다른 포트에서 병렬 실행하는 경우
    • $ pg_dumpall -p 5432 | psql -d postgres -p 5433
  • pg_upgrade를 통한 업그레이드

    • 다른 메이저 버전으로 현재 위치에 마이그레이션할 수 있다.

암호화 옵션

  1. 패스워드 암호화

    2. 데이터베이스 유저 암호는 해시값만 저장되며, SCRAM, MD5 방식 로그인의 경우 서버에 전송되기 전에 클라이언트에서 해싱을 수행하므로 평문을 알아내기는 어렵다. SCRAM이 MD5보다 안전하다고 한다

  2. 특정 컬럼 암호화

    3. pgcrypto 모듈은 특정 필드의 암호화를 허용한다. 클라이언트가 키를 제공하고, 서버에서 복호화되어 클라이언트로 전송된다

  3. 데이터 파티션 암호화

    4. 스토리지 암호화는 파일 시스템 레벨 또는 블록 레벨에서 수행된다. 드라이브가 도난 당한 경우 데이터가 탈취되지 않도록 해준다

  4. 네트워크 암호화

    5. SSL 연결을 이용한다. 추가로 중간자 공격을 방지하기 위해 SSL 호스트 인증을 사용할 수 있다. 이 경우 서버, 클라이언트 양쪽에 추가적인 설정이 필요하다

  5. 클라이언트 측 암호화

    6. 서버를 신뢰할 수 없는 경우 클라이언트가 데이터를 암호화한다

SSL을 사용한 TCP/IP 연결 보호

  1. 기본 설정 : postgresql.confg에서 ssl을 on으로 설정

    2. SSL 모드에서 시작하려면 서버 인증서가 포함된 파일과 개인 키가 있어야 한다. 이 파일들은 각각 서버 데이터 디렉터리에서 server.crt, server.key일 것으로 예상되지만, ssl_cert_file, ssl_key_file 환경 변수로 위치와 이름을 지정할 수 있다

  2. OpenSSL 설정

    3. PostgreSQL은 openssl version-d로 출력되는 디렉터리에서 openssl.cnf를 찾는다. OPENSSL_CONF 환경 변수로 직접 지정할 수 있다.

    PostgreSQL에서 사용할 알고리즘은 postgresql.conf의 ssl_ciphers를 변경하여 지정할 수 있다

  3. 인증서 생성
    4. openssl req -new -x509 -days 365 -nodes -text -out server.crt -keyout server.key -subj "/CN=dbhost.yourdomain.com" chmod og-rwx server.key
