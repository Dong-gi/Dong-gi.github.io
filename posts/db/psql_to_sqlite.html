PostgreSQL 덤프로부터 SQLite3으로 마이그레이션하는 과정 정리
  1. SQLite에 테이블 생성
    2. PostgreSQL DDL Dump에서 아래 사항들을 수정하면 SQLite에서 그대로 테이블 생성에 이용할 수 있다.
    • true → t
    • false → f
    • now\(\), DEFAULT \('now'::text\)::date → '2000-01-01 00:00:00'
    • timestamp\(0\) with time zone → timestamp with time zone
    • DEFAULT '(\w)*'::text → DEFAULT '$1'
    * sqlite에서 sql파일 로드 : .read dump.sql
  2. PostgreSQL 데이터 덤프
    1. \copy table_name to 'file_name.csv' delimiter ',' csv;
      2. 뒤에 header 옵션도 붙일 수 있지만... 나중에 SQLite에서 임포트하면 헤더 행도 데이터로 들어가버린다..
    2. csv 편집
      3. csv에서는 null을 빈칸으로밖에 표현할 수 없는데... SQLite 임포트 시 이를 지원해주지 않는다. 때문에 null을 ""이나 기타 문자열로 변경...
  3. SQLite에서 import
    4. .mode csv
    .import file_name.csv table_name
  4. 기타 사항
    • SQLite Studio 프로그램은 import/export 기능을 UI로 지원.
    • SQLite에서 csv 출력
      • .headers on
      .mode csv
      .output file_name.csv
      [SELECT 쿼리]
      .quit
