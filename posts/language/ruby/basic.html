연산자

  • 동등 비교 ==

    • ==는 내용물이 같으면 true 반환. 인스턴스 주소 비교에는 equal? 메서드 이용

  • 동등 비교 ===
    • 기본적으로 ==과 동일
    • Pattern === value # 매칭되면 true
    • Range === value # range에 포함되면 true
    • Proc === params # params를 인자로 실행한 결과 반환
    • Module, Class === value # 인스턴스가 호환되면 true
  • push 연산 <<

    • 배열, String, Integers 각각에 대해 서로 다른 동작 수행

  • 비교 연산 a <=> b
    • 1 : a가 b보다 뒤에 있음(a > b)
    • 0 : a와 b는 동일 순서
    • -1 : a가 b보다 앞에 있음(a < b)
  • a ||= 10

    • 변수 a가 아직 존재하지 않거나, false, nil이면 10을 할당

    ↓ruby

    (defined?(a) && a) ? a : a = 10
  • 다중 할당

    • ↓ruby

    a, b = 1, 2 a, b = [1, 2, 3] a, b, c = [1, 2] # c == nil a, *rest = [1, 2, 3, 4, 5]
  • 배열 해체 *

    • ↓ruby

    def plus(a, b) a + b end plus *[1, 2]
  • nil safe 참조 연산 &.

    • receiver가 nil이면 참조를 수행하지 않음

  • 정규식 매칭 text =~ pattern, text !~ pattern
  • 사용자 정의 연산자

    • 정의 가능 연산자 : | ^ & <=> == =~ << -@ + - * / % ** ~ [] []= ` != 등

    ↓ruby

    class Point attr_accessor :x attr_accessor :y def initialize(x = 0, y = 0) self.x = x self.y = y end def -@ # 단항 연산자는 뒤에 @가 붙는다 self.x *= -1 self.y *= -1 self end end
  • 메서드 호출 시 . 대신 :: 사용 가능

제어식

  • if

    • ↓if 1

    r = if expression1 # false, nil 외의 모든 값은 true로 간주 # ... elsif expression2 # ... else # ... end # 마지막 평가된 식의 값을 반환

    ↓if 2

    ... if expression # true면 앞 문장 실행

    ↓if 3

    if expression1 then ... elsif expression2 then ... else ... end
  • unless

    • ↓unless 1

    r = unless expression # false면 실행 # ... else # elsif에 대응하는 키워드는 없다 # ... end # 마지막 평가된 식의 값을 반환

    ↓unless 2

    ... unless expression # false면 앞 문장 실행
  • case

    • when 절과 최초로 일치(===)하는 블록만 실행

    ↓case 1

    r = case expression # 마지막 평가값 반환 when 'value' # ... when 'value1', 'value2' # ... when /pattern/ # ... when ClassName # ... else # div end

    ↓case 2

    case # 식을 지정하지 않은 경우, 최초로 true가 되는 when 블록 실행 when expression1 then ... # then을 이용해 한 줄로 표기 가능 when expression2 then ... else ... end
  • while, until

    • ↓while

    while expression # ... true인 동안 반복 end ... while expression # true인 동안 앞문장 반복 begin # ... 적어도 1번은 실행된다 end while expression

    ↓until

    until expression # ... false인 동안 반복 end ... until expression # false인 동안 앞문장 반복 begin # ... 적어도 1번은 실행된다 end until expression
  • for

    • each 메서드를 가진 객체는 for 순회 가능

    ↓for 1

    for val in %w(word1 word2 word3) # ... end puts val # 계속 사용 가능

    ↓for 2

    for item in {a: 1, b: 2} puts item[0], item[1] # 각각 키-값 end for key, val in {a: 1, b: 2} # ... end
  • 반복 구문 대체 메서드

    • ↓Kernel#loop

    loop do # ... 무한 루프 end

    ↓Integer#times

    5.times do # ... end
  • 점프
    • break[ expression]
      1. 루프 안에서 : 표현식의 값으로 루프 종료
      2. yield 호출 안에서 : 표현식의 값으로 메서드 종료
    • next[ expression]
      1. 루프 안에서 : 다음 루프 실행
      2. yield 호출 안에서 : 표현식의 값으로 yield 종료
    • catch/throw

      • ↓ruby

      r = catch :label1 do # ... r = catch :label2 do if ? throw label2 return_value # 특정 catch 블록 종료 else throw label1 # 반환값 지정 없으면 nil end end
  • 예외 처리

    • 예외 처리 구문은 메서드, 클래스, 모듈 정의 모두에서 사용 가능

    ↓rescue 1

    r = begin # begin, rescue, else 절의 마지막으로 평가된 식의 값을 반환 1 / 0 raise 'message' # RuntimeError raise StandardError 'message' # StandardError rescue Class1, Class2 => e retry # 예외 발생 블록 처음부터 다시 실행 rescue # 클래스 지정없는 경우, StandardError로 간주 raise # 다시 전파 else # 예외 발생하지 않은 경우 실행 # ... ensure # 다른 언어의 finally 기능 # ... end

    ↓rescue 2

    expression1 rescue expression2 # expression1 도중 StandardError 발생한 경우, expression2 평가값 반환

IO

문자열 IO

  • gets(sep=$/) -> string | nil
    gets(limit) -> string | nil
    gets(sep, limit) -> string | nil
    • ARGV의 파일 또는 $stdin으로부터 문자열을 입력받는다
    • 반환값은 $_ 변수에 저장된다
    • limit을 지정하면 최대 limit 바이트만 읽는다
    • gets(sep=nil) -> 파일 한 개를 모두 읽어 반환
    • 파일 끝에 도달한 경우 nil 반환
  • #p(*obj) -> [obj, ...]

    • 각 인자의 inspect() 결과를 줄바꿈 문자와 함께 $stdout으로 출력

  • #puts(*obj) -> nil

    • 각 인자의 to_s() 결과를 줄바꿈 문자와 함께 $stdout으로 출력

  • #print(*obj) -> nil
    • 각 인자의 to_s() 결과를 $stdout으로 출력
    • $, != nil이면 구분자로 출력
    • $\ != nil이면 마지막에 출력
    • 인자가 없는 경우 $_ 출력
  • sprintf(format_string, *arguments) -> string

    • ↓C언어와 유사

    %[flags][width][.precision]type
  • printf(io, format_string, *arguments) -> nil
    printf(format_string, *arguments) -> nil

    • ↓ruby

    (io || $stdout).write(sprintf(format_string, *arguments))

실행/프로세스

  • eval

    • 동적으로 코드 평가

    ↓ruby

    def get_binding(str) binding end str = "hello" eval "str + ' Fred'" # "hello Fred" eval "str + ' Fred'", get_binding("bye") # "bye Fred"
  • exit(status=true)

    • SystemExit 예외를 발생하여 루비 프로세스 종료 절차를 시작한다

  • exit!(status=false)

    • 현재 루비 프로세스를 즉시 종료한다. 종료 핸들러는 호출되지 않는다

  • trap(signal, command) -> obj
    trap(signal) {| | block } -> obj
      • 시스템 신호 처리 핸들러 등록
    • signal
      1. "SIGALRM", "SIGUSR1" 등의 신호 이름
      2. 선두 "SIG"는 생략해도 된다
      3. :EXIT, "EXIT", 0에 대해 처리를 등록하면 루비 프로세스 정상 종료 전 호출된다
    • command
      1. "IGNORE", "SIG_IGN" : 신호 무시
      2. "DEFAULT", "RUBY_DFL" : 루비 기본 핸들러가 처리
      3. "SYSTEM_DEFAULT" : 시스템 기본 핸들러가 처리
      4. 그 외의 경우 command(Proc 인스턴스) 또는 block 호출

    ↓ruby

    trap(0, proc { puts "Terminating: #{$$}" }) trap("CLD") { puts "Child died" }
  • `cmd` -> string
    • cmd를 실행하여 결과를 반환한다
    • %x 문자열도 이 메서드를 이용한다
    • 종료된 프로세스는 $?에 저장된다
  • system([**env,] *command [,**opt]) -> true | false | nil
    • 명령을 실행하여 0으로 종료하면 true, 그 외 종료는 false, 실패는 nil
    • 종료된 프로세스는 $?에 저장된다
    • command 형태
      1. 셸에서 실행할 문장 전체
      2. command_name, arg1, arg2, ... # no shell
      3. [command_name, argv0], arg1, arg2 ... # no shell
    • env, opt로 주어지는 해시는 Kernel.exec(), Kernel.spawn()과 동일
  • exec([**env,] *command [,**opt])

    • 현재 프로세스에서 커맨드 실행 후 종료

  • spawn([**env,] *command [,**opt]) -> pid
    • 자식 프로세스로 명령을 실행하고 PID를 반환한다
    • env
      1. key => value : 환경변수 설정
      2. key => nil : 환경변수 삭제
    • opt

      • Kernel.html#method-i-spawn 참고

외부 소스 코드 로드

  • require(path)
    • 절대경로 이용 가능
    • $LOAD_PATH 내장변수의 디렉터리를 기준으로 한 상대경로 이용 가능
    • ./, ../ 상대경로는 실행 디렉터리 기준
    • 파일을 찾지 못한 경우, 설치된 gems 중 매칭되는 것이 있는지 찾는다
    • 같은 path에 대해 중복으로 실행하지 않는다
  • require_relative(string) -> true | false

    • 호출하는 파일이 존재하는 디렉터리를 기준으로 한 상대경로의 파일을 실행한다

  • load(filename, wrap=false) -> true
    • 같은 filename에 대해 반복 호출하면, 스크립트도 반복 실행된다
    • wrap=true로 지정하면 익명 모듈 안에서 실행한다
  • autoload(module, filename) -> nil

    • module(String or symbol)을 처음으로 접근하는 순간 require(filename) 실행

  • autoload?(name, inherit=true) -> String or nil

    • autoload로 name이 등록된 경우, 해당 filename 반환

    ↓ruby

    autoload(:B, "b") autoload?(:B) # "b"
