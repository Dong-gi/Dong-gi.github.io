태그

  • HTML 문서는 요소들의 트리로 표현된다
  • 태그는 요소의 타입을 나타내는 이름으로, 이름은 사용자가 임의로 지어도 된다
  • Non-replaced elements

    • <p>, <h1>과 같은 요소들은 여는 태그와 닫는 태그 사이에 텍스트나 다른 요소들이 포함될 수 있다

  • Replaced elements

    • <img>, <input>과 같은 요소들은 UI 위젯이나 이미지와 같은 객체로 대체된다. 대체된 객체들은 기본 스타일을 가지며, 브라우저에 따라 사용자가 부분적으로 스타일을 변경할 수 있다

  • Void elements

    • <br>, <img>, <link>와 같이 태그 하나로 열고 닫는(self-closing) 요소들로, 텍스트나 다른 요소를 자식으로 가질 수 없다. 태그 끝에 '/'를 적어도 된다

속성

  1. 요소의 속성은 여는 태그 내부에 이름="값" 쌍으로 표현한다. 태그와 마찬가지로 이름은 사용자가 임의로 지어도 된다

    2. 열고 닫는 따옴표는 '', "" 모두 이용 가능하다. 속성 값으로 따옴표를 포함하려면 &apos;('), &quot;(")를 이용해야 한다

  2. Boolean 속성

    3. disabled와 같이 켜고 끄는 속성은 값을 필요로 하지 않는다. 태그 안에 속성이 작성되지 않은 경우에만 false로 간주되고, 아래의 모든 경우 true로 간주된다

    ↓ html

    <input type="text" disabled /> <input type="text" disabled="disabled" /> <input type="text" disabled="true" /> <input type="text" disabled="false" />
  3. 모든 요소에서 유효한 속성, 일부 요소에서만 유효한 속성들이 존재한다
    • html 요소의 lang 속성 : 문서의 주 언어
    • 기타 텍스트를 포함한 요소의 lang 속성 : 요소의 부 언어
    • title 속성 : 요소에 대한 툴팁 -- 커서를 올려둔 경우만 표시되므로, 사용이 권장되지는 않음
    • disabled 속성 : 지정한 input 요소를 수정할 수 없음
    • id 속성 : 요소의 고유한 id
    • class 속성 : 요소가 속한 그룹

HTML 문서 구조

  • <!DOCTYPE html>

    • 모든 HTML 문서의 서두엔 위 문구가 있어야 한다. 이는 HTML 요소가 아니라 DTD(Document Type Definition)의 일종이다. DOCTYPE은 브라우저가 문서를 어떤 모드로 렌더링할 지를 결정하는 데 이용된다. DOCTYPE html은 표준 모드 렌더링을 지시한다

  • <html>
    • <html> 요소는 HTML 문서의 루트 요소로, <head>, <body> 요소의 부모다
    • lang 속성을 지정하여 페이지의 기본 언어를 설정하는 것이 권장된다

      • ↓ html

      <html lang="en-US">
  • <head>

    • <head> 요소는 문서 또는 웹앱에 대한 메타데이터를 기술한다. 검색 엔진에 제공할 페이지 설명, CSS 스타일 정보, 문자 집합 선언 등

  • <body>

    • <body> 요소는 페이지에 표시할 모든 요소를 포함한다

Viewport

뷰 영역의 너비와 관계없이 컨텐츠를 보기 좋게 표현(반응형 웹)하기 위한 기본적인 설정을 기술한다. Viewport 설정은 사이트의 접근성 품질 검사 항목 중 하나로, 콘텐츠를 확대/축소 가능하고, 최대 크기 제한이 없으면 검사를 통과한다

↓ html

<meta name="viewport" content="width=device-width, initial-scale=1, user-scalable=yes">

Description

문서에 대한 짧은 설명으로, 검색 엔진의 검색 결과에 이용된다. 이전에는 키워드 설정도 따로 존재했지만, 스팸이 많아 더 이상 사용되지 않는다

↓ html

<meta name="description" content="문서에 대한 짧은 요약">

Script

스크립트는 기본적으로 CSS와 마찬가지로 다운로드 및 해석되기 전까지 렌더링이 차단되고, 이에 더해 다른 리소스의 다운로드까지 중지된다. 이러한 지연을 방지하기 위한 속성이 존재한다

  • defer : HTML 렌더링을 차단하지 않는다. 스크립트는 렌더링이 완료된 후 실행된다
  • async : 스크립트를 다운로드하는 동안은 HTML 렌더링을 차단하지 않는다. 다운로드 후 스크립트 실행 중에는 차단한다
  • 위 속성들은 외부 스크립트를 사용하는 경우 적용 가능하며, 인라인 스크립트에는 적용되지 않는다

CSP; Content-Security-Policy

  • 참고 자료 : content-security-policy

    • CSP에는 버전이 존재하므로, 오래된 브라우저는 최신 기능을 지원하지 않을 수 있다. 위 사이트에 CSP 버전 별로 대응하는 브라우저 버전들이 나열되어 있다

  • 원래는 HTTP 응답 헤더에 이용됐지만, <meta> 태그로도 설정 가능하다

    • 일부 directive들(frame-ancestors, report-uri, sandbox 등)은 헤더로만 설정 가능하다

Directive

CSP 값은 1개 이상의 지시문(directive)으로 구성되고, 위 예시와 같이 각 directive들은 ;으로 구별한다

  • default-src

    • 명시적으로 설정하지 않은 리소스 획득 정책에 대한 기본값으로 이용된다. JavaScript, 이미지, CSS, 폰트, XHR 등이 이에 포함되지만, 모든 것에 대한 기본값으로 이용되는 것은 아니다

  • script-src

    • JavaScript 및 웹 어셈블리 출처 제한

  • style-src

    • CSS 출처 제한

  • img-src

    • 이미지(favicon 포함) 출처 제한

  • connect-src

    • XHR, WebSocket, fetch(), <a ping>, EventSource를 통해 연결할 수 있는 출처 제한 -- 제한된 경우 브라우저는 400 응답 코드를 생성

  • font-src

    • 웹 폰트(@font-face) 출처 제한

  • object-src

    • <object>, <embed>, <applet>과 같은 플러그인 출처 제한

  • media-src

    • <audio>, <video>, <track>과 같은 미디어 출처 제한

  • frame-src

    • 프레임 출처 제한. CSP Level 2에서 deprecated 상태로 변경됐고, 대신 child-src를 이용하라고 안내

  • sandbox

    • 페이지가 해당 sandbox 속성을 가진 iframe을 통해 로드된 것처럼 취급된다 -- 동일 출처 정책 적용, 팝업 제한, 플러그인 제한, 스크립트 실행 제한 등

    샌드박스 환경에서 허용하고 싶은 것들은 플래그로 설정하면 된다 : allow-forms, allow-scripts, allow-popups 등

    ↓ sandbox directive 예

    sandbox allow-forms allow-scripts;
  • report-uri

    • CSP로 리소스가 차단된 경우 POST 보고할 URI. CSP Level 3에서는 deprecated 상태로 변경됐고, report-to를 이용하라고 안내

  • child-src

    • 웹 워커, 페이지에 포함된 프레임 컨텐츠(<frame>, <iframe>) 출처 제한

  • form-action

    • <form> action으로 설정할 수 있는 출처 제한

  • frame-ancestors

    • 임베딩되는 리소스들(<frame>, <iframe>, <object>, <embed>, <applet>)에 대한 출처 제한. 이 directive 값을 'none'으로 설정하면 X-Frame-Options: DENY 헤더를 설정한 것과 거의 동등한 효과.

    참고로 X-Frame-Options 헤더는 deprecated 상태이며, CSP 헤더를 이용할 것이 권장됨.

  • plugin-types

    • <object>, <embed>, <applet>과 같은 플러그인 MIME 타입 제한

  • manifest-src

    • 웹앱 매니페스트 파일 출처 제한

  • worker-src

    • 웹 워커 출처 제한

제목과 단락

  • 제목(heading)은 그 수준에 따라 <h1>(최상위 제목)부터 <h6>(최하위 제목)까지 존재한다
  • 텍스트 단락(paragraph)은 <p> 태그를 이용해 구분한다

하이퍼링크

  • <a>(anchor) 태그는 서로 다른 페이지(리소스) 사이의 연결을 표시하는 요소로, 웹을 웹답게 만드는 핵심 요소다
  • href 속성 : 이동할 위치나 주소

    • # : 현재 페이지의 특정 위치(id)로 이동하기 위한 표기. '#here'은 현재 페이지의 HTML 요소 중 id가 'here'인 것을 가리킨다

  • 링크를 이용해 이메일 작성으로 이동할 수 있다

    • ↓ html

    <a href="mailto:id1@domain.com,id2@domain.com?cc=id3@domain.com,id4@domain.com&subject=mail%20subject&body=mail%20body">이메일 보내기</a>
    이메일 보내기
  • target 속성 : _blank(새탭이나 새창에서 열기), _self, _parent, _top, framename
  • 링크 상태를 나타내는 스타일 속성이 별도로 존재한다

    • link, visited, hover, active

텍스트 강조

  1. <strong> : 중요한 텍스트. 굵게 표시
  2. <em> : 강조된 단어. 기울여 표시

    3. 예를 들어, 구어에서 특정 단어를 강조하여 비꼬는 뉘앙스를 주는 경우, 해당 단어를 <em> 태그로 감싸기에 적합하다

  3. <mark> : 형광펜으로 눈에 띄게 표시
  4. <del> : 취소선
  5. <sub> : 아랫첨자
  6. <sup> : 윗첨자

리스트

  • 번호를 매기지 않은 리스트 (unordered list) : <ul>
  • 번호를 매긴 리스트 : <ol>

  • 리스트 항목(list item) : <li>

시맨틱 태그

HTML5 전 웹페이지들은 table 태그를 주로 사용하여 디자인했다. 중첩된 table들로 이루어진 웹페이지는 검색 엔진을 비롯한 프로그램이 기계적으로 의미를 찾기 어려워 HTML5는 문서에 대한 의미를 태그가 포함하는 방향을 모색했다

  • header : 컨텐츠를 소개하는 부분으로, body 바로 아래에 정의된 경우 전체 페이지에 대한 소개, 특정 컨텐츠 아래에 정의된 경우 해당 컨텐츠에 대한 소개를 정의
  • nav : 페이지 내 주요 요소들로의 이동을 정의
  • main : 페이지의 컨텐츠를 감싸는 요소. 페이지당 하나만 사용하고, body 바로 아래에 정의하는 것을 권장
  • article : 페이지의 나머지 부분이 없어도 독립적으로 의미를 갖는 컨텐츠 블록
  • section : article과 유사하게 컨텐츠를 감싸는데, 기능적으로 연관 있는 요소들을 함께 묶을 때 이용. 각 section은 제목(h1~h6)으로 시작하는 것이 권장
  • aside : 메인 컨텐츠에 대한 추가 정보 제공 (예. 참고자료 목록, 저자 소개)
  • footer : 페이지를 끝맺는 컨텐츠

    1. 조절점1(x1, y1), 조절점2(x2, y2), 선분1(마지막 정점-조절점1), 선분2(조절점1-조절점2)를 생각한다
    2. 선분1, 선분2 모두와 거리가 radius만큼 떨어진 정점2를 생각한다. 그러면 정점2에서 반지름 radius인 원을 그렸을 때 선분1, 선분2와의 교점1, 교점2가 존재한다
    3. lineTo(교점1)
