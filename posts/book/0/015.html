개요

  • 컴퓨터의 기본적인 기능들은 네 가지로 분류된다
    • 데이터 처리
    • 데이터 저장
    • 데이터 이동
    • 제어
  • 운영체제는 두 가지 주요 목적을 가지고 있다
    1. 제멋대로 동작하는 응용프로그램들이 하드웨어를 잘못 사용하는 것을 막고
    2. 응용프로그램들이 단순하고 균일한 메커니즘을 사용해 복잡한 저수준 하드웨어 장치들을 조작
  • 운영체제는 이 두 가지 목표를 추상화를 통해 달성하고 있다
    • 파일 : 입출력장치의 추상화
    • 가상메모리 : 메인 메모리와 디스크 입출력장치의 추상화
    • 프로세스 : 프로세서, 메인 메모리, 입출력장치 모두의 추상화
  • 프로세스는 실행 중인 프로그램에 대한 운영체제의 추상화다. 프로세스들은 하나의 시스템에서 동시에 실행될 수 있으며, 각 프로세스는 하드웨어를 배타적으로 사용하는 것처럼 느낀다.
  • 커널은 운영체제 코드의 일부분으로 메모리에 상주한다. 프로그램이 운영체제에 의한 작업을 요청하면, 컴퓨터는 시스템 콜을 실행해서 커널에 제어를 넘겨준다. 그러면 커널은 요청된 작업을 수행하고 프로그램에 리턴한다. 커널은 별도의 프로세스가 아니다 -- 모든 프로세스를 관리하기 위해 시스템이 이용하는 코드와 자료 구조의 집합이다
  • 가상메모리는 프로세스들이 메인 메모리 전체를 독점적으로 사용하고 있는 것 같은 환상을 제공하는 추상화다. 각 프로세스는 0부터 시작하는 주소 공간에서, 몇 개의 정의된 영역을 갖는다

컴퓨터의 발전 과정과 성능

컴퓨터의 간략한 역사

제1세대: 진공관

  • ENIAC : 프로그래밍을 위해 손으로 스위치를 조작하고 케이블을 끼우거나 빼야했다
  • 폰 노이만 기계 : Stored-program 개념을 제안했다 -- 컴퓨터는 기억장치의 명령어를 읽어들여 실행만 하면 되고, 기존 프로그램을 변경하는 과정도 쉬워진다

제2세대: 트랜지스터

  • IBM 7094
    • 데이터 채널 사용 : I/O 전용 특수목적 프로세서가 관련 명령어를 처리하고, CPU에는 결과를 통지함으로써 CPU의 부담을 줄인다
    • 멀티플렉서 사용 : CPU와 데이터 채널들의 메모리 접근 순서를 결정하여 서로 독립적으로 동작하게 한다

제3세대~: 집적회로

  • DEC PDP-8 : 데이터 버스 사용 -- 주소와 데이터를 전송하는 데 사용되는 공용 신호 선들로, 사용은 CPU에 의해 제어된다
  • 인텔 4004 : 최초의 마이크로프로세서 -- CPU 모든 구성요소가 한 개의 칩에 존재

성능을 위한 설계

마이크로프로세서의 속도 향상을 위한 기술

  • 분기 예측 : 다음에 처리될 명령을 예측하여 미리 저장함으로써 명령어 인출 시간 축소
  • 데이터 흐름 분석 : 명령어 사이의 의존성을 해치지 않으면서 실행 순서를 변경하여 대기 시간 축소
  • 추측 실행 : 프로그램 실행 지점 이후의 명령어를 미리 실행하고 임시 저장하여 유휴 시간 축소

칩 조직과 구조상의 발전

  • 하드웨어 속도 증가 : 클록 증가, 칩 밀도 증가 -> 단위 면적 당 전력 및 발열량 증가, 저항 증가 등 한계 존재
  • 프로세서 - 메모리 사이의 캐시 사용 : 3-level 캐시 사용으로 메모리 IO 속도 격차 해소 -> 캐시 증설로 인한 성능 향상도 한계 존재
  • 프로세서 조직/구조 변경
    • 파이프라이닝 : 서로 다른 단계에 있는 명령어 동시 실행
    • 슈퍼스칼라 : 프로세서 내부에 여러 파이프라인을 두어, 독립적인 명령어 동시 실행
    • 멀티코어 프로세서 : 각 프로세서마다 서로 다른 프로그램 실행 or 멀티코어를 효과적으로 실행하는 프로그램 실행

최상위에서 본 컴퓨터의 기능 및 상호연결

버스 상호연결

버스 : 주소 선 + 데이터 선 + 제어 선 + 전력 선

전파 지연과 병목 현상을 해소하기 위해 버스 계층을 두고(예. 시스템 버스 / 고속 버스 / 저속 확장 버스), 각 버스 사이는 브릿지로 연결할 수 있다

내부 기억장치

반도체 주기억장치

  • DRAM; Dynamic RAM : 축전기로 전하를 저장하므로 주기적으로 재충전이 필요

    단위 면적당 용량과 가격이 SRAM보다 저렴하여 대용량 기억장치로 이용

  • SRAM; Static RAM : 플립-플롭을 이용하므로 재충전 불필요

    DRAM보다 성능이 좀 더 좋아 캐시로 이용

외부 기억장치

RAID

RAID 0

  • 모든 디스크가 하나의 논리적 디스크를 구성한다
  • 데이터는 각 디스크에 라운드-로빈 방식으로 분산 저장된다
  • 디스크 하나라도 결함이 생기면 서비스 불가

RAID 1

  • 하나의 논리적 디스크의 내용을 모든 디스크가 공유한다
  • 전체 디스크 n개 중 n-1개에 결함이 생겨도 서비스 가능하다
  • RAID 중 가장 큰 쓰기 부하가 발생한다

RAID 2

  • RAID 2, 3은 병렬 액세스 기법을 사용한다
  • 데이터 디스크들이 하나의 논리적 디스크를 구성하고, 에러 검출과 데이터 복구를 위한 패리티 디스크들이 추가로 필요하다
  • 전형적으로 해밍 코드가 사용되고, 데이터 디스크 수의 로그에 비례하여 패리티 디스크가 필요하다
  • 패리티 비트를 위해, IO 처리 시 모든 디스크의 동일한 비트 위치를 읽고 쓴다

RAID 3

  • RAID 2, 3은 병렬 액세스 기법을 사용한다
  • 데이터 디스크들이 하나의 논리적 디스크를 구성하고, 패리티 디스크 하나를 추가로 사용한다
  • 패리티 비트는 단순히 모든 디스크의 동일한 비트 위치의 XOR로 계산된다

    디스크 1개에 결함이 생겨도 서비스 가능

RAID 4

  • RAID 3과 유사하지만, 독립적인 액세스 기법을 사용한다
  • 패리티 계산 시 모든 디스크를 읽지 않는다 -- 대신 패리티 디스크의 부하가 크다

    (새 피리티 비트) = (이전 패리티 비트) xor (이전 데이터 비트) xor (새 데이터 비트)

RAID 5

  • RAID 4-6은 독립적인 액세스 기법을 사용한다
  • RAID 4와 유사하지만, 패리티 스트립은 모든 디스크에 분산된다

    RAID 4의 패리티 디스크 병목을 미연에 방지한다

RAID 6

  • RAID 4-6은 독립적인 액세스 기법을 사용한다
  • RAID 5와 유사하지만, 패리티 함수 1개를 추가로 사용한다

    따라서 디스크도 1개 추가로 필요

  • 추가된 패리티 덕분에 디스크 2개에 결함이 생겨도 서비스 가능

확장 RAID

  • RAID 10 : 두 개의 RAID 1로 RAID 0 구성
  • RAID 01 : 두 개의 RAID 0로 RAID 1 구성
  • RAID + Spare : 디스크 결함 시 즉시 복구에 사용될 여유 디스크를 예비

정보의 표현과 처리

  • 리틀 엔디언 컴퓨터에서 빅 엔디언 컴퓨터로 네트워크를 통해 데이터를 보내는 경우 수신된 워드들 내 바이트 순서가 바뀔 수 있다. 때문에 송신측은 네트워크 표준 인코딩(빅 엔디언)으로 데이터를 보내고, 수신측은 네트워크 표준 데이터를 자신의 내부 표현방식으로 변환한다.

IEEE 부동소수점

Rounding Mode

참고 자료 : IEEE_754#Rounding_rules

  1. 가까운 값으로 근사
    • Round to nearest, ties to even

      가장 가까운 유효숫자로 근사한다. 만일 두 수의 중간값인 경우, 짝수로 근사한다. 통계치의 바이어스를 줄이는 데 효과적이다

    • Round to nearest, ties away from zero

      가장 가까운 유효숫자로 근사한다. 만일 두 수의 중간값인 경우, 0에서 멀어지도록 근사한다

  2. 일정한 방향으로 근사
    • Round toward 0(a.k.a. truncation)

      0에 가까운 방향으로 근사한다

    • Round toward +∞(a.k.a. rounding up, ceiling)

      양의 방향으로 근사한다

    • Round toward -∞(a.k.a. rounding down, floor)

      음의 방향으로 근사한다

연산

  • 부동소수점 연산은 근삿값을 도출한다

    예. 덧셈의 교환법칙은 성립하지만 결합법칙은 성립하지 않는다. 따라서 아래의 코드를

    ↓ text

    x = a + b + c; y = b + c + d;

    아래와 같이 최적화하는 것은 오류다

    ↓ text

    t = b + c; x = a + t; y = t + d;
  • IO 처리에 모든 디스크가 참여
  • 디스크들은 동기화되어, 모든 디스크 헤드가 동일한 위치에 존재
  • 따라서 대용량 처리 성능은 좋지만, 요청 횟수가 높은 환경에는 부적합
  • 각 디스크가 독립적으로 작동하므로, 독립적인 IO 요청은 병렬로 처리 가능
  • 요청 횟수가 높은 환경에 적합
