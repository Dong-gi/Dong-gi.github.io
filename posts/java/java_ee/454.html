WebServlet 애너테이션으로 배포설명자 web.xml 대신 간결하게 서블릿을 기술할 수 있다.
* name : 배포설명자의 servlet-name에 해당.
* urlPatterns : 배포설명자의 url-pattern들의 배열에 해당.
* value : urlPatterns 요소와 동일하게 동작하기 때문에, 서블릿 매핑만을 기술할 거라면 편하게 WebServlet("/hello")처럼 기술할 수 있다.
* initParams : 배포설명자의 init-param들의 배열에 해당. GenericServlet::getInitParameter로 참조할 수 있으며, 이 메서드는 ServletConfig::getInitParameter를 호출한다. ServletConfig는 getServletContext 메서드로 참조할 수 있다.

아래는 비교를 위한 배포설명자 예시. 
<servlet>
    <servlet-name>hello5-servlet</servlet-name>
    <servlet-class>donggi.Hello2</servlet-class>
    <init-param>
        <param-name>name</param-name>
        <param-value>Default Name</param-value>
    </init-param>
</servlet>
<servlet-mapping>
    <servlet-name>hello5-servlet</servlet-name>
    <url-pattern>/hello5</url-pattern>
</servlet-mapping>
