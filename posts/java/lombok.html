val

  1. 자동 타입 추론을 통한 타입 명시 생략을 가능하게 해준다.
  2. 할당 연산이 있는 지역 변수와 foreach에서만 사용 가능.
  3. val로 선언된 변수는 final이다.
  4. JDK 10부터 var 키워드가 추가되었으므로, 해당 버전부터는 필요없음.

@NonNull

  1. 메서드, 생성자의 매개변수에 null 검사를 추가해준다. null이면 NullPointerException 발생

@CleanUp

  1. 지역 변수가 유효 스코프를 벗어나면 close()를 자동으로 호출한다.
  2. try ~ finally와 같은 방식으로 호출함을 보장한다.
  3. AutoCloseable을 구현할 필요가 없으며, 필요하다면 close 외 다른 인자 없는 메서드를 이용할 수 있다.

@Getter, @Setter

  1. 필드의 기본적인 getter와 setter를 자동 구현해준다.
  2. 기본 시그니처는 [TYPE] get[Name], boolean is[Name], void set[Name]
  3. 기본 접근 제한은 public. 가능한 설정은 PUBLIC, PROTECTED, PACKAGE, PRIVATE
  4. 클래스에 이용하는 경우, 모든 non-static 필드에 getter/setter를 생성한다.
  5. 클래스의 getter/setter 설정을 무시하려면 각 필드에 AccessLevel.NONE을 설정한다.
  6. @Getter(lazy=true)의 경우, 처음 호출할 때 초기화되며, 이후 값은 캐시된다.

@ToString

  1. 클래스명, 모든 non-static 필드값을 포함한 문자열을 반환하는 toString 메서드를 생성한다.
  2. 필드 이름을 포함하고 싶다면 includeFieldNames 필드를 사용(기본 true)
  3. 출력에 제외할 필드는 @ToString.Exclude로 마킹한다.
  4. callSuper 옵션으로 수퍼클래스 toString()을 결과에 포함
  5. @ToString.Include(name = "다른 표시 이름")
  6. @ToString.Include(rank = 3) // 클수록 먼저 출력

@EqualsAndHashCode

  1. 모든 non-static, non-transient 필드를 이용하는 equals, hashCode 메서드를 생성한다.
  2. callSuper를 설정하는 경우, 수퍼클래스의 hashCode() 값을 해시 알고리즘에 이용한다.

@NoArgsConstructor, @RequiredArgsConstructor, @AllArgsConstructor

  1. NoArgsConstructor는 기본생성자를 생성한다. final 필드로 인해 예외가 발생할 수 있으며, 이 경우 force 옵션을 사용하면 기본값으로 초기화된다.
  2. 생성된 팩터리 메서드는 제네릭을 추론한다.

@Data

  1. Target : ElementType.TYPE
  2. 모든 필드에 대한 @ToString, @EqualsAndHashCode, @Getter 생성
  3. 모든 non-final 필드에 대해 @setter생성
  4. @RequiredArgsConstructor 생성

@Value

  1. final한 @Data

@Builder

  1. 메서드에 @Builder가 붙으면 아래의 7가지가 생성된다.
    • (클래스 내부) 클래스명 + "Builder"를 이름으로 갖는 static inner class
    • (클래스 내부) builder() 메서드
    • (Builder 클래스 내부) 타깃 메서드 인자들과 같은 non-static, non-final 필드
    • (Builder 클래스 내부) 기본 생성자(AccessLevel.PACKAGE)
    • (Builder 클래스 내부) 각 필드의 이름으로 한 setter 메서드. 반환은 Builder 객체 자체
    • (Builder 클래스 내부) 타깃 메서드를 호출하는 build() 메서드
    • (Builder 클래스 내부) toString() 메서드
  2. 클래스에 @Builder가 붙으면, @AllArgsConstructor(access = AccessLevel.PACKAGE)를 클래스에 붙이고 생성된 생성자에 @Builder를 붙이는 것과 같다.

    3. 명시적으로 작성된 생성자가 없을 때만 유효하다. 그러한 생성자가 있다면 생성자에 @Builder를 붙이면 된다

  3. 컬렉션 필드에 @Singular를 적용하면 원소를 하나씩 추가하여 최종적으로 변경 불가능한 Collection 객체로 초기화할 수 있다.

    4. 원소는 필드의 단수명을 이름으로 하는 메서드를 이용해 추가할 수 있으며, 일반적인 단수명은 자동으로 추론하지만, 그 외의 경우에는 @Singuler("axis")처럼 value로 넘겨줘야 한다.

      지원하는 Collection

    • java.util : Iterable, Collection, List, Set, SortedSet, NavigableSet, Map, SortedMap, NavigableMap
    • com.google.common.collect : ImmutableCollection, ImmutableList, ImmutableSet, ImmutableSortedSet, ImmutableMap, ImmutableBiMap, ImmutableSortedMap, ImmutableTable
  4. toBuilder 옵션을 이용해 기존 객체로부터 시작하는 toBuilder 메서드를 생성할 수 있다.
  5. @Builder.ObtainVia를 필드/매개변수에 적용하면 해당 메서드로부터 값을 얻는다
  6. 기타 옵션 : builderClassName, buildMethodName, builderMethodName
  7. Since 1.16.16 : @Builder.Default를 필드/매개변수에 붙이면 빌드 세션에서 초기화하지 않아도 기본값으로 설정한다

@SneakyThrows

  1. org.apache.commons.lang3.exception.ExceptionUtils#rethrow보다 강하게 throws 절을 사용하지 않아도 되게 해준다.
  2. Runnable처럼 불필요하게 외부로 예외를 던지거나, 예외가 결코 일어나지 않는 경우 등에 한하여 사용함이 바람직하다

@Synchronized

  1. synchronized 키워드의 편리한 축약형.
  2. 인스턴스 메서드에 붙으면 $lock 객체를, static 메서드에 붙으면 $LOCK 객체를 이용해 키워드와 동일한 동작을 한다.
  3. @Synchronized("ownFieldName")처럼 다른 객체를 이용해 동기화할 수도 있다.

@With Since 1.18.10

  1. 변경 불가능한 객체의 일부 필드만 변경한 복사본을 반환하는 메서드를 생성한다
  2. non-static 필드 전체를 인자로 받는 생성자가 필요하다
  3. @Setter와 마찬가지로 접근 수준을 제한할 수 있다
  4. 실험 기능 : 자동 생성된 메서드에 애너테이션을 적용하고 싶다면 onMethod=@__({@AnnotationsHere})를 적용한다

@Log

  1. 클래스에 log 필드를 자동으로 생성해준다
  2. @CommonsLog : private static final org.apache.commons.logging.Log log = org.apache.commons.logging.LogFactory.getLog(LogExample.class);
  3. @Flogger Since 1.16.24 : private static final com.google.common.flogger.FluentLogger log = com.google.common.flogger.FluentLogger.forEnclosingClass();
  4. @JBossLog : private static final org.jboss.logging.Logger log = org.jboss.logging.Logger.getLogger(LogExample.class);
  5. @Log : private static final java.util.logging.Logger log = java.util.logging.Logger.getLogger(LogExample.class.getName());
  6. @Log4j : private static final org.apache.log4j.Logger log = org.apache.log4j.Logger.getLogger(LogExample.class);
  7. @Log4j2 : private static final org.apache.logging.log4j.Logger log = org.apache.logging.log4j.LogManager.getLogger(LogExample.class);
  8. @Slf4j : private static final org.slf4j.Logger log = org.slf4j.LoggerFactory.getLogger(LogExample.class);
  9. @XSlf4j : private static final org.slf4j.ext.XLogger log = org.slf4j.ext.XLoggerFactory.getXLogger(LogExample.class);
  10. @CustomLog Since 1.18.10 : private static final com.foo.your.Logger log = com.foo.your.LoggerFactory.createYourLogger(LogExample.class);

    11. lombok.config 파일에 lombok.log.custom.declaration = com.foo.your.Logger com.foo.your.LoggerFactory.createYourLog(TYPE) 필요
