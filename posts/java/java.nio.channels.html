interface Channel extends Closeable Since 1.4
  1. 채널이란 IO를 수행할 수 있는 개체에 대한 연결이다.
  2. boolean isOpen()
interface ReadableByteChannel extends Channel Since 1.4
  1. int read(ByteBuffer dst)
    2. dst.remaining()만큼 읽어 dst에 저장. 읽은 바이트 개수를 반환. 스트림의 끝이면 -1.
interface ScatteringByteChannel extends ReadableByteChannel Since 1.4
  1. 채널의 바이트들을 여러 버퍼에 순서대로 저장. 네트워크 헤더나 파일 메타데이터처럼 고정 길이의 헤더를 읽는 데 유용.
  2. long read(ByteBuffer[] dsts, int offset, int length)
    3. » dsts[offset]부터 length개의 버퍼가 각각의 remaining()만큼씩 바이트를 읽어 가져간다.
  3. long read(ByteBuffer[] dsts) : == read(dsts, 0, dsts.length)
interface WritableByteChannel extends Channel Since 1.4
  1. int write(ByteBuffer src)
    2. src.remaining() 부분을 채널에 쓴다.
interface GatheringByteChannel extends WritableByteChannel Since 1.4
  1. 버퍼들의 내용을 순서대로 채널에 쓴다.
  2. long write(ByteBuffer[] srcs, int offset, int length) srcs[offset]부터 length개의 버퍼의 remaining() 부분을 채널에 쓴다.
  3. long write(ByteBuffer[] srcs) : == write(srcs, 0, srcs.length)
interface ByteChannel extends ReadableByteChannel, WritableByteChannel Since 1.4
  1. 바이트 단위로 읽기/쓰기 모두 가능한 채널.
interface SeekableByteChannel extends ByteChannel Since 1.7
  1. 현재 위치를 기억하는 바이트 채널.
  2. long position()
  3. SeekableByteChannel position(long newPosition) : position을 변경하고 그대로 반환.
  4. long size() : 채널에 연결된 개체의 크기 반환
  5. SeekableByteChannel truncate(long size)
    6. » 채널에 연결된 개체를 size로 크기 조정. size가 현재 크기보다 작은 경우, 나머지는 버려진다. size가 현재 크기 이상인 경우 아무 변경 없다.
    » 현재 position이 size보다 큰 경우 size로 조정된다.
interface AsynchronousChannel extends Channel Since 1.7
  1. 비동기 IO는 보통 다음의 형태로 일어난다.
    2. » Future<V> operation(...) // operation : read, write, ...
    » void operation(... A attachment, CompletionHandler<V, ? super A> handler)
  2. 비동기적으로 close()가 수행되며, 이후 발생하는 IO 작업은 AsynchronousCloseException 발생
interface AsynchronousByteChannel extends AsynchronousChannel Since 1.7
  1. 비동기 IO를 지원하는 바이트 채널. 인자에 이용되는 ByteBuffers는 쓰레드-안전하지 않음에 유의.
  2. <A> void read(ByteBuffer dst, A attachment, CompletionHandler<Integer, ? super A> handler)
    3. » handler의 첫 인자는 읽은 바이트 수. 기본 동작은 ReadableByteChannel::read와 동일. 쓰기가 완료(실패)되면 handler가 호출된다.
  3. Future<Integer> read(ByteBuffer dst)
    4. » Fureture의 get 메서드로 바이트 수를 얻을 수 있다.
  4. <A> void write(ByteBuffer src, A attachment, CompletionHandler<Integer, ? super A> handler)
  5. Future<Integer> write(ByteBuffer src)
interface NetworkChannel extends Channel Since 1.7
  1. 네트워크 소켓에 대한 채널.
  2. NetworkChannel bind(SocketAddress local)
  3. SocketAddress getLocalAddress()
  4. <T> NetworkChannel setOption(SocketOption<T> name, T value)
  5. <T> T getOption(SocketOption<T> name)
  6. Set<SocketOption<?>> supportedOptions()
