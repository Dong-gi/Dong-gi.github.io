Netty?

  • 비동기, event-driven 네트워크 애플리케이션 프레임워크
  • FTP, SMTP, HTTP, WebSocket 등 주요 프로토콜 구현 제공

주요 인터페이스

Channel

I/O 작업을 지원하는 네트워크 연결. 모든 I/O 작업은 비동기적으로 실행되어 ChannelFuture를 즉시 반환한다

ChannelFuture

  • Channel의 I/O 작업 결과
  • 가능한 상태 : Uncompleted(!isDone), Success(isDone && isSuccess), Failed(isDone && cause != null), Cacelled(isDone && isCancelled)
  • addListener(s) 메서드를 이용해 콜백 설정 가능

ChannelHandler

  • I/O 이벤트 수신
  • ChannelInboundHandler, ChannelOutboundHandler

ChannelHandlerContext

  • ChannelHandler는 Context를 통해 ChannelPipeline 및 다른 핸들러와 상호작용한다.

    • ChannelHandler는 여러 ChannelPipeline에 등록될 수 있으므로, Context 또한 여러 개 가질 수 있다

  • 이벤트 통지 : ChannelPipeline 상의 다음 핸들러에 이벤트 전달
  • Pipeline 수정 : pipeline()으로 ChannelPipeline를 획득하여 직접 수정 가능
  • 나중에 사용 : 획득한 ChannelHandlerContext는 다른 스레드에서도 사용 가능하다
  • 상태 저장 : attr(AttributeKey)로 상태 저장 및 공유

ChannelPipeline

  • 하나의 Channel 상에서 inbound 이벤트와 outbound 작업을 처리하는 ChannelHandler 리스트

    • 각 Channel 생성 시 자동으로 ChannelPipeline을 신규로 하나 갖는다

  • Inbound 핸들러는 순서대로, outbound 핸들러는 역순으로 실행된다
