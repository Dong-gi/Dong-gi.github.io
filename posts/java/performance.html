JVM 파라미터

  1. Boolean flag
    2. XX:+FlagName XX:-FlagName
  2. Value flag
    3. -XX:FlagName=value

주요 플래그

  • 최대 코드 캐시 크기 : 기본값 32 or 48M
    • -XX:ReservedCodeCacheSize=N
  • 힙 전체 크기
    • -XmsN # 초기값 -XmxN # 최대값
  • 힙 = 영 제너레이션 + 올드 제너레이션

    • 영 제너레이션이 큰 경우 : 덜 자주 수집되고, 더 적은 객체가 올드 제너레이션으로 이동. 대신 올드 제너레이션이 작아 풀 GC는 더 자주 일어난다

    -XX:NewSize=N # 영 제너레이션 초기 크기 -XX:MaxNewSize=N # 영 제너레이션 최대 크기 -XmnN # NewSize, MaxNewSize 동일하게 설정 -XX:NewRatio=N # (영 제너레이션 초기 크기) = (힙 초기 크기) / (1 + NewRatio) -XX:+HeapDumpOnOutOfMemoryError # 메모리 부족시 힙 덤프 생성 -XX:HeapDumpPath={path} # 힙 덤프 작성 위치 -XX:+HeapDumpBeforeFullGC # 풀 GC 전에 힙 덤프 생성 -XX:+HeapDumpAfterFullGC # 풀 GC 후에 힙 덤프 생성 -XX:GCHeapFreeLimit=N # 풀 GC로 힙의 N% 이상 해제되면 OK
  • GC 병렬 스레드 개수
    • -XX:ParallelGCThreads=N

Java 옵션(JAVA_OPTS)

  1. user.timezone 프로퍼티 기본값 지정
    2. $ java -Duser.timezone=Asia/Seoul
  2. 인코딩 관련 설정
    3. $ java -Dfile.encoding=UTF-8 -Dfile.client.encoding=UTF-8 -Dclient.encoding.override=UTF-8
  3. 리모트 디버깅 설정 introclientissues005
    4. $ java -Xdebug -Xnoagent -Xrunjdwp:transport=dt_socket,address=*:54433,server=y,suspend=n # address : 수신 허용 범위 # server : 서버로 작동할지 여부 # suspend : 디버거 부착까지 기다릴지 여부

jcmd Since Java 8

JDK 기본제공 분석 도구

  1. 자바 프로세스 목록
    2. $ jcmd
  2. 사용 가능 명령
    3. $ jcmd {pid} help
  3. 힙 히스토그램
    4. $ jcmd {pid} GC.class_histogram
  4. 힙 덤프
    5. $ jcmd {pid} GC.heap_dump ./out.hprof
  5. 시스템 프로퍼티
    6. $ jcmd {pid} VM.system_properties
  6. 쓰레드 덤프
    7. $ jcmd {pid} Thread.print

jhsdb Since Java 9

JDK 기본제공 디버거; Java Hotspot Debuggerjhsdb

  1. JVM에 디버거 부착 시 실행이 일시중지됨에 유의
  2. 사용
    3. # clhsdb 커맨드라인 디버거 $ jhsdb clhsdb [--pid pid | --exe executable --core coredump] # hsdb GUI 디버거 $ jhsdb hsdb [--pid pid | --exe executable --core coredump] # debugd 리모트 디버그 서버 시작 $ jhsdb debugd [options] pid [server-id]|[option] executable core [server-id]
