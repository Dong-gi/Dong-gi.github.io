1. Object는 모든 참조 자료형의 부모로, 가장 기본적인 클래스다.
  2. protected Object clone() : 객체의 복사본을 생성하여 반환한다.(얕은 복사)
    Object.clone()의 접근제한자는 protected이다. 이는 얕은 복사본의 이용으로 원본 객체까지 변경되는 것을 막기 위함이다.
    clone()을 재정의할 때, Cloneable 마커 인터페이스를 구체화하는데, 마커 인터페이스는 추상 메소드 없이, 클래스가 단순히 해당 기능을 가짐을 나타낸다.
  3. boolean equals(Object obj) : 현재 객체가 obj와 같은지 검사한다.
  4. void finalize() : 객체 폐기. 보통 GC가 알아서 처리해주지만, non-Java resource를 해제할 때 사용한다. GC가 언제, 정말로 호출될지 알 수 없으므로, 필요하다면 finalize()와 별개의 메서드를 정의하는 것이 낫다.
  5. Class<?> getClass() : 객체의 Class를 반환
  6. int hashCode() : 객체의 hash code 반환 // HashSet, HashMap 등에서 이용
  7. void notify() : 해당 객체에 대해 대기하는 스레드를 깨운다.
  8. void notifyAll() : 해당 객체에 대해 대기하는 모든 스레드를 깨운다.
  9. String toString() : 객체의 문자열 표현 반환
  10. void wait() : 해당 객체에 대해 스레드를 대기.
  11. void wait(long timeout) : 깨워질 때까지 최대 timeout ms동안 대기.
  12. void wait(long timeout, int nanos) : timeout ms + nanos ns
