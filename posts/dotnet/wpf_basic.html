실습 1

  1. XAML(eXtensible Application Markup Language)로 컨트롤을 작성하는 것과 코드에서 생성하는 것은 결과적으로 동일하다.
  2. Blend에서 디자인을 따로 할 수 있다. 분업-협업 가능.
  3. 참조 → 참조 추가에서 .NET 어셈블리 or 제작한 프로젝트 참조 추가
  4. NuGet 패키지 관리에서 다른 사용자들이 배포하는 프레임워크를 설치할 수 있다.
  5. 네임스페이스 접두어 xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
  6. Window : Initialized → Activated → Loaded → ContentRendered → 실행 → Closing → Unloaded → Deactivated → Closed

    7. Activated, Deactivated는 키보드/마우스 포커스를 얻거나 잃는 이벤트를 의미한다.

  7. 다루는 것 : Grid.Background, Button.Content, Application.Current.Properties, MessageBox.Show(), MessageBoxResult, Button_Click, Window_Loaded, Window_Closing

레이아웃

  1. 디자이너도 아니고... 종류만 알아보고 넘어간다.MS WPF Docs - Layout »
  2. Canvas : 캔버스 내부의 상대 좌표를 이용해 자식 요소들을 배치한다.
  3. DockPanel : Top, Left, Bottom, Right에 자식 요소들을 배치.
  4. Grid : 영역을 행과 열로 나누어 요소들을 배치한다. 나눠진 그리드 하나를 여러 요소가 공유할 수도, 한 요소가 여러 그리드를 차지할 수도 있다.
  5. StackPanel : 수직/수평적으로 자식 요소들을 나열.
  6. VirtualizingPanel : Panel에 대한 추상 클래스.
  7. WrapPanel : 자식 요소들을 좌에서 우로 차례대로 배치하다, 패널의 경계를 넘어서면 다음 줄(바로 아래)로 넘겨 배치한다.

  1. Event Tunneling : 부모 컨트롤로부터 자식 컨트롤로 Preview 접두어가 붙은 이벤트가 전달됨.

특정 조건에 따라 스타일을 적용시키는 방법.

바인딩

  • MVVM : Model - View - ViewModel
  • View의 컨트롤들을 ViewModel과 바인딩 : Control의 값 변경 + 이벤트를 ViewModel에 통지, Model의 값 변경을 View에 통지.
  • ViewModel은 INotifyPropertyChanged을 구현. 프로퍼티 setter에서 변경되었음을 통지.
    • public event PropertyChangedEventHandler PropertyChanged; PropertyChanged(this, new PropertyChangedEventArgs("DateTime"));
  • 바인딩이랑 바인딩 소스로부터 바인딩 타깃으로 프로퍼티 값을 전달하는 것
  • BindingMode : OneWay, TwoWay, OneTime, OneWayToSource
    • <ListBox xmlns:g="clr-namespace:System.Globalization;assembly=mscorlib" ItemsSource="{Binding Source={x:Static g:DateTimeFormatInfo.CurrentInfo}, Path=DayNames, Mode=OneTime}"/>
    • TwoWay : 바인딩 소스는 타깃이 입력 포커스를 잃을 때까지 값이 갱신되지 않는다. 설정을 변경하려면 Binding의 UpdateSourceTrigger를 변경하면 된다.

      • UpdateSourceTrigger : LostFocus(TextBox의 Text를 위한 기본값), PropertyChanged(대부분의 피로퍼티에 기본), Explicit

    • Explicit : 해당 엘리먼트로부터 BindingExpression 인스턴스를 얻고 UpdateSource()를 호출
      • var bindExp = textBox.GetBindingExpression(TextBox.TextProperty); bindExp.UpdateSource();
  • DataContext는 자식 요소에까지 적용된다
  • 데이터바인딩으로 사용자 정의 클래스 사용
    1. 코드에서 DataContext 설정 : DataContext 프로퍼티에 할당
    2. 클래스 이름을 ElementName으로 사용 : FrameworkElement를 상속
    3. Resources에 객체 생성 후 참조
  • 코드에서 바인딩 설정
  • IValueConverter : 데이터가 바인딩 소스에서 타깃으로 전달될 때 변환 수행
    • public class MyConverter : IValueConverter { public object Convert(object value, Type typeTarget, object param, CultureInfo culture) { // value : 변환할 객체 // typeTarget ; 변환 대상 타입; 변환할 수 없으면 null 반환 // param : Binding의 ConvertParameter 프로퍼티가 가리키는 객체 } public object ConvertBack(object value, Type typeTarget, object param, CultureInfo culture) { } public string MyProperty { get; set; } }
    • 코드에서 컨버터 연결 : binding.Convert = new MyConverter();
    • XAML에서 컨버터 연결 : Resources 절에 객체 생성 후 요소에서 참조
    <Window.Resources> <!-- src는 프로젝트 네임스페이스와 연결 --> <src:MyConverter x:Key="conv" MyProperty="Hello"/> </Window.Resources> <StackPanel> <ScrollBar Name="scroll" Maximum="100"/> <Label Content="{Binding ElementName=scroll, Path=Value, Converter={StaticResource conv}, ConverterParameter=2}"/> </StackPanel>
  • IMultiValueConverter

    • 여러 소스로부터 얻어진 객체들을 하나의 타깃 객체로 변환. IMultiValueConverter 인터페이스 구현
