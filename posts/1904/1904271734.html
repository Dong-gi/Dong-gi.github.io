1. 데이터베이스 ● 데이터베이스에 저장되는 데이터의 특징- Integrated Data- Stored Data- Operational Data- Shared Data ● 데이터베이스 특징- Real Time Accessibility- Continuous Evolution- Concurrent Sharing- Content Reference- Independence : 논리적 독립 + 물리적 독립논리적 독립 : 데이터의 논리적 구조가 변경되더라도 응용프로그램은 불변물리적 독립 : 저장 장치가 변경되더라도 응용프로그램은 불변 ● Data Language- DDL(Data Definition Language)- DML(Data Manipulation Language)- DCL(Data Control Language) ● 데이터베이스 사용자- DBA(DataBase Administrator)- Data Administrator- Data Architect ● DBMS : DataBase Management System기능 : 정의 기능, 조작 기능, 제어 기능 ● Schema개념 : 데이터베이스의 구조와 제약 조건에 관한 명세. 데이터 사전에 저장된다.- 외부 스키마 : 사용자가 필요로 하는 데이터베이스의 논리적 구조- 개념 스키마 : 개체 간 관계와 제약 조건을 나타내고 데이터베이스의 접근 권한, 보안 정책 및 무결성 규정에 관한 명세- 내부 스키마 : 데이터베이스의 물리적 구조 ● 데이터베이스 설계요구 조건 분석 – 개념적 설계 – 논리적 설계 – 물리적 설계 – 구현 ● 키- 수퍼키 : 속성들의 집합으로 이루어진 키.- 후보키 : 수퍼키 + 유일성과 최소성 만족.- 복합키 : 여러 속성으로 이루어진 키- 기본키 : 후보키 중 선정된 키. 중복될 수 없다. NULL일 수 없다.- 대체키 : 기본키 외의 후보키- 외래키 : 다른 릴레이션의 기본키를 참조하는 속성의 집합 ● 무결성 : 데이터와 실제 값이 일치하는 정확성- NULL 무결성 : 특정 속성 값이 NULL이 될 수 없는 규정- 고유 무결성 : 각 속성의 값이 유일해야 한다는 규정- 도메인 무결성 : 속성의 값이 도메인에 속해야 한다는 규정- 키 무결성 : 하나의 릴레이션에 적어도 하나의 키가 존재해야 한다는 규정- 관계 무결성 : 릴레이션에 어느 한 튜플의 삽입 가능 여부 또는 한 릴레이션과 다른 릴레이션의 튜플들 사이의 관계에 대한 적절성 여부를 지정한 규정- 참조 무결성 : 외래키 값은 NULL이거나 참조 릴레이션의 기본키 값과 동일해야 한다는 규정- 개체 무결성 : 기본키를 구성하는 어떤 속성도 NULL일 수 없다는 규정 ● 관계대수 : 원하는 정보를 어떻게 유도하는가를 기술하는 절차적 언어- SELECT : 특정 조건을 만족하는 튜플의 부분집합을 구한다. - PROJECT : 릴레이션의 특정 속성만을 추출. - JOIN : 공통 속성을 중심으로 2개의 릴레이션을 하나로. ⋈ CARTESIAN PRODUCT연산을 수행한 다음 SELECT 연산을 수행한 것과 같다. ● 관계해석수학의 술어해석(Predicate Calculus)에 기반을 두고 관계 데이터베이스를 위해 제안. 관계대수와 관계해석은 기능과 능력 면에서 동등. 원하는 정보가 무엇인지만 정의하는 비절차적 특성. ● VIEW : 기본 테이블에서 유도되는, 물리적으로 존재하지는 않는 가상 테이블.SELECT문의 결과로서 뷰를 생성 ● 프로시저 : 특정한 작업을 수행하는 SQL문을 논리적으로 그룹화한 것. ● Stored Procedure : SQL문들을 하나로 모아 SQL 서버에 미리 컴파일해서 저장해 놓은 것. ● 이상 : 테이블 일부 속성들의 종속으로 인해 데이터의 중복이 발생하고, 이로 인해 테이블 조작 시 문제가 발생하는 현상.- 삽입 이상(의도치 않게 삽입 불가)- 삭제 이상(의도치 않게 다른 값들도 삭제)- 갱신 이상(일부만 갱신되어 정보 불일치). ● 함수적 종속 : 어떤 테이블 R의 속성 부분 집합 X, Y에 대하여, 시간에 관계없이 항상 X의 값 각각에 대해 Y의 값 하나만 연관될 때. X -> Y로 표기. 함수적 종속 다이어그램 : 기본키 전체로 결정되는 경우 : 완전 함수 종속 기본키의 부분만으로 결정되는 경우 : 부분적 함수 종속 ● 정규화 : 테이블을 무손실 분해하여 가능한 한 중복을 제거하여 이상 발생을 줄이는 것. - 제1정규형 1NF : 모든 속성의 도메인이 원자 값만으로 되어 있다. - 제2정규형 2NF : 제1정규형 + 기본키가 아닌 모든 속성이 기본키에만 완전 함수 종속. - 제3정규형 3NF : 2정규형 + 기본키가 아닌 모든 속성이 기본키에 대해 이행적 함수 종속을 만족하지 않음. - BCNF 결정자 : 어떤 함수 종속 관계 X -> Y에서의 X. Y는 종속자 BCNF는 모든 결정자가 후보키인 정규형. - 제4정규형 4NF : 다치 종속 A->>B가 존재할 경우, 모든 속성이 A에 대해 함수 종속을 만족. - 제5정규형 5NF : 모든 조인 종속이 후보키를 통해서만 성립. - 역정규화 : 지나친 정규화는 응답 속도를 저하시키므로, 어느 정도는 테이블을 합치는 것. * 정규화(normalization)로 분해된 릴레이션을 대상으로 질의할 때, 같은 정보를 얻기 위해 보다 많은 릴레이션에 접근해야 하므로 JOIN의 필요성이 증가 ● 인덱스 데이터 레코드(튜플)에 빠르게 접근하기 위해 <키, 포인터> 쌍으로 구성 색인이 없으면 검색마다 모든 데이터 페이지를 다 뒤지는 TABLE SCAN 발생 ● 트랜잭션 데이터베이스에서 하나의 논리적 기능을 수행하기 위한 일련의 연산 집합으로, 작업의 단위. 정상적으로 종료될 경우 COMMIT, 아니면 ROLLBACK ● 트랜잭션의 특성 - 원자성(Atomicity) : 트랜잭선은 모두 반영되든지 아예 반영되지 않아야 한다. - 일관성(Consistency) : 트랜잭션 수행이 완료되면 일관적인 상태를 유지 - 독립성(Isolation) : 한 트랜잭션이 데이터를 갱신하는 동안 다른 트랜잭션이 접근하지 못함. - 영속성(Durability) : 트랜잭션의 결과는 영구적으로 보존. ● 트랜잭션의 상태 활동(Active), 실패(Failed), 철회(Aborted), 부분 완료(Partially Committed), 완료 ● 회복 : 트랜잭션 수행 중 장애가 발생하여 이전의 정상 상태로 복구하는 작업. - 연기 갱신 기법 : 트랜잭션이 완료된 후에야 실질적으로 갱신 - 즉각 갱신 기법 : 데이터의 변경을 바로바로 기록하고, 로그를 관리 - 그림자 페이지 기법 : 그림자 페이지에 바로바로 기록하고 나중에 실제로 반영 - 검사점 기법 : 검사점을 로그에 기록. 장애시 마지막 검사점부터 회복 수행 - 미디어 회복 기법 : 데이터베이스를 안전한 저장 장치에 덤프해 놓는다. ● 병행 제어 동시에 여러 개의 트랜잭션을 병행 수행할 때 실행되는 트랜잭션들이 일관성을 파괴하지 않도록 트랜잭션 간 상호 작용을 제어. - 필요성 : 갱신 분실, 모순성, 연쇄 복귀 - 잠금(Lock) : 한 트랜잭션에 사용되는 데이터를 다른 트랜잭션이 접근하지 못하게 함. * 2-단계 잠금 규약 : 확장 단계(잠금만 가능), 축소 단계(잠금 해제만 가능) * 타임스탬프 순서 기법 : 트랜잭션 발생 시각 순서대로 실행. ● 데이터 마이닝 - 연관(Association) - 연속(Sequence) : 시계열적으로 분석하여 향후 발생 가능성 예측 - 분류(Classfication) - 클러스터링(Clustering) - 특성화(Characterization) - 패턴 분석 - 경향 분석 ● Hadoop 오픈 소스를 기반으로 한 분산 컴퓨팅 플랫폼. 일반 PC급 컴퓨터들로 가상화된 대형 스토리지를 형성하고 그 안에 보관된 빅데이터를 병렬로 처리할 수 있도록 개발된 자바 소프트웨어 프레임워크. ● WebSocket 웹 브라우저와 웹 서버 사이에 전이중 통신을 제공하는 프로토콜. ● 식별 관계 : A의 기본키가 B의 외래키인 동시에 B의 기본키 ● 분포도가 10% 이하인 경우엔 인덱스 스캔, 10~15% 이상인 경우 Full Table Scan ● 정규화로 생성된 릴레이션 R1, R2가 NATURAL JOIN으로 손실 없이 R로 복귀되는 경우, R은 R1, R2로 무손실 분해되었다고 한다. ● Durability is the ability of the DBMS to recover the committed transactions ● Social Engineering : 인간 상호 작용의 깊은 신뢰를 바탕으로 사람들을 속여 정상 보안 절차를 깨트리기 위한 비기술적 시스템 침입 수단
