WWW-Authenticate 헤더 : 보안 영역 설명 HTTP/1.0 401 Authorization Required WWW-Authenticate: Basic realm="문서 집합 정보" 3. 클라이언트 인증 GET 요청. Authorization 헤더 : 인증 알고리즘, 사용자 이름, 비밀번호 GET /family/jeff.jpg HTTP/1.0 Authorization: Basic "ID:PW base-64 인코딩" 4. 200 OK * 기본 인증을 안전하게 이용하는 유일한 방법은 SSL과 결합해서 사용하는 것. 다이제스트 인증 * 비밀번호를 네트워크를 통해 보내지 않는다. → 비밀번호를 비가역적으로 뒤섞은 지문(fingerprint), 요약(digest)을 보낸다. * 재전송 공격을 방지하기 위해 서버는 클라이언트에 nonce를 넘기고, nonce는 다이제스트 생성에 관여한다. nonce는 밀리초 혹은 인증할 때마다 변경되므로, 재전송 공격에 안전하다. GET /cgi-bin/checkout?cart=17854 HTTP/1.1 Authorization: Digest username="bri" realm="Shoppint Cart" nonce="66C4EF58..." uri="/cgi-bin/checkout?cart=17854" qop="auth" nc=0000001, cnonce="CFA920710..." 계산 단방향 해시 함수 H(d), 요약 함수 KD(s, d) // s : sccret, d : data RFC 2617에서 제안된 알고리즘은 MD5(기본), MD5-sess H(d) = MD5(d), KD(s, d) = MD5(s:d) nonce = BASE64(timestamp H(timestamp":"ETag":"private_key)) cnonce : 클라이언트가 서버를 인증할 목적으로 생성. HTTPS * 디지털 서명 : 메시지가 위/변조되지 않았음을 입증하는 체크섬 노드 A는 가변 길이 메시지를 정제하여 고정된 길이의 요약(digest)으로 만든다. 노드 A는 요약에 사용자의 개인 키를 매개변수로 하는 서명 함수를 적용한다. 노드 A는 메시지와 그에 대한 서명 둘 다를 전송한다. * 디지털 인증서 : 신뢰할 수 있는 조직에 의해 서명되고 검증된 신원 확인 정보 암호(chipher) 보통 연결을 수립할 때는 느린 공개키 암호를 사용하고, 이후 메시지 전송에는 빠른 대칭키 암호를 사용. 부록 - MIME 타입
