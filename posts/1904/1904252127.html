1. abs(x) : 절댓값을 반환한다. 복소수의 경우 절댓값으로 magnitude(= sqrt(a^2+b^2))가 반환된다.

2. all(iterable) : iterable 모든 요소가 True면 True 반환. iterable이 빈 경우도 True 반환.

3. any(iterable) : iterable 하나의 요소라도 True면 True 반환. iterable이 빈 경우 False 반환.

4. ascii(object) : repr()처럼 객체의 문자열 표현을 반환한다. ASCII 문자가 아닌 경우 이스케이프된다.
>>> ascii("안녕 세상!")
"'\\uc548\\ub155 \\uc138\\uc0c1!'"

5. bin(x) : 정수를 "0b"로 시작하는 2진 문자열로 변환한다. 다른 객체의 경우 정수를 반환하는 __index__()를 정의하면 그것을 이용한다.
>>> bin(5), format(5, '#b'), format(5, 'b'), f'{5:#b}', f'{5:b}'
('0b101', '0b101', '101', '0b101', '101')

6. class bool([x]) : int의 서브클래스로, 더 상속할 수 없다. 클래스의 인스턴스는 True, False 둘 뿐이다.

7. class bytearray([source[, encoding[, errors]]])
» 변경할 수 있는 바이트 배열. 생성자에 인자가 없으면 크기가 0인 바이트 배열을 반환한다.
» source가 문자열인 경우 : encoding이 반드시 설정되어야 하며, str.encode()를 통해 변환된다.
» source가 정수인 경우 : 해당 바이트 크기 만큼의 배열이 0 초기화된다.
» source가 buffer 인터페이스와 호환되는 객체일 경우 : 객체의 읽기 전용 버퍼로 배열을 초기화한다.
» source가 iterable인 경우 : 반복자 모든 요소가 [0, 255] 사이의 정수여야 한다.

8. class bytes([source[, encoding[, errors]]])
» bytearray의 변경 불가 버전.
» Bytes 리터럴 표기를 통해 객체를 생성할 수도 있다. 예. b"Hello World" 자세히

9. chr(i) : 유니코드 i에 해당하는 문자를 문자열로 반환. 가능한 i 값의 범위는 [0, 0x10FFFF]

10. @classmethod
» 메서드를 클래스 메서드로 꾸민다. 클래스 메서드는 명시적으로 클래스를 첫번째 인자로 받으며, 호출한 클래스 객체가 전달된다.
» 파이썬은 class method와 static method를 구분한다.
class C:

    def f(cls, arg1, arg2, ...): ...

11. compile(source, filename, mode, flags=0, dont_inherit=False, optimize=-1)
» source를 code나 AST 객체로 컴파일한다. Code 객체는 exec()나 eval()을 통해 실행할 수 있다.
» source로는 문자열, 바이트 문자열, AST 객체가 가능하다.
» filename : 코드를 읽어온 파일. 파일로부터 읽은 것이 아니라면 '<string>'를 보통 대신 넘긴다.
» mode : 'exec'(여러 문장들 실행), 'eval'(하나의 표현식 계산), 'single'(한 줄 실행)
» optimize : 0(최적화 없음, __debug__ is true), 1(asserts 제거, __debug__ is false), 2(docstring도 제거)
>>> exec(compile("a = 1\nb = 2\nprint(a+b)", "<string>", "exec"))
3

3. dir([object])
» 인자가 없으면 현재 local scope의 이름들을 나열한다. 인자가 있으면 객체의 속성들을 알파벳순으로 나열한다.
» 객체에 __dir__()이 정의되었다면 그것을 이용한다. __getattr__() 또는 __getattribute__()를 정의하여 dir()의 출력을 정의할 수 있다.
» __dir__()이 정의되지 않았다면 __dict__ 속성을 이용한다. dir()의 결과는 완성되지 않을 수 있으며, __getattr__()이 정의되었다면 정확하지 않을 수도 있다.

4. divmod(a, b)
» 정수 연산 : (a // b, a % b)와 같다.
» 실수 연산 : (q, a % b) where q는 math.floor(a / b)이거나 이보다 1 작은 값, a % b는 b의 부호를 따르며 0 <= abs(a % b) < abs(b)를 만족한다. 어떤 경우든 q * b + a % b는 a에 가깝다.

7. class int(x=0)
class int(x, base=10)
» x가 숫자면 x.__int__()을 반환한다. x가 __int__()대신 __trunc__()을 정의했다면 이를 호출한다. 부동소수의 경우 소수를 0으로 잘라낸다.
» base가 0일 경우 정수 리터럴 문자열을 그대로 반환한다. 결과적으로 2, 8, 10, 16진수를 해석하고, '0'으로 시작하는 경우 여러가지로 해석되므로 ValueError가 발생한다.
» Since 3.4 base가 int가 아닐 때, base.__int__를 이용하던 것에서 base.__index__를 이용하도록 변경
» Since 3.6 '_'로 그루핑된 정수 리터럴이 허용

4. open(file, mode='r', buffering=-1, encoding=None, errors=None, newline=None, closefd=True, opener=None)
» 파일을 열고 file 객체를 반환한다.
» file : pathname 또는 file descriptor. closefd가 True인 경우, 반환된 file 객체가 닫히면 file descriptor도 닫힌다. filename이 주어진 경우 closefd는 True여야만 한다.
» mode : r(reading, default), w(writing, 파일 처음부터), x(파일이 없으면 생성), a(appeding), b(binary), t(text, default)
» encoding : 텍스트 모드에서만 사용. 주어지지 않은 경우 locale.getpreferredencoding()를 이용. 가능한 인코딩 목록은 codecs 모듈에 정의됨.
파이썬은 텍스트 IO와 이진 IO를 구별하며, 전자의 경우 str을, 후자의 경우 bytes를 반환한다.
» buffering : 0(이진모드만, 버퍼링 없음), 1(텍스트모드만, 라인 단위), > 1(바이트 사이즈 버퍼링)
» errors : 'surrogateescape'(보충 평면 문자 처리), 'xmlcharrefreplace'(&#nnn으로 대체, 쓰기모드만), Since 3.5 'namereplace'(\N{...}로 대체, 쓰기모드만)
» Since 3.3 opener : custom opener
» 반환값 : file 객체. 텍스트모드의 경우 io.TextIOBase의 후손, 이진모드의 경우 io.BufferedIOBase의 후손이다.
» Since 3.4 반환된 file 객체는 상속 불가능하다.
