부팅 과정

  1. BIOS 실행
  2. POST 수행 : 장치 검사, 초기화
  3. 부트로더 실행 : 커널 로드. BIOS 종료

    4. GRUB : GNU 프로젝트의 부트로더. 대부분 운영 체제의 커널을 불러올 수 있다

  4. init 프로세스 실행 : /etc/inittab

    5. systemd가 사용되는 경우, inittab은 사용되지 않는다

  5. init 프로세스 중에 /etc/rc.d/rc.local 스크립트를 실행한다

    6. 해당 파일은 호환을 위해 아직 남아있지만, 사용자 정의 systemd 서비스나 udev 규칙을 만드는 것이 권장된다

shutdown : 시스템 종료 관련

  1. shutdown [OPTIONS...] time [warning-message]
  2. -H : 시스템 종료
  3. -P : 시스템 종료 + 전원 차단(하드웨어 지원 필요)
  4. -r : 재부팅
  5. -c : 대기중인 shutdown 취소
  6. time : now | +number | yymmddhhmm

    7. +number 단위(기본: 분)를 지정하려면 접미사 "s", "sec", "m", "min", "h", "hour" 이용

  7. warning-message : 로그인한 유저들에게 보낼 브로드캐스팅 메시지

init : 일시적인 runlevel 변경

  1. init runlevel
  2. runlevel : 0(종료), 6(재부팅), 1(복구), 2(multiuser without NFS), 3(multiuser; TUI), 5(X11; GUI)
  3. 시스템 기본 runlevel 확인
    4. systemctl get-default

사용자 관리

  1. useradd [options] USER_NAME

    2. -d(HOME_DIR), -g(group id or name), -p(password), -D(기본값 출력/설정), 등...

  2. passwd [options] USER_NAME

    3. -d(삭제), -l(잠금), -u(잠금 해제), -e(강제 만료)

  3. usermod [options] USER_NAME

    4. -d(HOME_DIR), -m(HOME_DIR 변경 시 파일 이동), -L(잠금), -p(password), 등...

  4. userdel [options] USER_NAME
  5. groupadd [options] GROUP_NAME

    6. -g(수동 GID), -p(password)

  6. groupdel [options] GROUP_NAME
  7. su [options] [-] [USER_NAME [arg]...]

    8. 다른 사용자의 권한으로 쉘을 실행한다. USER_NAME이 없으면 root로 간주한다. 단순히 '-'만 있으면 '-l'과 동일하다.

  8. 사용자 목록 : /etc/passwd
    9. ID:Password(x):UID:GID:Comment:HOME_DIR:SHELL root:x:0:0:root:/root:/bin/bash dgkim:x:1000:1000:dgkim:/home/dgkim:/bin/bash
  9. 비밀번호 목록 : /etc/shadow

    10. !!는 비밀번호가 설정되지 않았음을 의미

  10. 사용자의 홈디렉터리는 /etc/skel의 파일을 /home/${USER_NAME}에 복사하여 만들어진다

파일 시스템

  1. 디렉터리 구조

    2. /bin(명령어), /dev(장치 파일), /etc(환경설정), /lib(라이브러리, 커널 모듈), /media(이동식 디스크), /usr(응용프로그램), /var(자주 변경)

  2. I-node

    3. 파일의 이름을 제외한 모든 정보. ID는 각 파일에 고유하게 부여된다

  3. pwd

    4. Print working directory; 현재 디렉터리 절대경로

  4. cd

    5. cd(자신의 HOME_DIR로), cd DIR(디렉터리 이동), cd ~USER_NAME(유저 HOME_DIR로)

  5. ls [options] [paths]

    6. -a(.로 시작하는 경로 포함), -c(with -lt: 시간순, with -l: 이름순), -X(확장자순), -S(크기순), -r(역정렬), -l(자세히), 등...
    d rwx r-x r-x : 디렉터리, 소유자 권한, 그룹 권한, 기타 사용자 권한 755(read, write, execute)
    - rw- r-- r-- : 파일, (상동) 644
    l r-- r-- r-- : 링크, (상동) 444

  6. mkdir [options] [directories]

    7. -m(권한, 기본 755), -p(경로상의 모든 디렉터리 생성)

  7. rmdir [options] [directories]

    8. 빈 디렉터리만 삭제 가능. -p(경로 상의 모든 디렉터리 삭제)

  8. rm [options] [paths]

    9. -f(파일이 없으면 무시), -i(각 파일 삭제 확인), -r(재귀 실행), 등...
    rm -rf ./my_dir # my_dir 디렉터리 및 내부 파일 모두 삭제

  9. touch [options] [files]

    10. 각 파일에 대해 시간 정보를 현재 시각으로 변경한다. 파일이 없으면 생성한다(크기 0).

  10. chmod [options] 8진수 [paths]

    11. -R : 재귀 실행 // + chown(파일 소유자 및 그룹 변경), chgrp(사용자 그룹 변경)

  11. cp [options] source dest_path
    cp [options] [sources] [target_directory]

    12. -a(파일 속성 유지), -b(목적지에 이미 있으면 백업하고 복사), -r(재귀 실행), -s(심볼릭 링크 생성), 등...

  12. mv : cp + 원본 삭제
  13. cat [options] [files]

    14. -n(줄 번호), -E(줄 끝에 $), -s(반복적인 빈 라인 생략), -T(탭을 ^T로 표기)

  14. find

    15. 파일명 표현에 와일드카드 *, ?가 허용된다. 표현식 앞에 !이나 -not을 붙일 수 있다. 표현식들 사이를 -and, -or로 엮을 수 있다.

  15. grep [options] pattern [files]

    16. -E(ERE), -G(BRE), -P(Perl Regexp), -e(정규표현식), -i(ignore case), -w(패턴에 단어 전체가 매칭되는 경우만), -x(패턴에 줄 전체가 매칭되는 경우만), -m(최대 출력 개수), -n(줄 번호 같이 출력), -o(매칭된 부분만 출력), -l(매칭된 파일들의 이름만 출력), -L(매칭되지 않는 파일들의 이름만 출력), -A(매칭 후 n개 라인도 출력), -B(매칭 전 n개 라인도 출력) 등...

  16. |

    17. 프로세스들의 출력과 입력을 잇는 파이프라인을 구성한다.

    history | grep psql
  17. 리다이렉션

    18. > : 파일이 없다면 생성하고, 있다면 덮어쓴다
    >> : 파일이 없다면 생성하고, 있다면 append
    < : 파일로부터 표준 입력을 받는다

  18. tar [options] file : 압축해제
    tar [options] file paths : 압축

    19. Main Operation : -A(append), -r(append), -c(create), -d(파일 시스템과의 차분 출력), -t(목록 출력), -u(update), -x(압축해제)
    포맷 : --bzip2, --xz, --lzip, --lzma, --gzip
    -h(링크 자체 대신, 링크가 가리키는 파일을 압축한다)
