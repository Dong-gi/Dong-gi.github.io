기본 명령어

  1. 실행 : bin\startup.bat
  2. 종료 : bin\shotdown.bat
  3. webapps

    4. webapps 디렉터리의 서브디렉터리들을 웹 애플리케이션 디렉터리라 하며, 각각이 하나의 프로젝트 산출물이라 보면 된다. 기본 웹 애플리케이션 디렉터리는 webapps/ROOT

  4. 기본 설치 예제

    5. http://localhost:8080/examples/

  5. 관리자 페이지

    6. http://localhost:8080/manager/html

  6. 계정 추가 : conf/tomcat-users.xml 수정
    7. <user username="admin" password="admin" roles="manager-gui,admin-gui"/>
  7. JSP 버전 설정

    8. conf/web.xml에서 compilerSourceVM, compilerTargetVM 값 변경

  8. 애플리케이션 배포 및 취소

    9. webapps 디렉터리에 war 파일을 옮기면 자동으로 압축을 푼다. 반대로 삭제하면 자동으로 정리한다. 또는 관리자 페이지의 deploy/undeploy를 이용.

JVM 타임존 설정

-Duser.timezone=Asia/Seoul

톰캣 JVM 실행 인코딩/문자셋 설정

  1. catalina.bat
    2. set "JAVA_OPTS=%JAVA_OPTS% -Dfile.encoding=UTF-8 -Dfile.client.encoding=UTF-8 -Dclient.encoding.override=UTF-8"
  2. catalina.sh
    3. JAVA_OPTS="$JAVA_OPTS -Dfile.encoding=UTF-8 -Dfile.client.encoding=UTF-8 -Dclient.encoding.override=UTF-8"

톰캣 리모트 디버깅 설정

  1. catalina.bat
    2. set "JAVA_OPTS=-Xdebug -Xnoagent -Xrunjdwp:transport=dt_socket,address=*:54433,server=y,suspend=n"
  2. catalina.sh
    3. JAVA_OPTS="-Xdebug -Xnoagent -Xrunjdwp:transport=dt_socket,address=*:54433,server=y,suspend=n"
  3. 허용 서브넷 명시 : address=*:55555, address=0.0.0.0:55555
  4. Eclipse - Debug Configuration - Remote Java Application

서버 URI 인코딩 설정 : conf/server.xml

<Connector port="8080" protocol="HTTP/1.1" connectionTimeout="20000" redirectPort="8443" URIEncoding="UTF-8" /> <Connector port="8009" protocol="AJP/1.3" redirectPort="8443" URIEncoding="UTF-8" />

톰캣 utf8 필터 설정 : {APP_NAME}/web.xml

<filter> <filter-name>setCharacterEncodingFilter</filter-name> <filter-class>org.apache.catalina.filters.SetCharacterEncodingFilter</filter-class> <init-param> <param-name>encoding</param-name> <param-value>UTF-8</param-value> </init-param> <async-supported>true</async-supported> </filter> <filter-mapping> <filter-name>setCharacterEncodingFilter</filter-name> <url-pattern>/*</url-pattern> </filter-mapping>
