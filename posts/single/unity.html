Component

  1. Animation

    2. Window - Animation - Create

  2. Animator
    • Window - Animator : 애니메이션 상태도를 편집할 수 있다.
    • 기본 노드 : Entry(애니메이션 시작), Any State(상태와 무관), Exit(애니메이션 종료)
    • Animator.SetTrigger() : 트리거 설정하여 애니메이션 상태를 전이시킨다
    • Animator.speed : 재생 속도
  3. Empty GameComponent

    4. Transform 컴포넌트만 가진 게임 객체. 형식상의 부모 컴포넌트로 이용하거나, 다른 작업의 기준점으로 이용할 수 있다.

  4. Outlet

    5. public GameObject 필드를 선언하면 Inspector에 해당 필드가 표시되고, 다른 컴포넌트를 드래그-앤-드롭하여 참조를 설정할 수 있다.

  5. GetComponent<T>() : 구성 컴포넌트 획득.
  6. Rigidbody.AddForce() : 외력 작용. F = ma이므로, 강체의 질량에 따라 추가되는 가속도가 달라진다
  7. Rigidbody.velocity : 속도 벡터

  12. Transform.Translate() : 위치 이동

Event Handler

  1. OnCollisionEnter(Collision) : 3D 충돌 발생 시 호출

    2. 충돌체 Collider는 컴포넌트트의 래퍼 컴포넌트로, 유니티에서는 Box Collider, Sphere Collider, Capsule Collider 세 종류의 기본 충돌체를 제공한다. Collider는 IsTrigger 속성을 가지며, True인 경우 충돌의 감지(OnTriggerEnter)는 되지만 물리적인 효과(OnCollisionEnter)는 발생하지 않는다.

  2. Start() : 첫 프레임 Update 전에 호출
  3. Update() : 매 프레임마다 호출

Debug

  1. Debug.Break() : 일시정지
  2. Debug.Log() : 로깅

Image

  1. Image Type을 Filled로 설정하면 이미지를 동적으로 일부분만 그릴 수 있다.

Input

  1. Input.GetButton() : 유니티 정의 입력이 들어왔는지 여부 반환. Edit - Project Settings - Input
  2. Input.GetKey(KeyCode.UpArrow) : 키 눌렸는지 여부 반환
  3. Input.GetAxis("Horizontal") : ↔ 방향 입력이 얼마큼 입력됐는지 반환. [-1.0, 1.0]
  4. Input.GetAxis("Vertical") : ↕ 방향 입력이 얼마큼 입력됐는지 반환. [-1.0, 1.0]

Object

  2. Object.Instantiate() : 유니티 객체 생성
  3. Object.Destroy() : 유니티 객체 해제
