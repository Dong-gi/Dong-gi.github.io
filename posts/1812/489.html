1. 개요
    • 플랫폼 : 응용 프로그램들을 실행하는 데 사용되는 하드웨어와 소프트웨어의 집합
    • 소프트웨어 플랫폼 : 운영체제와 무관하게 한 언어로 작성한 프로그램을 실행할 수 있게 해주는 소프트웨어적 플랫폼
    • 프레임워크 : 제어 역전이 있는 대규모 라이브러리. 프로그램의 초기화부터 종료까지 직접 관리하며, 개발자는 일부분만을 작성.
    • 닷넷 (프레임워크) : CLR + BCL(Base Clas Library) + 기타 부가 구성 요소
    • C# : 닷넷을 가장 잘 표현하는 대표 언어
    • 공용 언어 런타임 CLR
      • CLI 사양을 따르는 대표적인 가상 머신으로, 윈도우 운영체제용으로 배포된다. 중간 언어를 JIT 컴파일러를 이용해 기계어로 변환하고, GC(Garbage Collector)를 운용한다.
    • 공용 언어 기반구조 CLI(Common Language Infrastructure)
      • 마이크로소프트에서 ECMA 표준으로 제출한 공개 규약. CTS 명세, 중간 언어 정의, 메타데이터와 이를 포함한 이진 파일의 구조까지 표준 사양으로 기술한다. 자바와 마찬가지로 CLI 사양을 준수한 구현체에서 동작하는 닷넷 파일은 다른 구현체에서도 실행된다.
      • 공통 중간 언어 CIL(Common Intermediate Language) == IL
        • IL은 자체 문법을 가지며 ILASM.EXE라는 전용 컴파일러도 있다.
      • 공용 타입 시스템 CTS(Common Type System)
        • 닷넷 호환 언어가 지켜야 할 타입의 표준. 전체를 구현하진 않아도 된다.
      • 공용 언어 사양 CLS(Common Language Specification)
        • 닷넷 호환 언어가 지켜야 하는 최소 사양
      • 메타데이터
        • CLR에서 동작하는 실행 파일(exe/dll)은 완전하게 자기 서술적인(self-descriptive) 메타데이터를 제공하며, 외부에서는 리플렉션을 통해 이를 사용할 수 있다.
      • 어셈블리
        • 닷넷에서는 exe/dll 실행파일을 어셈블리라고 부른다.
      • 모듈
        • 어셈블리는 1개 이상의 모듈로 구성되며, 하나의 모듈은 하나의 파일에 대응된다.
      • 매니페스트
        • 여러 모듈이 하나의 어셈블리를 구성하는 경우, 모듈 중 하나는 모듈의 목록을 관리하는 매니페스트 데이터를 갖는다. 매니페스트를 포함하는 경우 확장자는 exe/dll, 그 외의 모듈은 보통 netmodule이지만, 일반적으로 모든 어셈블리는 1개의 파일로 구성된다.
    • C# 컴파일러는 소스 코드를 IL(Intermediate Language) 중간 언어로 exe/dll파일 생성하고, exe파일에 프로그램이 시작하자마자 CLR(Common Language Runtime)을 로드하는 코드를 추가한다. CLR이 IL코드를 OS에 맞는 네이티브 코드로 변환해 실행한다.
  2. 개발 환경
    3. 윈도우에는 기본적으로 닷넷 프레임워크가 설치(설치 위치는 C:\Windows\Microsoft.NET\Framework$버전)된다. 그렇지 않다면 MS센터에서 다운로드.
    C# 컴파일러 : csc.exe
    닷넷 역컴파일러 : .NET Reflector(유료), JustDecompile(무료) 등
  3. 문법 요소
    4. 예약어, 키워드, 식별자, 리터럴, 변수(Value type, Reference type), 상수(const), 연산자, 문장부호
  4. 기본 자료형
    • 정수형
      • sbyte : 부호 있는 1바이트. System.Sbyte
      byte : 1바이트. System.Byte
      short : 2바이트. System.Int16
      ushort : 부호 없는 2바이트. System.UInt16
      int : 4바이트. System.Int32
      uint : 부호 없는 4바이트. System.UInt32
      long : 8바이트. System.Int64
      ulong : 부호 없는 8바이트. System.UInt64
    • 실수형
      • float : 4바이트. System.Single
      double : 8바이트. System.Double
      decimal : 16바이트. System.Decimal
    • 문자형
      • char : 2바이트. System.Char
      string : 동적 크기. System.String. Indexer를 구현했기 때문에 인덱스 참조 가능.
    • Boolean
      • bool : true or false. System.Boolean
  5. 배열
    6. * 배열 초기화시 기본값으로 설정된다.
    * 다차원 배열 : [,](배열의 차원이 고정), Jagged Array : [][](배열의 차원이 가변적)
    int[] arr1 = new int[3];
    int[] arr2 = new int[] { 1, 2, 3 };
    int[] arr3 = { 1, 2, 3 };
    int[,] arr4 = new int[2, 2];
    int[,] arr5 = new int[,] { { 1, 2 }, { 3, 4 } };
  6. 형변환
    7. * 암시적 형변환 : 보다 큰 정의의 자료형으로 자동 변환.
    * 명시적 변환 : 괄호 이용.
    * as 연산자를 이용한 명시적 변환은, null을 허용하는 참조 자료형만 가능하다.
    * Boxing : Value type 데이터를 Reference type으로 변환. 컴파일러가 묵시적으로 수행.
    * Unboxing : Reference type 데이터를 Value type으로 변환. 캐스팅을 통해 명시적으로 수행.
  7. 오버플로 검사
    8. checked { /* 문장 */ } : OverflowException 발생.
    unchecked { /* 문장 */ } : 기본적으로 오버플로 검사하지 않지만, 의도를 명시
  8. 표준 입출력
    9. Console.Read() : 한 글자
    Console.ReadLine() : 한 줄.
    Console.Write(), Console.WriteLine()
    * Formatted output : {N[,W][:formatCharacter]}
    N : 매개 변수 위치 지정. 0부터 시작
    W : 출력될 자릿수 폭. '-'를 붙이면 왼쪽 정렬.
    Format character : d(10진수), e(지수), f(고정 소수점), g(e, f중 간략한 형태), n(','로 구분한 10진수), p(%), x(16진수)
  9. 제어문
    10. ● short-circuit : true || ?, false && ? // 뒷부분 판정 생략 ● if ~ else if ~ else ● switch
    표현식으로 정수형, 문자, 문자열 가능
    case 다음이 빈 문장일 때만 break 생략 가능.
    default 생략 가능 ● for(초기식; 조건식; 증감식;)
    ● foreach(자료형 변수명 in Array|Collection)
    ● while(조건식), do ~ while(조건식)
    ● goto : 관례상 레이블은 대문자로만
