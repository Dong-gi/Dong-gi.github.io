태그

  • HTML 문서는 요소들의 트리로 표현된다
  • 태그는 요소의 타입을 나타내는 이름으로, 이름은 사용자가 임의로 지어도 된다
  • Non-replaced elements

    • <p>, <h1>과 같은 요소들은 여는 태그와 닫는 태그 사이에 텍스트나 다른 요소들이 포함될 수 있다

  • Replaced elements

    • <img>, <input>과 같은 요소들은 UI 위젯이나 이미지와 같은 객체로 대체된다. 대체된 객체들은 기본 스타일을 가지며, 브라우저에 따라 사용자가 부분적으로 스타일을 변경할 수 있다

  • Void elements

    • <br>, <img>, <link>와 같이 태그 하나로 열고 닫는(self-closing) 요소들로, 텍스트나 다른 요소를 자식으로 가질 수 없다. 태그 끝에 '/'를 적어도 된다

속성

  1. 요소의 속성은 여는 태그 내부에 이름="값" 쌍으로 표현한다. 태그와 마찬가지로 이름은 사용자가 임의로 지어도 된다
  2. 모든 요소에서 유효한 속성, 일부 요소에서만 유효한 속성들이 존재한다
    • html 요소의 lang 속성 : 문서의 주 언어
    • 기타 텍스트를 포함한 요소의 lang 속성 : 요소의 부 언어
    • title 속성 : 요소에 대한 툴팁(커서를 올려두면 표시)
    • disabled 속성 : 지정한 입력 요소를 수정할 수 없음
    • id 속성 : 요소의 고유한 id
    • class 속성 : 요소가 속한 그룹
  3. 단순히 해당 속성이 존재함을 표현하는 걸로 충분한 일부 속성들은 값을 가지지 않는다

HTML 문서 구조

  • <!DOCTYPE html>

    • 모든 HTML 문서의 서두엔 위 문구가 있어야 한다. 이는 HTML 요소가 아니라 DTD(Document Type Definition)의 일종이다. DOCTYPE은 브라우저가 문서를 어떤 모드로 렌더링할 지를 결정하는 데 이용된다. DOCTYPE html은 표준 모드 렌더링을 지시한다

  • <html>

    • <html> 요소는 HTML 문서의 루트 요소로, <head>, <body> 요소의 부모다

  • <head>

    • <head> 요소는 문서 또는 웹앱에 대한 모든 메타데이터를 기술한다

head

  • Charactr set

    • <head> 요소의 첫 번째 요소로 문자 집합을 지정함으로써, 제목과 이하 문서들을 올바르게 표시할 수 있게 함이 좋다

    ↓html

    <meta charset="utf-8">
  • Title

    • 브라우저 탭, 열린 윈도우, 방문 기록, 검색 결과 등에서 보여지는 제목

    ↓html

    <title>My Homepage</title>
  • Viewport

    • 뷰 영역의 너비와 관계없이 컨텐츠를 보기 좋게 표현(반응형 웹)하기 위한 기본적인 설정을 기술한다. Viewport 설정은 사이트의 접근성 품질 검사 항목 중 하나로, 콘텐츠를 확대/축소 가능하고, 최대 크기 제한이 없을 때 검사를 통과한다

    ↓html

    <meta name="viewport" content="width=device-width, initial-scale=1, user-scalable=yes">
  • Stylesheet link

    • 스타일을 먼저 지시함으로써 렌더링한 요소를 다시 그리는 작업을 피할 수 있다. CSS의 다운로드 및 구문 해석이 완료되기 전까지 렌더링이 차단되므로, 첫 화면에서 미사용되는 스타일이 많을수록 렌더링까지의 지연 시간이 늘어날 수 있다

  • Icon

    • ↓html

    <link rel="icon" href="/imgs/favicon.png"/> ↑ /favicon.ico 대신 사용할 즐겨찾기 아이콘 지정 <link rel="apple-touch-icon" href="/imgs/iOS-icon.png"/> ↑ 비표준. 사이트를 iOS 기기의 홈 화면에 추가한 경우의 아이콘
  • Script

    • 스크립트는 기본적으로 CSS와 마찬가지로 다운로드 및 해석되기 전까지 렌더링이 차단되고, 이에 더해 다른 리소스의 다운로드까지 중지된다. 이러한 지연을 방지하기 위한 속성이 존재한다

    • defer : HTML 렌더링을 차단하지 않는다. 스크립트는 렌더링이 완료된 후에야 실행된다
    • async : 스크립트를 다운로드하는 동안은 HTML 렌더링을 차단하지 않는다. 다운로드 후 스크립트 실행 중에는 차단한다
    • 위 속성들은 외부 스크립트를 사용하는 경우 적용 가능하며, 인라인 스크립트에는 적용되지 않는다

    다른 요소의 속성에 스크립트를 작성할 수도 있다

    ↓html

    <a href:"javascript: myFunction();">
  • Base URL

    • 모든 상대경로에 대한 기본 주소를 지시한다

    ↓html

    <base href="https://example.com">

    한 가지 문제는, <a href="#ref">를 <a href="https://example.com#ref">로 변경해 버린다는 것

CSP; Content-Security-Policy

CSP에는 버전이 존재하며, 오래된 브라우저는 최신 기능을 지원하지 않을 수 있다

Fetch directives
  • child-src : 웹 워커, 페이지에 포함된 프레임 컨텐츠의 출처를 제한한다

    • frame-src, worker-src로 각각 따로 정의 가능

    ↓e.g.

    <meta http-equiv="content-security-policy" content="child-src https://youtube.com https://m.youtube.com">
  • connect-src : XHR, WebSocket, EventSource를 통해 연결할 수 있는 출처를 제한한다
  • default-src : 명시적으로 설정하지 않은 fetch directive에 대한 기본값
  • font-src : 웹 폰트(@font-face) 출처 제한
  • frame-src : 프레임 컨텐츠 출처 제한
  • img-src : 이미지(favicon 포함) 출처 제한
  • manifest-src : 웹앱 매니페스트 파일 출처 제한
  • media-src : <audio>, <video>, <track> 출처 제한
  • script-src : 스크립트 및 웹 어셈블리 출처 제한
  • style-src : CSS 출처 제한
  • worker-src : 웹 워커 출처 제한
Document directives
  • base-uri : <base> 요소의 URL을 제한한다
  • sandbox : 페이지가 해당 sandbox 속성을 가진 iframe을 통해 로드된 것처럼 취급된다
Navigation directives
  • form-action : 폼을 제출할 수 있는 URL 제한
Values
  • none : 전부 비허용
  • self : 현재 origin 허용
  • strict-dynamic : hash(e.g. sha256-xxx) 또는 nonce(e.g. nonce-xxx) 값을 이용한 출처 허용 제한을 간략하게 해준다

    • CSP Level 3 기능이며 strict-dynamic를 사용한 경우, 이를 지원하는 브라우저는 출처 목록 중 아래 사항들을 무시한다

    • 'unsafe-inline'
    • 'self'
    • Host 리스트
    • Scheme 리스트

    ↓사용 예시

    script-src 'nonce-OneTimeUsedRandomString' 'strict-dynamic';default-src 'self';

    ↓허용되는 스크립트

    <script src="/script-loader.js" nonce="OneTimeUsedRandomString"></script> <script nonce="OneTimeUsedRandomString"> doWhatever(); </script> // script-loader.js 안에서, let s = document.createElement('script'); s.src = "https://cdn.example.com/some-script-you-need.min.js"; document.body.appendChild(s);
  • unsafe-inline : 인라인 리소스(자바스크립트 & CSS) 허용

    • 인라인 리소스를 비허용하는 것이 CSP를 사용함으로써 얻을 수 있는 가장 큰 보안 이점이라는 점은 유의

  • unsafe-eval : eval(), setImmediate(), window.execScript() 허용
  • Host : 특정 호스트 주소 지정. 스킴, 포트, 경로(prefix)는 선택사항

    • e.g. example.com, *.example.com, https://example.com/api

  • Scheme : e.g. https:, data:, blob: 등
HTTP 응답 헤더로도 설정 가능하다

일부 directive들(frame-ancestors, report-uri, sandbox 등)은 헤더에서만 설정 가능하다

    Bootstrap, W3C.CSS과 같은 인기 있는 CSS Framework들을 사용하면 체계적으로 정의된 클래스들 덕분에 수월하게 문서의 의미 정보를 뽑아낼 수 있다

Canvas

  • canvas를 지원하지 않는 브라우저를 위해 태그 안에 fallback content(대체 텍스트 또는 DOM)를 적어주면 좋다
  • 스크립트에서 canvas의 지원을 확인하는 방법 : DOM이 HTMLCanvasElement 클래스인지 확인
  • 캔버스의 기본 그리기 요소는 직사각형과 path(일련의 점들을 선분으로 연결) 두 가지
    • 연습용 캔버스1

interfaceHTMLCanvasElement : HTMLElement

  • width, height : 각각 캔버스의 너비, 높이. 기본값은 300, 150
  • getContext(contextType[, contextAttributes])

    • contextType : '2d'(CanvasRenderingContext2D), 'webgl'(WebGLRenderingContext), 'webgl2'(WebGL2RenderingContext), 'bitmaprenderer'(ImageBitmapRenderingContext)

  • toDataURL([type][, encoderOptions])

    • type : default 'image/png', encoderOptions : default 0.92, 이미지 퀄리티[0, 1]

  • toBlob(callback[, mimeType][, qualityArgument])

    • callback : Blob을 먹는 함수, mimeType : default 'image/png', qualityArgument : 이미지 퀄리티[0, 1]

사각형 그리기

  • clearRect(x, y, width, height)

    • 지정 사각형을 rgba(0,0,0,0)로 초기화. 이후 path를 그릴 땐 beginPath()를 다시 호출해야 함에 유의

  • fillRect(x, y, width, height)

    • 지정 사각형을 현재 fillStyle로 채우기 == rect() + fill()

  • strokeRect(x, y, width, height)

    • 지정 사각형에 현재 strokeStyle로 외각선 그리기 == rect() + stroke()

텍스트 그리기

  • fillText(text, x, y[, maxWidth])

    • 텍스트를 그리고 글자 내부는 현재 fillStyle로 채운다

  • strokeText(text, x, y,[, maxWidth])

    • 텍스트를 그리고 글자 외각선은 현재 fillStyle로 채운다

  • measureText(text)

    • 텍스트를 그린 결과가 어떻게 될지 미리 알 수 있는TextMetrics 객체 반환

path 생성

    아래 메서드들은 실제 그리기 작업을 수행하지는 않는다

  • beginPath() : 그리기 시작 명시
  • closePath() : path 조각이 하나 이상 추가되었고, 마지막 정점이 시작점과 다른 경우 두 점을 잇는 선분을 추가한다
  • moveTo(x, y) : path 조각의 시작점을 설정
  • lineTo(x, y) : 마지막 정점으로부터 (x, y)까지의 선분을 추가
  • bazierCurveTo(cp1x, cp1y, cp2x, cp2y, x, y)

    • 마지막 정점으로부터 (x, y)까지를 잇는, 2개의 조절점을 이용하는베지에 곡선을 추가한다

  • quadraticCurveTo(cpx, cpy, x, y)

    • 마지막 정점으로부터 (x, y)까지를 잇는, 1개의 조절점을 이용하는베지에 곡선을 추가한다

  • arc(x, y, radius, startAngle, endAngle[, anticlockwise])

    • (x, y)를 기준점으로 하는 반지름 radius인 호를 추가한다. anticlockwise의 기본값은 false. anticlockwise를 true로 한다고 각도의 증감 방향이 바뀌진 않는다

  • arcTo(x1, y1, x2, y2, radius)
    1. 조절점1(x1, y1), 조절점2(x2, y2), 선분1(마지막 정점-조절점1), 선분2(조절점1-조절점2)를 생각한다
    2. 선분1, 선분2 모두와 거리가 radius만큼 떨어진 정점2를 생각한다. 그러면 정점2에서 반지름 radius인 원을 그렸을 때 선분1, 선분2와의 교점1, 교점2가 존재한다
    3. lineTo(교점1)
    4. 교점1부터 교점2까지의 호를 추가한다. arc(정점2, radius, ...)
  • ellipse(x, y, radiusX, radiusY, rotation, startAngle, endAngle[, anticlockwise])

    • rotation은 라디안 단위의 값으로, 해당 값만큼 타원을 회전시킨다

  • rect(x, y, width, height)

  • fill([fillRule])

      • 현재 또는 지정된 path의 영역 내부를 칠한다

    drawFocusIfNeeded(path, element)

    • element가 focus를 얻은 상태면 현재 또는 지정된 path 주변에 강조선을 그린다

  • scrollPathIntoView([path])

    • Experimental. Element.scrollIntoView()와 유사하게, 현재 또는 지정된 path가 스크린에 표시되도록 스크롤을 이동한다

  • clip([fillRule])
    clip(path[, fillRule])

    • fillRule은 fill()과 동일. 현재 또는 지정된 path를 관심영역으로 설정한다. 이후의 그리기는 관심영역 내부의 것만 렌더링된다

  • isPointInPath(x, y[, fillRule])
    isPointInPath(path, x, y[, fillRule])

    • fillRule은 fill()과 동일. 정점이 현재 또는 지정된 path의 내부에 있는지 판별한다

  • isPointInStroke(x, y)
    isPointInStroke(path, x, y)

    • 정점이 현재 또는 지정된 path의 외각선 위에 있는지 판별한다

    + lineDashOffset : 선분을 그리기 시작할 offset. default 0.0

변환

DOMMatrix

  • 2D 및 3D 변환을 나타내는 4x4 행렬
  • 속성
    • readonly is2D : true면 2D 변환행렬, false면 3D
    • readonly isIdentity : 단위행렬인지 여부
    • m11, m12, ..., m43, m44 : 행렬 성분. double
    • a, b, c, d, e, f : 2D 변환에 사용되는 값. 차례로 2x2 변환행렬, dx, dy에 해당하며, m11, m12, m21, m22, m41, m42에 대응한다
  • setTransform(a, b, c, d, e, f)
    setTransform(matrix) + getTransform()
  • transform(a, b, c, d, e, f)

    • old = ctx.getTransform();
    식[m_{old} = \begin{bmatrix} old.a & old.c & old.e \\ old.b & old.d & old.f \\ 0 & 0 & 1 \end{bmatrix}]식
    식[m_{new} = m_{old} \begin{bmatrix} a & c & e \\ b & d & f \\ 0 & 0 & 1 \end{bmatrix}]식

  • scale(x, y)

    • 기본적으로 캔버스 내의 단위는 정확히 px과 같지만 이를 변경할 수 있다. 음수 스케일링은 요소를 뒤집는 효과가 있다

  • rotate(angle) : (0, 0)을 기준으로 시계방향 회전. 단위는 라디안
  • translate(x, y)

    • 현재의 dx, dy에 각각 x, y만큼 추가한다

전역 속성

  • globalAlpha

    • 캔버스에 그려지기 전에 도형, 이미지 등에 적용되는 투명도. default 1.0

  • globalCompositeOperation

    • 캔버스의 기존 내용에 새로운 도형, 이미지를 추가할 때의 합성 방법. 기본값은 덮어쓰기('source-over').

비트 조작

ImageData

  • 캔버스 비트 배열 정보를 갖는 인터페이스. CanvasRenderingContext2D 메서드 또는 생성자로 인스턴스를 얻을 수 있다
  • 생성자
    • ImageData(array, width[, height])

      • array는 RGBA값을 담은 Uint8ClampedArray. 길이가 정확하지 않으면 에러 발생

    • ImageData(width, height) : 검은 사각형
  • 속성
    • data : RGBA값을 가진 Uint8ClampedArray
    • width, height
  • getImageData(sx, sy, sw, sh)

    • 지정 사각형 영역의 정보를 가진 ImageData 반환. 여기서 지정한 사각형은 변환 행렬의 영향을 받지 않으며, 캔버스 바깥을 포함하는 경우 RGBA(0, 0, 0, 0)로 채워진다

  • putImageData(imageData, dx, dy[, dirtyX, dirtyY, dirtyWidth, dirtyHeight])

    • 지정 위치에 imageData를 덮어쓴다. dirty* 매개변수로 그릴 imageData 영역을 지정할 수 있다

  • createImageData(width, height)
    createImageData(imagedata)

    • RGBA(0, 0, 0, 0)으로 채워진 새 ImageData 인스턴스를 반환한다. imagedata로 호출한 경우, imagedata의 width, height만 사용된다

상태 저장/복구

  • canvas : 연관 HTMLCanvasElement 객체. null일 수 있음
  • save()

    • 다음 항목들을 저장한다 : 변환행렬, 관심영역, lineDash, strokeStyle, fillStyle, globalAlpha, lineWidth, lineCap, lineJoin, miterLimit, lineDashOffset, shadowOffsetX, shadowOffsetY, shadowBlur, shadowColor, globalCompositeOperation, font, textAlign, textBaseline, direction, imageSmoothingEnabled

  • restore() : save()로 저장한 사항 복구
