Box Model

CSS가 표시하는 모든 것은 박스다

각 박스는 하위 영역들(다시 박스)로 구성된다; '컨텐츠 박스' > '패딩 박스' > '테두리 박스' > '여백 박스'

  • 컨텐츠 박스 : 자식 요소들이 포함될 수 있고, 컨텐츠 박스의 크기가 자식 요소들의 크기에 의해 (동적으로) 결정되도록 할 수도 있다
  • 패딩 박스 : 컨텐츠 박스를 감싸는 영역으로, 요소에 적용된 배경은 이 영역까지 표시된다. 또, 오버플로 속성으로 인한 스크롤바도 이 영역에 표시된다
  • 테두리 박스 : 패딩 박스를 감싸는 영역으로, 박스의 경계를 표시한다. border 속성으로 제어한다.
  • 여백 박스 : 박스 바깥의 공간으로, 여백의 크기는 박스의 크기에 영향을 미치지 않는다.

박스 모델 제어

  • 모든 웹 브라우저는 HTML 문서에 기본 스타일시트를 적용한다
  • display 속성 : 요소를 block 또는 inline 요소를 취급하고, 자식 박스들을 어떻게 배치할 지 제어한다
  • box-sizing 속성 : width, height를 컨텐츠 박스까지 적용할 지(기본값), 테두리 박스까지 적용할 지 제어한다

선택자

단순 선택자

  • 범용 선택자

    • * : 모든 요소를 선택한다

  • 타입 선택자

    • div : 모든 div 요소를 선택한다

  • 클래스 선택자

    • .dialog : dialog 클래스를 가진 모든 요소를 선택한다

  • ID 선택자

    • #target : target ID를 가진 요소를 선택한다

  • 속성 선택자
    • [attr] : attr 속성을 갖는 요소
    • [attr='value'] : attr 속성값이 'value'와 일치하는 요소
    • [attr='value' s] : attr 속성값이 'value'와 일치하는 요소(대소문자 구별 O)
    • [attr='value' i] : attr 속성값이 'value'와 일치하는 요소(대소문자 구별 X)
    • [attr*='value'] : attr 속성값이 'value'를 포함하는 요소
    • [attr^='value'] : attr 속성값이 'value'로 시작하는 요소
    • [attr$='value'] : attr 속성값이 'value'로 끝나는 요소
  • 그룹화 선택자

    • 콤마(,)를 이용해 같은 스타일을 여러 요소에 적용시킬 수 있다

    ↓css

    /* strong 요소, em 요소, lang 속성을 가진 요소의 글자 색을 빨강으로 한다 */ strong, em, [lang] { color: red; }

의사 클래스 및 의사 요소

의사 클래스

  • 요소들의 상태를 기준으로 선택하는 데 사용할 수 있는 의사 클래스가 정의되어 있다. ':'을 이용해 의사 클래스를 지정한다
  • a: active, hover, link, visited
  • input: checked, disabled, enabled, focus, in-range, out-of-range, valid, invalid, read-only, read-write, required, optional
  • :first-child, last-child, nth-child(n), nth-last-child(n)
  • :not(selector)

의사 요소

  • 요소의 특정 부분을 선택하거나, 요소의 앞뒤에 내용을 추가하는 데 사용한다. '::'를 이용해 의사 요소를 지정한다
  • ::first-letter : 첫 번째 글자
  • ::first-line : 첫 번째 줄
  • ::marker : 리스트 아이템의 글머리 기호(또는 번호)
  • ::before : 요소 앞부분에 추가
  • ::after : 요소 뒷부분에 추가
  • ::selection : 블록 지정된 요소

결합기(Combinator)

결합기는 두 선택자 사이에 위치하여, 특정한 관계를 만족하는 요소를 선택할 수 있게 해준다

  • '>' : 자식 요소
  • ' '(스페이스) : 후손 요소
  • '+' : 바로 다음 형제 요소
  • '~' : 다음 모든 형제 요소

The cascade

요소에 대해 둘 이상의 스타일이 존재할 때, 브라우저는 아래 순서에 따라 적용할 스타일을 결정한다.

스타일의 위치와 순서

  • 동일한 CSS 안에서의 충돌 : 스크립트 뒷부분에 정의된 것이 우선
  • 두 CSS 간 충돌 : 보다 나중에 로드/정의된 것이 우선
  • 요소 내부의 style 속성으로 지정한 것이 요소 외부의 것보다 우선

선택자의 특정성

위치상 후순위 스타일이더라도, 선택자가 요소를 보다 자세하게 특정하는 경우 보다 높은 우선순위를 가질 수 있다

선택자점수비고
*0
타입1
클래스, 의사 클래스, 속성10:not() 자체는 점수가 없지만, 인자로 넘겨진 선택자에 대해선 점수가 부여된다
ID100
inline style1,000
!important10,000

출처

우선순위가 낮은 순서부터,

  1. 브라우저 기본 스타일
  2. OS(e.g. 기본 폰트) 또는 브라우저 확장 스타일
  3. 직접 작성된 CSS
  4. 직접 작성된 !important
  5. OS 또는 브라우저 확장 !important
  6. 브라우저 기본 !important

중요성

어떤 스타일은 기본적으로 다른 것보다 우선한다. 우선순위가 낮은 순서부터,

  1. 일반
  2. animation
  3. !important
  4. transition

상속

  • 일부 스타일 속성은 자식 요소로 상속된다

    • color, font, direction, line-height, text-align, visibility 등

  • inherit 키워드를 이용한 명시적 상속

    • ↓css

    strong { font-weight: 900; } .my-class { font-weight: 500; } .my-class strong { font-weight: inherit; } /* 500이 적용된다 */
  • initial 키워드를 이용한 상속 취소(기본값 사용)
  • unset 키워드
    • 속성이 상속 가능하면 inherit으로, 아니면 initial로 작동
    • "all: unset;"으로 간편하게 모든 속성 상속 취소 가능

Layout

display 속성의 이해

↓text

display = [ <display-outside> || <display-inside> ] | <display-listitem> | <display-internal> | <display-box> | <display-legacy>

<display-outside>

Flow layout(기본 배치; inline/block)에서 요소를 어떻게 배치할 지 제어한다

  • inline

    • 너비에 여유가 있다면 inline 요소들은 한 줄에 같이 표시된다. margin, padding 모두 무시된다.

  • block

    • 각 block 요소는 한 줄을 온전히 차지한다; 박스 전후에 개행을 삽입한다

<display-inside>

자식 요소들을 내부에서 어떻게 배치할 지 제어한다

  • flow

    • Flow layout에 따라 자식 요소들을 배치한다

  • flow-root

    • flow + 새로운 block formatting context(BFC; 블록 박스가 자리하고, float 요소가 상호작용하는 공간)를 생성한다

  • table

    • <table> 요소처럼 취급된다. "display: table-cell;" 속성을 가진 자식들은 테이블 열이 제각기 너비를 차지하듯이 부모의 너비를 나눠갖는다. 추가로 부모 요소에서 table-layout 스타일 속성도 효과를 갖는다

  • flex

    • Flexible box layout에 따라 자식 요소들을 배치한다. 자식 요소들은 자동으로 flex item이 된다

  • grid

    • Grid layout에 따라 자식 요소들을 배치한다

  • ruby

    • <ruby> 요소처럼 취급된다

<display-listitem>

  • list-item

    • 요소를 리스트 항목으로 취급한다

<display-internal>

table, ruby와 같이 복합 구조를 갖는 요소들의 자식들이 갖는 역할을 제어한다

  • table-row-group : <tbody>에 대응
  • table-header-group : <thead>에 대응
  • table-footer-group : <tfoot>에 대응
  • table-row : <tr>에 대응
  • table-cell : <td>에 대응
  • table-column-group : <colgroup>에 대응
  • table-column : <col>에 대응
  • table-caption : <caption>에 대응
  • ruby-base : <ruby>에 대응
  • ruby-text : <rt>에 대응
  • ruby-base-container : 익명 박스 생성
  • ruby-text-container : <rtc>에 대응

<display-box>

  • contents

    • 요소 자체는 박스를 생성하지 않는다. 그러나 자식 요소들과 의사 요소들은 정상적으로 박스를 생성한다. 즉, 요소는 트리 상에서만 존재하는 컨테이너가 된다

    예를 들어 display: flex;는 자식 요소들만을 flex item으로 취급하는데, 자식 요소에 display: contents;를 적용하여, 손자 요소들을 flex item처럼 표시할 수 있다

  • none

    • 레이아웃에 요소를 표시하지 않는다. 자식들에게도 상속되어 영향을 미친다. 자리는 차지하되 렌더링만 하지 않기를 원한다면 visibility 속성을 사용할 수 있다

<display-legacy>

CSS 2에서는 단일 키워드로 결합된 값들을 정의했다

  • inline-block : inline flow-root와 동일
  • inline-table : inline table과 동일
  • inline-flex : inline flex와 동일
  • inline-grid : inline grid와 동일

Flexible box layout

일직선 상에서 자식 요소들의 크기를 유연하게 조정한다 -- 크기를 넓혀 빈 공백을 채우거나, 오버플로되지 않도록 축소

Basic concepts of flexbox

  • Flex container의 자식들(flex item)은 기본적으로 아래와 같이 배치된다
    • flex-direction: row; 한 줄로 표시되며, 주 축의 시작 위치부터 배치된다
    • flex-wrap: nowrap; 공간이 부족해도 개행하지 않고 오버플로된다
    • flex-grow: 0; 빈 공간이 존재하더라도 아이템 크기가 늘어나지 않는다
    • flex-shrink: 1; 오버플로 되는 경우 아이템 크기가 줄어들 수 있다
    • flex-basis: auto; 각 아이템 크기는 요소에 지정된 크기. 없으면 요소의 컨텐츠 크기
    • align-items: stretch; 아이템 크기를 직교 축 끝까지 확장한다
  • 컨테이너 flex-direction 속성 : 요소를 배치할 축을 설정할 수 있다; row, row-reverse, column, column-reverse
  • 컨테이너 flex-wrap 속성 : nowrap(한 줄에 다 못채우면 오버플로), wrap(한 줄에 다 못채우면 개행. 각 줄을 별개의 컨테이너로 생각해야 한다)
  • 컨테이너 flex-flow 단축 속성 : flex-direction flex-wrap
  • 아이템 flex-basis 속성 : 줄에 빈 공간이 존재하는 경우, 아이템 크기를 지정한 크기까지 키운다

    • 기본값 auto : 요소에 지정된 크기가 존재하면 해당 값을 사용하고, 지정된 크기가 없으면 요소의 컨텐츠 크기를 사용한다

  • 아이템 flex-grow 속성 : 모든 아이템이 flex-basis 만큼 차지하고도 공간이 남으면, 각 가중치만큼 나눠갖는다
  • 아이템 flex-shrink 속성 : flex-grow와 다르게, 오버플로 시 수축하는 비중을 지정한다
  • 아이템 flex 단축 속성 : flex-grow flex-shrink flex-basis
    • flex: initial == flex: 0 1 auto
    • flex: auto == flex: 1 1 auto
    • flex: none == flex: 0 0 auto
    • flex: n == flex: n 1 0

Aligning Items in a Flex Container

  • 컨테이너 justify-content 속성 : 주 축에 대하여 아이템을 어떻게 배치할 지 지정한다
    • flex-start(기본값) : 주 축의 시작 위치부터 배치한다
    • flex-end : 주 축의 끝부터 배치한다
    • center : 가운데 정렬
    • space-around : 가운데 정렬 + 각 아이템 좌우에 동일한 여백
    • space-between : 가운데 정렬 + 각 아이템 사이에 동일한 여백
    • space-evenly : 가운데 정렬 + 모두 동일한 여백
  • 컨테이너 align-items 속성 : 직교 축에 대하여 아이템을 어떻게 배치할 지 지정한다
    • stretch(기본값) : 직교 축의 전체 공간을 차지한다
    • flex-start : 직교 축의 시작부터 자리를 차지한다
    • flex-end : 직교 축의 끝부터 자리를 차지한다
    • center : 직교 축의 중간에 위치한다
