프로시저

  • 프로시저 호출
    1. CREATE PROCEDURE로 정의된 내장 프로시저를 호출한다

      2. ↓ sql

      CALL sp_name([parameter[, ...]])
    2. 매개변수가 없다면 괄호를 생략할 수 있다

      3. ↓ sql

      CALL sp_name[()]
    3. OUT, INOUT 매개변수를 출력으로 이용할 수 있다. 이는 프로시저 내부에서의 호출에서도 유효하다

      4. PREPARE와 EXECUTE를 이용한 prepared CALL 구문에서는 IN, OUT, INOUT 인자에 placeholder를 이용할 수 있다

      ↓ sql

      mysql> SET @increment = 10; mysql> PREPARE s FROM 'CALL p(?, ?)'; mysql> EXECUTE s USING @version, @increment; mysql> SELECT @version, @increment;
  • 프로시저 작성
    1. 가정 1 : 유저 데이터를 4개 DB(user_1 ~ user_4), 총 32개의 테이블(_1 ~ _32)로 분산 저장

      2. 예. 유저의 소지 아이템과 관련된 i_user_item 테이블에 대하여, user_1에는 i_user_item_1, i_user_item_5, ..., i_user_item_29까지 존재

    2. 가정 2 : 각 테이블에 데이터 입력 시, 테이블 suffix는 user_id % 32 + 1로 결정된다
    3. 분산 저장된 DB와 테이블들을 한꺼번에 검색하는 프로시저

이벤트

  1. 서버 실행 중 이벤트 사용 설정

    2. ↓ sql

    mysql> SET GLOBAL event_scheduler = on; mysql> SET @@global.event_scheduler = on; mysql> SET GLOBAL event_scheduler = 1; mysql> SET @@global.event_scheduler = 1;
  2. 서버 재실행 후 이벤트 사용 설정

    3. 종료 후 my.cnf or my.ini 에 옵션(event_scheduler=On)을 추가하고 mysql 시작

  3. 이벤트 생성

    4. ↓ sql

    SET GLOBAL event_scheduler = ON; delimiter $$ create event if not exists reset_quest_clear_count_daily_event on schedule every 0 DAY_HOUR do begin update user.i_user_quest_status set clear_count = 0; end $$ delimiter;
  4. 이벤트 확인

    5. ↓ sql

    SHOW PROCESSLIST; SHOW EVENTS;

CSV 입출력

  • CSV로부터 입력

    • ↓ sql

    LOAD DATA LOCAL INFILE "/~.csv" replace into table table_name fields terminated by ',' enclosed by '\"' IGNORE 1 LINES;
  • CSV로 출력
    • 기본 명령

      • ↓ sql

      SELECT select_clause FROM table_expression INTO OUTFILE "~.csv" FIELDS TERMINATED BY "," LINES TERMINATED BY "\n";
    • 출력 가능 경로 확인 : SHOW VARIABLES LIKE 'secure_file%';
    • 기본 출력은 멀티 라인 텍스트를 \로 연결하여 출력하고, null값을 \N로 출력한다. 아래 툴을 이용하면, 멀티 라인 문자열과 빈 문자열만 ""로 감싸고, null값은 비워서 표현하는 select_clause를 얻을 수 있다

자잘한 것들

  • 테이블 컬럼 순서 변경

    • ↓ sql

    ALTER TABLE table_name MODIFY COLUMN col_name1 TYPE AFTER col_name2; ALTER TABLE table_name CHANGE COLUMN before_col_name after_col_name TYPE AFTER col_name;
  • 테이블 정의 확인

    • ↓ sql

    DESC table_name; SHOW [FULL] COLUMNS FROM table_name [WHERE...];
  • 테이블 복사 생성

    • ↓ sql

    create table new_table_name like base_table_name;
