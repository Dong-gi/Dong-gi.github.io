개요

OAuth(Open Authorization)는 3rd party에게 유저를 대신하여 특정 private resource에 대한 특정 권한을 특정 기간 동안 이용할 수 있는 토큰을 발행함으로써, 인증과 권한 부여 두 가지 기능을 모두 수행한다.

역할

  • Resource owner : 자원에 대한 권한을 부여할 수 있는 주체. 보통 사용자.
  • Client : Resource server에 자원을 요청하고, 이를 이용한 서비스를 제공하는 애플리케이션
  • Resource server : 자원에 대한 서비스 API를 제공하는 서버
  • Authorization server : 인증된 Client에 권한을 부여하는 서버

단일 Authorization server에 대해 여러 Resource server가 존재하더라도 검증 문제가 발생하지 않는다; SSO 용이.

권한 승인 절차

  1. Client -> Resource owner : 권한 요청
  2. Resource owner --> Client : 권한 승인
  3. Client -> Authorization server : 토큰 요청

    4. Client는 미리 Authorization server에 등록되거나, 요청 내 정보로 검증되어 신뢰할 수 있음이 보장된다

  4. Authorization server --> Client : 토큰 발행(access token + refresh token)

    5. refresh token은 access token 만료 등의 사유로 신규 access token을 발급받기 위해 요청할 때 사용

  5. Client -> Resource server : 자원 요청
  6. Resource server -> Authorization server : 토큰 검증 요청
  7. Authorization server --> Resource server : 토큰 검증
  8. Resource server --> Client : 자원 전송
