태그
  • 문서 요소의 타입을 나타내는 이름으로, 사용자가 임의로 만들 수도 있다.
  • 태그는 <과 > 사이에 이름을 적음으로써 표현되며, 여는 태그와 닫는 태그 사이에 내용이 포함될 수 있다.
  • 여는 태그는 <name>, 닫는 태그는 </name>로 표현한다. 아래는 HTML 문서의 기본 골격을 나타내는 태그들이다.
    1. <!DOCTYPE html> : 문서가 HTML5로 작성되었음을 의미한다.
    2. <html></html> : root HTML element
    3. <head></head> : 문서에 대한 meta information
    4. <title></title> : 제목표시줄에 표시될 제목
    5. <body></body> : page content
속성
  1. 모든 HTML 태그는 속성을 가질 수 있다. 속성은 여는 태그 내부에 이름='값' 쌍으로 표현한다.
  2. 속성 값을 표현하기 위해 '', "" 모두 이용 가능하다. 아래는 대표적인 속성들이다.
    • html태그 lang : 문서의 국가 언어.
    • title : 노드에 대한 툴팁(마우스 올려두면 표시).
    • style : CSS 속성을 지시
    • disabled : 입력 태그 요소를 사용할 수 없음.
    • id : 태그에 고유한 id를 부여한다.
    • class : 태그에 특정 그룹의 class를 부여한다.
기본 태그
  • heading : h1 ~ h6
  • paragraph : p
  • link : a
    1. href 속성 : 이동할 위치나 주소
      2. # : 현재 페이지의 특정 위치(id)로 이동하기 위한 표기. '#here'은 현재 페이지의 HTML 요소 중 id가 'here'인 것을 가리킨다.
    2. 링크 상태를 나타내는 속성 : link, visited, hover, active
    3. target 속성 : _blank(새탭이나 새창에서 열기), _self, _parent, _top, framename
  • 이미지 : img
    1. src 속성 : 이미지 리소스의 위치나 주소
    2. alt 속성 : 이미지를 불러오지 못할 때, 혹은 시각장애인을 위한 대체 문자열
    3. width, height 속성 : 이미지 크기
  • line break : br
  • 수평선 : hr
  • 해석 없이 출력 : pre
  • text
    1. <b></b> : bold
    2. <strong></strong> : bold & important
    3. <i></i> : italic
    4. <em></em> : italic & important
    5. <small></small> : 작은 글자
    6. <mark></mark> : 형광펜으로 칠한 것처럼
    7. <del></del> : 취소선
    8. <ins></ins> : 삽입 문구에 밑줄
    9. <sub></sub>, <sup></sup> : 아랫첨자, 윗첨자
  • 주석 : <!-- 주석 -->
  • table
    1. table은 thead와 tbody로 이루어진다.
    2. thead와 tbody는 tr들로 이루어진다.
    3. tr은 th와 td들로 이루어진다.
    4. colgroup을 이용해 열들을 논리적으로 분리할 수 있다.
  • list
    1. unordered list : <ul></ul>
    2. ordered list : <ol></ol>
      3. type 속성 : "1"(default), "A", "a", "I"(로마숫자), "i"
    3. list item : <li></li>
  • div : 가로 100%를 점유하는 block element 대표
  • span : 가로 공간이 여유롭다면 병렬로 표시되는 inline element 대표
  • script
    1. <script></script> 사이에 JavaScript 코드 작성
    2. <noscript></noscript> 사이에 지원하지 않는 브라우저에 대한 메시지 작성
    3. <a href="javascript:;">처럼 속성으로 넣는 것도 가능
  • meta
    1. 인코딩 명시 : <meta charset="UTF-8">
    2. 검색엔진을 위한 키워드 설정 : <meta name="keywords" content="Word1, Word2, Word3">
    3. 페이지 설명 : <meta name="description" content="~~">
    4. 페이지 업데이트 설정 : <meta http-equiv="refresh" content="30"> (30초마다 갱신)
    5. 반응형 웹을 위한 viewport 설정 : <meta name="viewport" content="width=device-width, initial-scale=1.0">
  • base : 페이지에서 사용하는 모든 상대주소의 시작점
    • 예. <base href="https://dong-gi.github.io/" target="_blank">

  3. form 속성 : action="주소", target="_blank | _top | ...", method="get | post"
  4. <fieldset> : 입력 요소 그루핑, <legend> : 그룹 설명
  5. <select name=""><option value=""></option> ~ </select>
  6. <textarea rows="" cols="">, <button onclick="">
  7. <input type=""> : color, date, email, month, number, range, time, ...
  8. 입력 요소 속성 : name, max, maxlength, min, pattern, readonly, required, disabled, size, ...
  9. 자세한 설명은여기로
HTML5
  • 시맨틱; Semantic
    • HTML5 이전의 웹페이지들은 table 태그를 주로 사용하여 디자인했다. 중첩된 table들로 이루어진 웹페이지는 검색 엔진을 비롯한 프로그램이 기계적으로 의미를 찾기 어려워 HTML5는 문서에 대한 의미를 태그가 포함하는 방향을 모색했다.
  • 시맨틱 태그
    • header, nav, section, article, aside, footer, ... 자세한 설명은 여기로
  • 시맨틱 CSS
    • HTML5로 접어들면서 HTML 태그들은 골격만을 정의하고, 디자인은 JavaScript와 CSS로 보다 확실히 분리되었다. 디자인의 특징은 의미 범주가 같은 것들을(class) 비슷하게 표현한다는 것이며, 따라서 CSS 선택자들은 문서의 의미 정보를 내포하고 있다고 할 수 있다.
    Bootstrap, W3C.CSS과 같은 인기 있는 CSS Framework들을 사용하면 체계적으로 정의된 클래스들 덕분에 수월하게 문서의 의미 정보를 뽑아낼 수 있다.
  • 반응형 웹
    1. 스마트폰 대중화 이전의 일반 사용자 컴퓨팅 환경은 가로 768px 이상의 직사각형 모니터가 대부분이었다.
    2. 스마트폰이 대중화 되면서 다양한 사용자 웹서핑 환경에 맞춰 알맞게 페이지를 보여줄 필요가 생겼다.
    3. viewport로 대표되는 screen 속성을 이용해 페이지를 다시 로드할 필요 없이 디자인이 변경되는 반응형 웹 기술이 발전하게 되었다.

Canvas
  • canvas를 지원하지 않는 브라우저를 위해 안에 fallback content(대체 텍스트 또는 DOM)를 적어주면 좋다
  • 스크립트에서 canvas의 지원을 확인하는 방법 : DOM이 HTMLCanvasElement 클래스인지 확인
  • 캔버스의 기본 그리기 요소는 직사각형과 path(일련의 점들을 선분으로 연결) 두 가지
interface HTMLCanvasElement : HTMLElement
  • width, height : 각각 캔버스의 너비, 높이. 기본값은 300, 150
  • getContext(contextType[, contextAttributes])
    • contextType : '2d'(CanvasRenderingContext2D), 'webgl'(WebGLRenderingContext), 'webgl2'(WebGL2RenderingContext), 'bitmaprenderer'(ImageBitmapRenderingContext)
  • toDataURL([type][, encoderOptions])
    • type : default 'image/png', encoderOptions : default 0.92, 이미지 퀄리티[0, 1]
  • toBlob(callback[, mimeType][, qualityArgument])
    • callback : Blob을 먹는 함수, mimeType : default 'image/png', qualityArgument : 이미지 퀄리티[0, 1]
interface CanvasRenderingContext2D
사각형 그리기
  • clearRect(x, y, width, height)
    • 지정 사각형을 rgba(0,0,0,0)로 초기화. 이후 path를 그릴 땐 beginPath()를 다시 호출해야 함에 유의.
  • fillRect(x, y, width, height)
    • 지정 사각형을 현재 fillStyle로 채우기 == rect() + fill()
  • strokeRect(x, y, width, height)
    • 지정 사각형에 현재 strokeStyle로 외각선 그리기 == rect() + stroke()
텍스트 그리기
  • fillText(text, x, y[, maxWidth])
    • 텍스트를 그리고 글자 내부는 현재 fillStyle로 채운다.
  • strokeText(text, x, y,[, maxWidth])
    • 텍스트를 그리고 글자 외각선은 현재 fillStyle로 채운다.
  • measureText(text)
    • 텍스트를 그린 결과가 어떻게 될지 미리 알 수 있는 TextMetrics 객체 반환.
path 생성
    아래 메서드들은 실제 그리기 작업을 수행하지는 않는다.
  • beginPath() : 그리기 시작 명시
  • closePath()
    • path 조각이 하나 이상 추가되었고, 마지막 정점이 시작점과 다른 경우 두 점을 잇는 선분을 추가한다.
  • moveTo(x, y) : path 조각의 시작점을 설정
  • lineTo(x, y) : 마지막 정점으로부터 (x, y)까지의 선분을 추가.
  • bazierCurveTo(cp1x, cp1y, cp2x, cp2y, x, y)
    • 마지막 정점으로부터 (x, y)까지를 잇는, 2개의 조절점을 이용하는 베지에 곡선을 추가한다.
  • quadraticCurveTo(cpx, cpy, x, y)
    • 마지막 정점으로부터 (x, y)까지를 잇는, 1개의 조절점을 이용하는 베지에 곡선을 추가한다.
  • arc(x, y, radius, startAngle, endAngle[, anticlockwise])
    • (x, y)를 기준점으로 하는 반지름 radius인 호를 추가한다. anticlockwise의 기본값은 false. anticlockwise를 true로 한다고 각도의 증감 방향이 바뀌진 않는다
  • arcTo(x1, y1, x2, y2, radius)
    1. 조절점1(x1, y1), 조절점2(x2, y2), 선분1(마지막 정점-조절점1), 선분2(조절점1-조절점2)를 생각한다
    2. 선분1, 선분2 모두와 거리가 radius만큼 떨어진 정점2를 생각한다. 그러면 정점2에서 반지름 radius인 원을 그렸을 때 선분1, 선분2와의 교점1, 교점2가 존재한다
    3. lineTo(교점1)
    4. 교점1부터 교점2까지의 호를 추가한다. arc(정점2, radius, ...)
  • ellipse(x, y, radiusX, radiusY, rotation, startAngle, endAngle[, anticlockwise])
    • rotation은 라디안 단위의 값으로, 해당 값만큼 타원을 회전시킨다
  • rect(x, y, width, height)
path 그리기
  • fill([fillRule])
    fill(path[, fillRule])
      • 현재 또는 지정된 path의 영역 내부를 칠한다.
    • fillRule : 'nonzero'(default) | 'evenodd'
    • path : Path2D 인스턴스
  • stroke([path])
    • 현재 또는 지정된 path의 외각선을 그린다(outline)
  • drawFocusIfNeeded(element)
    drawFocusIfNeeded(path, element)
    • element가 focus를 얻은 상태면 현재 또는 지정된 path 주변에 강조선을 그린다
  • scrollPathIntoView([path])
    • Experimental. Element.scrollIntoView()와 유사하게, 현재 또는 지정된 path가 스크린에 표시되도록 스크롤을 이동한다.
  • clip([fillRule])
    clip(path[, fillRule])
    • fillRule은 fill()과 동일. 현재 또는 지정된 path를 관심영역으로 설정한다. 이후의 그리기는 관심영역 내부의 것만 렌더링된다.
  • isPointInPath(x, y[, fillRule])
    isPointInPath(path, x, y[, fillRule])
    • fillRule은 fill()과 동일. 정점이 현재 또는 지정된 path의 내부에 있는지 판별한다
  • isPointInStroke(x, y)
    isPointInStroke(path, x, y)
    • 정점이 현재 또는 지정된 path의 외각선 위에 있는지 판별한다
path 스타일
  • lineWidth : default 1.0
  • lineCap : 종점 표시 종류. 'butt'(default), 'round', 'square'
  • lineJoin : 교점 표시 종류. 'round', 'bevel', 'miter'(default)
    • + miterLimit
  • getLineDash() + setter
    • 선분 요소와 공백(gap)의 길이로 이루어진 배열. 원소가 홀수개라면 자동으로 뒤에 똑같은 배열이 덧붙여져 짝수 크기가 된다.
    + lineDashOffset : 선분을 그리기 시작할 offset. default 0.0
텍스트 스타일
  • font : default '10px sans-serif'
  • textAlign : 'start'(default), 'end', 'left', 'right', 'center'
  • textBaseline : 'top', 'hanging', 'middle', 'alphabetic'(default), 'ideographic', 'bottom'
  • direction : 'inherit'(default), 'ltr'(left to right), 'rtl'
fillStyle, strokeStyle
    default '#000', 값으로 가능한 타입은 아래 3가지.
  1. DOMString : CSS 색상으로 파싱될 수 있는 문자열
  2. CanvasGradient
    • CanvasRenderingContext2D.createLinearGradient(x0, y0, x1, y1)
      • 선형 그래디언트의 시작점(x0, y0), 종점(x1, y1)
    • CanvasRenderingContext2D.createRadialGradient(x0, y0, r0, x1, y1, r1)
      • 원형 그래디언트의 시작 원(x0, y0, r0), 끝 원(x1, y1, r1)
  3. CanvasPattern
    • CanvasRenderingContext2D.createPattern(image, repetition)
      • image : HTMLImageElement, SVGImageElement, HTMLVideoElement, HTMLCanvasElement, ImageBitmap, OffscreenCanvas
      • repetition : 'repeat' | 'repeat-x' | 'repeat-y' | 'no-repeat'

DOMMatrix
  • 2D 및 3D 변환을 나타내는 4x4 행렬
  • 속성
    • readonly is2D : true면 2D 변환행렬, false면 3D
    • readonly isIdentity : 단위행렬인지 여부
    • m11, m12, ..., m43, m44 : 행렬 성분. double
    • a, b, c, d, e, f : 2D 변환에 사용되는 값. 차례로 2x2 변환행렬, dx, dy에 해당하며, m11, m12, m21, m22, m41, m42에 대응한다
  • setTransform(a, b, c, d, e, f)
    setTransform(matrix) + getTransform()
  • transform(a, b, c, d, e, f)
    • old = ctx.getTransform();
    식[m_{old} = \begin{bmatrix} old.a & old.c & old.e \\ old.b & old.d & old.f \\ 0 & 0 & 1 \end{bmatrix}]식
    식[m_{new} = m_{old} \begin{bmatrix} a & c & e \\ b & d & f \\ 0 & 0 & 1 \end{bmatrix}]식
  • scale(x, y)
    • 기본적으로 캔버스 내의 단위는 정확히 px과 같지만 이를 변경할 수 있다. 음수 스케일링은 요소를 뒤집는 효과가 있다
  • rotate(angle) : (0, 0)을 기준으로 시계방향 회전. 단위는 라디안.
  • translate(x, y)
    • 현재의 dx, dy에 각각 x, y만큼 추가한다
