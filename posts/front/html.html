태그
  • 문서 요소의 타입을 나타내는 이름으로, 사용자가 임의로 만들 수도 있다.
  • 태그는 <과 > 사이에 이름을 적음으로써 표현되며, 여는 태그와 닫는 태그 사이에 내용이 포함될 수 있다.
  • 여는 태그는 <name>, 닫는 태그는 </name>로 표현한다. 아래는 HTML 문서의 기본 골격을 나타내는 태그들이다.
    1. <!DOCTYPE html> : 문서가 HTML5로 작성되었음을 의미한다.
    2. <html></html> : root HTML element
    3. <head></head> : 문서에 대한 meta information
    4. <title></title> : 제목표시줄에 표시될 제목
    5. <body></body> : page content
속성
  1. 모든 HTML 태그는 속성을 가질 수 있다. 속성은 여는 태그 내부에 이름='값' 쌍으로 표현한다.
  2. 속성 값을 표현하기 위해 '', "" 모두 이용 가능하다. 아래는 대표적인 속성들이다.
    • html태그 lang : 문서의 국가 언어.
    • title : 노드에 대한 툴팁(마우스 올려두면 표시).
    • style : CSS 속성을 지시
    • disabled : 입력 태그 요소를 사용할 수 없음.
    • id : 태그에 고유한 id를 부여한다.
    • class : 태그에 특정 그룹의 class를 부여한다.
기본 태그
  • heading : h1 ~ h6
  • paragraph : p
  • link : a
    1. href 속성 : 이동할 위치나 주소
      2. # : 현재 페이지의 특정 위치(id)로 이동하기 위한 표기. '#here'은 현재 페이지의 HTML 요소 중 id가 'here'인 것을 가리킨다.
    2. 링크 상태를 나타내는 속성 : link, visited, hover, active
    3. target 속성 : _blank(새탭이나 새창에서 열기), _self, _parent, _top, framename
  • 이미지 : img
    1. src 속성 : 이미지 리소스의 위치나 주소
    2. alt 속성 : 이미지를 불러오지 못할 때, 혹은 시각장애인을 위한 대체 문자열
    3. width, height 속성 : 이미지 크기
  • line break : br
  • 수평선 : hr
  • 해석 없이 출력 : pre
  • text
    1. <b></b> : bold
    2. <strong></strong> : bold & important
    3. <i></i> : italic
    4. <em></em> : italic & important
    5. <small></small> : 작은 글자
    6. <mark></mark> : 형광펜으로 칠한 것처럼
    7. <del></del> : 취소선
    8. <ins></ins> : 삽입 문구에 밑줄
    9. <sub></sub>, <sup></sup> : 아랫첨자, 윗첨자
  • 주석 : <!-- 주석 -->
  • table
    1. table은 thead와 tbody로 이루어진다.
    2. thead와 tbody는 tr들로 이루어진다.
    3. tr은 th와 td들로 이루어진다.
    4. colgroup을 이용해 열들을 논리적으로 분리할 수 있다.
  • list
    1. unordered list : <ul></ul>
    2. ordered list : <ol></ol>
      3. type 속성 : "1"(default), "A", "a", "I"(로마숫자), "i"
    3. list item : <li></li>
  • div : 가로 100%를 점유하는 block element 대표
  • span : 가로 공간이 여유롭다면 병렬로 표시되는 inline element 대표
  • script
    1. <script></script> 사이에 JavaScript 코드 작성
    2. <noscript></noscript> 사이에 지원하지 않는 브라우저에 대한 메시지 작성
    3. <a href="javascript:;">처럼 속성으로 넣는 것도 가능
  • meta
    1. 인코딩 명시 : <meta charset="UTF-8">
    2. 검색엔진을 위한 키워드 설정 : <meta name="keywords" content="Word1, Word2, Word3">
    3. 페이지 설명 : <meta name="description" content="~~">
    4. 페이지 업데이트 설정 : <meta http-equiv="refresh" content="30"> (30초마다 갱신)
    5. 반응형 웹을 위한 viewport 설정 : <meta name="viewport" content="width=device-width, initial-scale=1.0">
  • base : 페이지에서 사용하는 모든 상대주소의 시작점
    • 예. <base href="https://dong-gi.github.io/" target="_blank">
Form
  1. form은 HTML 페이지로부터 데이터를 전송하기 위한 정형화된 틀이다. form 내부는 input 태그들로 채워진다.
  2. <input type=""> : text, password, radio, checkbox, submit, reset...
  3. form 속성 : action="주소", target="_blank | _top | ...", method="get | post"
  4. <fieldset> : 입력 요소 그루핑, <legend> : 그룹 설명
  5. <select name=""><option value=""></option> ~ </select>
  6. <textarea rows="" cols="">, <button onclick="">
  7. <input type=""> : color, date, email, month, number, range, time, ...
  8. 입력 요소 속성 : name, max, maxlength, min, pattern, readonly, required, disabled, size, ...
  9. 자세한 설명은여기로
HTML5
  • 시맨틱; Semantic
    • HTML5 이전의 웹페이지들은 table 태그를 주로 사용하여 디자인했다. 중첩된 table들로 이루어진 웹페이지는 검색 엔진을 비롯한 프로그램이 기계적으로 의미를 찾기 어려워 HTML5는 문서에 대한 의미를 태그가 포함하는 방향을 모색했다.
  • 시맨틱 태그
    • header, nav, section, article, aside, footer, ... 자세한 설명은 여기로
  • 시맨틱 CSS
    • HTML5로 접어들면서 HTML 태그들은 골격만을 정의하고, 디자인은 JavaScript와 CSS로 보다 확실히 분리되었다. 디자인의 특징은 의미 범주가 같은 것들을(class) 비슷하게 표현한다는 것이며, 따라서 CSS 선택자들은 문서의 의미 정보를 내포하고 있다고 할 수 있다.
    Bootstrap, W3C.CSS과 같은 인기 있는 CSS Framework들을 사용하면 체계적으로 정의된 클래스들 덕분에 수월하게 문서의 의미 정보를 뽑아낼 수 있다.
  • 반응형 웹
    1. 스마트폰 대중화 이전의 일반 사용자 컴퓨팅 환경은 가로 768px 이상의 직사각형 모니터가 대부분이었다.
    2. 스마트폰이 대중화 되면서 다양한 사용자 웹서핑 환경에 맞춰 알맞게 페이지를 보여줄 필요가 생겼다.
    3. viewport로 대표되는 screen 속성을 이용해 페이지를 다시 로드할 필요 없이 디자인이 변경되는 반응형 웹 기술이 발전하게 되었다.
