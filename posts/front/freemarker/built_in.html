Built-ins for strings
  1. length
    2. 길이 반환
  2. boolean
    3. "true", "false" 문자열에 대해 true, false 반환
  3. number
    4. 문자열을 숫자로 변환. 지수형 표기 인식. NaN, INF, -INF, Infinity, -Infinity 인식
  4. string
    5. 문자열 값을 반환한다. multi-type인 경우 쓸 일이 있을지도
  5. word_list
    6. FreeMarker 단어 정의의 기준. 공백 문자를 제외한 문자들의 연속
  6. 대소문자
    • cap_first
      • 첫 번째 단어의 첫번째 글자를 대문자로 변환하여 반환 ↔ uncap_first
    • capitalize
      • 각 단어의 첫글자만 대문자가 되도록 변환하여 반환
    • lower_case
      • 전부 소문자로 ↔ upper_case
  7. chop_linebreak
    8. 마지막 line-break를 제거. 없으면 그대로 반환. \n만 없애준다... \r은 그대로 남는다...
  8. trim
    9. 전후의 연속적인 공백 문자를 제거한다.
  9. index_of, last_index_of
    10. 정규표현식은 지원하지 않는다.
  10. contains
    11. 부분 문자열을 포함하는지 여부 반환. contains는 정규표현식 사용 불가
  11. date, time, datetime
    12. 문자열을 date, time, date-time 값으로 변환. 매개변수 없이 호출하는 경우 기본 로캐일의 기본 포맷만 인식한다
  12. starts_with, ends_with, ensure_starts_with, ensure_ends_with
    13. ensure : 없으면 concat해서 반환
  13. remove_beginning, remove_ending
    14. 해당 부분 문자열로 시작/끝나지 않음을 보장한다
  14. keep
    15. 일치하는 부분 문자열 전/후의 나머지 문자열 반환. 정규표현식 가능
    • keep_after
      • 처음 부분 문자열과 일치하는 부분까지 제거한 나머지 반환. 일치하는 부분이 없으면 빈 문자열, 부분 문자열 길이가 0이면 원본 그대로 반환.
    • keep_after_last
      • 마지막으로 부분 문자열과 일치하는 부분까지 제거한 나머지 반환
    • keep_before
      • 처음 부분 문자열과 일치하는 위치 전까지 반환. 일치하는 부분이 없으면 원본 그대로, 부분 문자열 길이가 0이면 빈 문자열 반환.
    • keep_before_last
      • 마지막으로 부분 문자열과 일치하는 위치 전까지 반환
  15. left_pad, right_pad
    16. 기본적으로 공백으로 채운다
  16. esc
    17. output_format이 이스케이핑 대상인 동시에 auto_esc가 꺼져있는 경우, ?esc를 호출하면 이스케이프된 문자열로 반환한다. ↔ no_esc
  17. j_string, js_string, json_string, url, url_path
    18. 각각 Java, JavaScript, JSON, url 문자열 표현에 맞게 이스케이프한다.
    url_path는 '/'를 이스케이프하지 않는다.
  18. matches, groups
    19. 반환 값은 multi-type.
    boolean : 전체 문자열이 매칭되는지 여부
    sequence : 길이 0 이상의, 매칭되는 부분 문자열 시퀀스. 기대하는 바와는 다르게, 매칭되는 부분 문자열 전후의 다른 문자열이 포함될 수 있다.
  19. replace
    20. 정규표현식 가능
  20. split
    21. 정규표현식 가능. 반환된 시퀀스의 첫 번째 문자열은 빈 문자열이 되지 않는다. 정규표현식을 사용한 경우 마지막 문자열도 빈 문자열이 되지 않는다. 빈 문자열로 분리하는 경우 각 문자들이 분리된다
  21. Common Flags
    22. Built-in i(ignore case) r(reg. exp.) m(multi-line mode) s(dot-all mode) c(whitesp. and comments) f(first only)
    replace Yes Yes Only with r Only with r Only with r Yes
    split Yes Yes Only with r Only with r Only with r No
    matches Yes Ignored Yes Yes Yes No
    keep_after Yes Yes Yes Yes Yes Ignored
    keep_after_last Yes Yes Yes Yes Yes Ignored
    keep_before Yes Yes Yes Yes Yes Ignored
    keep_before_last Yes Yes Yes Yes Yes Ignored
    ensure_starts_with Yes Ignored Yes Yes Yes Ignored

    4. 엑셀 열 표기처럼 1, 2, 3, ..., 27, ... 순서대로 a, b, c, ..., aa, ... 로 매핑한다

Built-ins for sequences
  1. size
  2. first, last
  3. reverse
  4. min, max
    5. min/max는 모든 요소가 숫자 | date-(time)일 때만 작동
  5. join
    6. join(separator[], empty_value[, list_ending]])
    * empty_value : 시퀀스가 빈 경우 대체 문자열. 기본값 ""
    * list_ending : 시퀀스가 비지 않은 경우 종결 문자열. 기본값 ""
  6. sort
    7. 모든 요소가 문자열 | 숫자 | date-(time) | boolean일 때만 작동
  7. sort_by
    8. 요소가 해시인 경우, 해시의 키를 지정하여 해당 값으로 sort 적용
  8. seq_contains
    9. 시퀀스가 해당 요소를 포함하는지 여부. Objects#equals가 아닌 == 이용
  9. seq_index_of, seq_last_index_of
    10. 요소의 순번. 없으면 -1
  10. chunk
    11. 시퀀스를 고정 길이로 잘라 반환

Etc built-ins
  1. switch
    2. switch directive의 인라인 버전. 모든 타입에 대해 작동한다.
  2. byte, double, float, int, long, short
  3. number_to_date, number_to_time, number_to_datetime
    4. 숫자 변수에 대한 변환값을 반환한다. date-(time) 변수에 대해서 long을 지원한다
  4. eval
    5. FTL을 동적으로 실행하여 문자열로 반환. 따라서 평가될 문장의 결과가 자동으로 string으로 변환될 수 있어야 한다
  5. 타입 체크
    6. is_string, is_number, is_boolean, is_date_only, is_date_like, is_time, ...
  6. sequence
    7. list directive에 이용할 수 있는 값을 sequence 값으로 변환. seq[index], seq?size 등을 이용할 수 있게 된다
