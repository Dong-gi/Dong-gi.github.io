내용

보안 요건

기밀성 confidentiality, 무결성 integrity, 가용성 availability, 인증 authentication, 부인 방지 NonRepudiation

소스 간 자원 공유 CORS;Cross-Origin Resource Sharing

  1. 본문이 text/plain이고, 커스텀 헤더가 없는 GET, POST 요청에는 Origin 헤더가 추가된다.
  2. Origin 헤더에는 요청을 보내는 페이지의 소스(프로토콜, 도메인, 포트)가 있으며, 서버는 이를 근거로 응답할 지 여부를 정한다.
  3. 서버가 응답하기로 결정하면 Access-Control-Allow-Origin 헤더를 반송한다.
  4. Access-Control-Allow-Origin 헤더의 소스가 맞지 않으면 브라우저에서 요청을 거부할 수 있다.
  5. 브라우저의 크로스 도메인을 회피하기 위해 서버에서 대신 요청을 수행해줄 수 있다.
기본 인증
  1. 기본 인증을 안전하게 이용하는 유일한 방법은 SSL과 결합해서 사용하는 것.
  2. 클라이언트 최초 GET 요청
    3. GET /family/jeff.jpt HTTP/1.0
  3. 서버 401 Unauthorized 응답
    4. HTTP/1.0 401 Authorization Required WWW-Authenticate: Basic realm="문서 집합 정보"

    WWW-Authenticate 헤더 : 보안 영역 설명

  4. 클라이언트 인증 GET 요청
    5. GET /family/jeff.jpg HTTP/1.0 Authorization: Basic "ID:base-64_encoded_password"

    Authorization 헤더 : 인증 알고리즘, 사용자 이름, 비밀번호

  5. 200 OK

다이제스트 인증

  1. 비밀번호를 네트워크를 통해 보내지 않는다.

    2. 비밀번호를 비가역적으로 뒤섞은 지문(fingerprint), 요약(digest)을 보낸다.

  2. 재전송 공격을 방지하기 위해 서버는 클라이언트에 nonce를 넘기고, nonce는 다이제스트 생성에 관여한다. nonce는 밀리초 혹은 인증할 때마다 변경되므로, 재전송 공격에 안전하다.
  3. 클라이언트 최초 요청
    4. GET /cgi-bin/checkout?cart=17854 HTTP/1.1
  4. 서버 401 Unauthorized 응답
    5. HTTP/1.1 401 Unauthorized WWW-Authenticate: Digest realm="Shopping Cart" qop="auth,auth-int" nonce="66C4EF58..."
  5. 클라이언트 인증 GET 요청
    6. GET /cgi-bin/checkout?cart=17854 HTTP/1.1 Authorization: Digest username="bri" realm="Shoppint Cart" nonce="66C4EF58..." uri="/cgi-bin/checkout?cart=17854" qop="auth" nc=0000001, cnonce="CFA920710..."
  6. 계산
    1. H(d) := 단방향 해시 함수
    2. KD(s, d) := 요약 함수 // s : sccret, d : data
    3. RFC 2617에서 제안된 알고리즘은 MD5(기본), MD5-sess
    4. H(d) = MD5(d), KD(s, d) = MD5(s:d)
    5. nonce = BASE64(timestamp H(timestamp":"ETag":"private_key))
    6. cnonce : 클라이언트가 서버를 인증할 목적으로 생성.

HTTPS

  • 디지털 서명

    • 메시지가 위/변조되지 않았음을 입증하는 체크섬

    1. 노드 A는 가변 길이 메시지를 정제하여 고정된 길이의 요약(digest)으로 만든다.
    2. 노드 A는 요약에 사용자의 개인 키를 매개변수로 하는 서명 함수를 적용한다.
    3. 노드 A는 메시지와 그에 대한 서명 둘 다를 전송한다.
  • 디지털 인증서

    • 신뢰할 수 있는 조직에 의해 서명되고 검증된 신원 확인 정보
