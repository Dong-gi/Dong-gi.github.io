1. URI syntax and components
» syntax : [scheme:]scheme-specific-part[#fragment]
» Absolute URI : 스킴을 명시한 URI
» Relative URI : not absolute URI
» Opaque URI : scheme-specific-part가 '/'로 시작하지 않는 absolute URI
* 예: "mailto:java-net@java.sun.com", "urn:isbn:096139210x"
» Hierarchical URI : scheme-specific-part가 '/'로 시작하는 absolute or relative URI
* 예: "http://example.com/languages/java", "sample/a/index.html#28", "../../demo/b/index.html"
* [scheme:][//authority][path][?query][#fragment]
- Server-based authority : [user-info@]host[:port]
- Registry-based authority : not server-based authority

2. Operations on URI instances
» Normalization : "."과 ".."을 제거
» Resolution : 상대 URI와 base URI를 이용해 단일 URI를 찾는 작업
» Relativization : Resolution의 역. 임의의 정규화된 두 URI u, v에 대하여, u.relativize(u.resolve(v)).equals(v)와 u.resolve(u.relativize(v)).equals(v)는 항상 성립한다.
