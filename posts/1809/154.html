interface SocketOptions Since 1.1
  1. @Native static final int TCP_NODELAY : Nagle 알고리즘 작동 중지
  2. @Native static final int SO_BINDADDR : Local 주소 바인딩. SocketImpl, DatagramSocketImpl의 생성자에서 설정해야 한다.
  3. @Native static final int SO_REUSEADDR : MulticastSocket에서만 사용되며, 기본 설정된다.
  4. @Native static final int SO_REUSEPORT Since 9 : 여러 소켓이 같은 주소와 포트를 listen.
  5. @Native static final int SO_BROADCAST Since 1.4 : 프로세스가 브로드캐스트 메시지를 보낼 수 있는지. 브로드캐스트를 지원하는 네트워크에서, 데이터그램 소켓을 통해서만 지원된다. 데이터그램 소켓은 이를 기본으로 설정한다.
  6. @Native static final int IP_MULTICAST_IF : 밖으로 나가는 인터페이스가 멀티캐스트 패킷을 보내는지 여부. 호스트가 여러 네트워크 인터페이스를 가진 경우, 시스템 기본이 아닌 다른 것을 이용할 때 InetAddress를 인자로 설정. DatagramSocketImpl에서 유효.
  7. @Native static final int IP_MULTICAST_IF2 Since 1.4 : IPv6 지원.
  8. @Native static final int IP_MULTICAST_LOOP Since 1.4 : 멀티캐스트 데이터그램 루프백 여부. MulticastSocket은 기본으로 설정.
  9. @Native static final int IP_TOS Since 1.4 : TCP/UDP 소켓의 IP 헤더에 type-of-service 또는 traffic class field 설정
  10. @Native static final int SO_LINGER : TCP 소켓의 close()와 관련. linger-on-close timeout을 설정하여 close()가 즉시 리턴하도록 할 수 있다.
  11. @Native static final int SO_TIMEOUT : ServerSocket::accept, SocketInputStream::read, DatagramSocket::receive에 대한 timeout 설정
  12. @Native static final int SO_SNDBUF : 전송 버퍼 크기에 대한 힌트
  13. @Native static final int SO_RCVBUF : 수신 버퍼 크기에 대한 힌트
  14. @Native static final int SO_KEEPALIVE : 2시간 동안 아무 교환이 없으면 https://docs.oracle.com/javase/10/docs/api/java/net/SocketOptions.html#SO_KEEPALIVE의 동작을 수행

final class URI implements Comparable<URI>, Serializable Since 1.4
  1. URI syntax and components
    2. » syntax : [scheme:]scheme-specific-part[#fragment]
    » Absolute URI : 스킴을 명시한 URI
    » Relative URI : not absolute URI
    » Opaque URI : scheme-specific-part가 '/'로 시작하지 않는 absolute URI
    * 예: "mailto:java-net@java.sun.com", "urn:isbn:096139210x"
    » Hierarchical URI : scheme-specific-part가 '/'로 시작하는 absolute or relative URI
    * 예: "http://example.com/languages/java", "sample/a/index.html#28", "../../demo/b/index.html"
    * [scheme:][//authority][path][?query][#fragment]
    - Server-based authority : [user-info@]host[:port]
    - Registry-based authority : not server-based authority
  2. 2. Operations on URI instances
    3. » Normalization : "."과 ".."을 제거
    » Resolution : 상대 URI와 base URI를 이용해 단일 URI를 찾는 작업
    » Relativization : Resolution의 역. 임의의 정규화된 두 URI u, v에 대하여, u.relativize(u.resolve(v)).equals(v)와 u.resolve(u.relativize(v)).equals(v)는 항상 성립한다.
