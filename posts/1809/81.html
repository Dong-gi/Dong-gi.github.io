1. 일련의 char 값들을 표현한다. 두 CharSequence 객체를 비교하는 결과는 정의되어 있지 않으며, 각 서브 클래스들이 일관적인 결과를 도출할 것이라는 보장은 없다. 따라서 CharSequence를 비교하는 메서드를 만드는 것은 적절하지 않다.
  2. int length() : 길이는 16비트 char의 개수
  3. char charAt(int index)
  4. CharSequence subSequence(int start, end)
    » start부터 end-1까지. start == end일 경우 빈 CharSequence 객체가 반환된다.
  5. String toString()
  6. default IntStream chars() Since 1.8
    » char들을 int로 0패딩하여 스트림 생성. 두 char로 쪼개진 유니코드 보충 평면의 문자를 하나로 합치진 않는다.
  7. default IntStream codePoints() Since 1.8
    » 유니코드 값들을 스트림으로 생성. 보충 평면의 문자는 Character.toCodePoint 메서드를 통해 하나의 int 값으로 형성.
