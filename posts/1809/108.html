1. 현재 쓰레드의 StackFrame에 대한 스트림을 열고 특정 작업을 수행한다. 스트림은 실행 지점인 맨 위 frame부터 아래로 내려가면서 작업한다.
    예. 현재 쓰레드의 상위 10개 스택 프레임 가져오기.
    List<StackFrame> stack = StackWalker.getInstance().walk(s -> s.limit(10).collect(Collectors.toList()));
  2. static enum StackWalker.Option Since 9
    • RETAIN_CLASS_REFERENCE : StackWalker가 작업하는 StackFrame들에 클래스 정보 유지
    • SHOW_HIDDEN_FRAMES : 모든 숨겨진 프레임 표시.
    • SHOW_REFLECT_FRAMES : 모든 reflection 프레임 표시
  3. static interface StackWalker.StackFrame Since 9
    • 각 StackFrame 객체는 StackWalker로부터 반환된, 메서드 호출을 표현한다.
    • String getClassName() : 이 프레임이 나타내는 메서드가 정의된 클래스를 binary name으로 반환
    • String getMethodName() : 이 프레임이 나타내는 메서드의 이름 반환.
    • Class<?> getDeclaringClass() : 이 프레임이 나타내는 메서드가 정의된 클래스의 객체 반환. Option.RETAIN_CLASS_REFERENCE가 설정되어야 지원된다
    • default MethodType getMethodType() Since 10
    • default String getDescriptor() Since 10
    • int getByteCodeIndex() : 실행 지점을 포함하는 Code 속성에서의 인덱스 반환.
    • String getFileName() : 이 프레임이 가리키는 실행 지점이 포함된 소스 파일 이름 반환.
    • int getLineNumber()
    • boolean isNativeMethod()
    • StackStraceElement toStackTraceElement() : 이 프레임에 관한 StackTraceElement 반환.
  4. static StackWalker getInstance() : 숨겨진 프레임을 모두 생략. 클래스 정보 미포함.
  5. static StackWalker getInstance(StackWalker.Option option)
  6. static StackWalker getInstance(Set<StackWalker.Option> options)
  7. static StackWalker getInstance(Set<StackWalker.Option> options, int estimateDepth)
  8. <T> T walk(Function<? super Stream<StackWalker.StackFrame>, ? extends T> function)
  9. void forEach(Consumer< super StackWalker.StackFrame> action)
  10. Class<&?> getCallerClass()
    » 이 메서드의 호출자의 클래스 객체를 반환.
    » 예를 들어, Util::getResourceBundle 메서드는 이 메서드를 통해 얻은 호출자의 클래스로부터 클래스 로더를 얻고, 클래스 로더를 통해 리소스를 로드한다.
