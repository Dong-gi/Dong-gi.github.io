  2. 환경변수 설정
    • JAVA_HOME, JRE_HOME 예시 » C:\Program Files\Java\jdk-10.0.2
    • Path » %JAVA_HOME%\bin
  3. IDE 설치
    4. Eclipse, IntelliJ Community 등 취향(?)에 맞게 설치한다. Eclipse로 이동
    Eclipse 환경 설정 : Window » Preferences
    • 폰트 설정
      • General - Appearance - Colors and Fonts : Basic - Text Font 및 기타 폰트를 시력에 맞게 설정
    • 포매터 설정
      • General - Editors - Text Editors : Insert spaces for tabs 체크, Show line numbers 체크
      Java → Code Style → Formatter → New → Indentation → Tab Policy = Space Only
      기존 파일에 Formatter 적용하고자 하는 경우 : 적용할 소스 or 프로젝트 우클릭 → Source → Format
    • 인코딩 설정
      • General - Workspace : Text file encoding을 UTF-8로 설정
      Web - CSS Files, HTML Files, JSP Files : Encoding을 UTF-8로 설정
    • JDK 설정
      • Java - Installed JREs : jre가 아닌 jdk로 설정
이클립스 HelloWorld 프로젝트
  1. 프로젝트 생성
    2. "File" 메뉴 클릭, 또는 "Package Explorer"에서 우클릭 - New - Java Project. 없으면 New - Other... - Java - Java Project를 선택
  2. Main 클래스 생성
    3. "Package Explorer"의 해당 프로젝트에서 우클릭 - New - Class

    Name에는 "HelloWorld"(공백 없음에 유의)를 적고, public static void main(String[] args)를 체크
  3. 실행
    4. 와 같이 코드를 작성하고 Run.
    아무것도 보이지 않는다면 Window - Show View - Console 선택

주석
  1. 주석(Comment)은 프로그래머를 위한 것으로, 컴파일러는 무시하는 문장들이다.
  2. // : '//'부터 줄의 끝까지 무시한다.
  3. /* */ : '/*'와 '*/' 사이의 모든 텍스트를 무시한다.
식별자
  • 식별자(Identifier)는 의미 있는 코드에 붙이는 이름을 의미한다. 의미 있는 코드에는 변수(Variable), 상수(Constant), 메서드, 클래스 등이 있다.
  • 자바 식별자는 '_', '$'의 사용을 허용한다. 단, '_' 단독으로는 사용할 수 없다.
  • 식별자 관례 : 상수는 SNAKE_CASE, 상수를 제외한 식별자는 camelCase 이용. 클래스는 대문자로 시작.
  • 카멜 케이스 설명 스네이크 케이스 설명
자료형, 연산
  • 자료형은 메모리 상에서 변수가 존재하는 형식을 정의한다.
  • 기본 자료형(Base Type; Primitive Type)은 사용자가 정의하지 않은, 이미 자바 언어에 정의된 자료형을 의미한다.
  • 자바의 8가지 기본 자료형 : boolean, char, byte, short, int, long, float, double
  • 정수 0으로 나누는 것은 허용되지 않지만, 실수 0.으로 나누는 것은 허용된다. 다만, 0.으로 나눈 결과를 비교할 때는 예시와 같이 Double 클래스 메서드를 이용하는 것이 정확하다.
  • 식별자 없이 값 자체를 나타내는 코드를 리터럴(Literal)이라 부른다. 각 자료형에 대응하는 리터럴 표기는 예시 코드를 참고.
    예시에서 '+' 연산자가 기본 자료형 각각에 대해 문자열 연결을 해줌을 볼 수 있다. 하지만 자바는 연산자 재정의를 지원해주지는 않으므로 오해하지 말아야 한다.
  • 자바는 강한 타입 검사를 하며, 자바스크립트와 같이 null, '', 0 등을 false로 간주하거나 하는 일도 없다.
  • 참조 자료형(Reference Type)은 기본 자료형을 제외한 모든 자료형을 포함하며, 메모리의 주소를 통해 값에 접근한다. C/C++의 포인터와 같지만, 자바는 포인터 연산을 지원하지는 않는다. 예시 코드는 자바의 배열에 대해 보인다. 자바의 다차원 배열은 기본적으로 Jagged Array이다.
  • 기본 자료형 변수는 정적 할당, 참조 자료형 변수는 동적 할당된다. 동적 할당된 객체들은 자동으로 GC 대상이 된다.
    » 정적 할당 : 컴파일러가 메모리 점유를 정확하게 예측할 수 있는 (지역)변수를 생성하는 방법으로, 선언과 동시에 스택 영역에 할당되어 코드 블록이 끝나는 순간 해제된다.
    » 동적 할당 : 동적으로 할당되는 객체들은 힙 영역에 생성되며, 그 참조가 유지되는 한 프로그램 종료시까지 유지된다.
    • 형변환 묵시적 형변환 : 자식 타입 변수는 부모 타입 변수에 그냥 할당 가능. 산술 연산시에도 자동으로 이루어진다. 명시적 형변환 : 부모 타입 변수를 자식 타입 변수에 할당하는 경우, 명시 필요. Checked
