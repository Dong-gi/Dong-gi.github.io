struct - Interpret bytes as packed binary data 1. struct는 bytes 객체로 표현된 C 구조체와 Python 값 사이의 변환을 지원한다.

2. Functions and Exceptions
» exception struct.error
» struct.pack(fmt, v1, v2, ...) : 포맷 문자열 fmt에 따라 v1, v2, ... 값들을 포장한 bytes 객체를 반환한다.
» struct.pack_into(fmt, buffer, offset, v1, v2, ...) : pack과 기능은 동일.
» struct.unpack(fmt, buffer) : 포맷 문자열 fmt에 따라 기존에 포장된 buffer를 풀어 값들의 튜플로 반환한다.
» struct.unpack_from(fmt, buffer, offset=0) : unpack과 기능은 동일
» struct.iter_unpack(fmt, buffer) Since 3.4 : unpack과 비슷하다. 같은 크기의 아이템들을 순회하는 iterator를 반환한다. buffer의 크기는 반드시 fmt의 정수배여야 하며, calcsize(fmt)로 구할 수 있다.
» struct.calcsize(fmt) : 필요한 struct 바이트 크기

3. Format Strings
» Byte Order, Size, and Alignment
* @(default) : native order : native size : native alignment // C 컴파일러의 sizeof로 결정된다.
* = : native : standard : none
* < : little-endian : standard : none
* > : big-endian : standard : none
* ! : network(=big-endian) : standard : none
» Format Characters
* x : pad byte(in C) : no value(in Python)
* c : char : bytes of length 1 : 1(size)
* b : signed char : integer : 1
* B : unsigned char : integer : 1
* ? : _Bool : bool : 1
* h : short : integer : 2
* H : unsigned short :integer : 2
* i : int : integer : 4
* I : unsigned int : integer : 4
* l : long : integer : 4
* L : unsigned long : integer : 4
* q : long long : integer : 8
* Q : unsigned long long : integer : 8
* n : ssize_t : integer : @에서만 가능
* N : size_t : integer : @에서만 가능
* f : float : float : 4
* d : double : float : 8
* s : char[] : bytes
* p : char[] : bytes
* P : void* : integer : @에서만 가능
** 포맷 문자 앞에 정수가 오면 반복을 의미한다. 포맷 문자 사이의 공백은 무시한다.
** s의 경우 앞에 오는 정수는 반복이 아닌 바이트 크기다. 주어지지 않는다면 기본은 1. 패킹 과정에서 크기를 맞추기 위해 null 바이트가 포함될 수 있다. 언패킹에선 bytes 객체의 크기가 딱 알맞다.
** 0s는 빈 문자열을 의미한다.(0c는 0개 문자)
** p는 "Pascal string"으로, 첫 바이트는 문자열 길이를 담는다.

4. 예제
» https://docs.python.org/3/library/struct.html#examples
>>> from struct import *
>>> pack('hhl', 1, 2, 3)
b'\x00\x01\x00\x02\x00\x00\x00\x03'
>>> unpack('hhl', b'\x00\x01\x00\x02\x00\x00\x00\x03')
(1, 2, 3)
>>> calcsize('hhl')
8
