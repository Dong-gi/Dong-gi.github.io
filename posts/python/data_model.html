Objects, values, and types

  1. 모든 객체는 id, type, value를 갖는다.
  2. id는 생성된 후에는 불변하며, CPython의 경우 id(obj)는 obj의 메모리 주소다.
  3. type(obj)으로 obj의 타입을 얻을 수 있으며, id와 마찬가지로 불변하다.
  4. Container : 다른 객체에 대한 참조를 포함하는 객체

The standard type hierarchy

  • None

    • 식별자 "None"으로 이 타입의 싱글톤 객체를 얻을 수 있다. 값이 없음을 알리기 위해 사용된다.

  • NotImplemented

    • 식별자 "NotImplemented"로 이 타입의 싱글톤 객체를 얻을 수 있다. 산수 메서드, 비교 메서드는 연산이 구현되지 않은 경우 이를 반환해야 한다.

  • Ellipsis

    • 리터럴 ... 이나 식별자 "Ellipsis"로 이 타입의 싱글톤 객체를 얻을 수 있다. 다음은 NumPy에서 "전체 차원"을 의미하는 ...

numbers.Number; 수

  1. numbers.Integral; 정수
    1. int : 크기 제약 없는 정수
    2. bool : 이며, True, False는 대부분 상황에서한다.
  2. numbers.Real (float) : double precision 부동 소수
  3. numbers.Complex : 실수부, 허수부 모두 float인 복소수.

시퀀스

  1. 음이 아닌 정수로 인덱싱(seq[i]) 될 수 있는 유한 길이의 순서 있는 집합
  2. 내장함수 len(seq)는 시퀀스가 가진 항목의 수를 반환한다
  3. seq[i:j], seq[i:j:k] 슬라이싱 지원

불변 시퀀스

  1. 문자열
    • U+0000 ~ U+10FFFF 유니코드 코드 포인트를 표현하는 값들의 시퀀스. 코드 포인트 값은 내장 함수 ord()로 획득 가능
    • char 타입이 없는 대신, 각 문자열의 코드 포인트들은 길이 1인 문자열 객체.
    • str.encode()는 주어진 인코딩으로 str을 bytes로 변환하고, bytes.decode()는 그 역을 수행한다.
    • bytes와 bytearray는 str이 제공하는 메서드들을 거의 지원한다

      • casefold(), isdecimal(), encode() 등 유니코드 관련 메서드도 지원한다. 이진 시퀀스로 정규 표현식을 컴파일하면 정규 표현식도 이진 시퀀스에 적용 가능

  2. 튜플; tuple
    • 임의 객체들로 이루어진 시퀀스.
    • 항목이 2개 이상 : 콤마로 분리한 표현식의 나열로 튜플 생성
    • 항목이 1개 : 표현식에 콤마를 붙여서 생성
    • 항목이 0개 : 한 쌍의 빈 괄호로
  3. bytes
    • 바이트(0 ~ 255) 단위 불변 배열
    • 리터럴 생성 : b'Hello World'
    • 생성자 이용 : bytes()

가변 시퀀스

  1. Subscription(seq[i])과 슬라이싱은 대입문과 del 문의 대상으로 사용될 수 있다
  2. list

    3. []안에 표현식을 콤마로 분리하여 표현함으로써 생성

  3. bytearray

    4. bytes와 같은 인터페이스와 기능을 제공.

  4. 확장 모듈 array, collections는 추가적인 가변 시퀀스 타입을 제공한다

    5. array.array : frombytes(), fromfile(), fromlist(), tobytes(), tofile(), tolist(), byteswap(), typecode
    collections.deque : append()와 popleft()는 원자성을 갖는다. 초과되면 오래된 항목을 버린다.

  5. bisect 모듈

    6. 정렬된 시퀀스에 대하여, 정렬을 유지한 채로 삽입 등의 연산이 이뤄지도록 해준다.

  6. memoryview

    7. 버퍼 프로토콜을 지원하는 객체의 메모리 뷰를 만든다. 버퍼 프로토콜을 지원하는 내장 객체에는 bytes와 bytearray가 있다.
    기저 객체가 쓰기 가능하면, 메모리 뷰는 일차원 슬라이스 대입을 지원한다. 크기 변경은 허용되지 않는다

  7. queue 모듈

    8. Queue, LifoQueue, PriorityQueue : 초과되면 빈 공간이 생길 때까지 대기

  8. multiprocessing 모듈

    9. Queue, JoinableQueue

  9. asyncio 모듈

    10. Queue, LifoQueue, PriorityQueue, JoinableQueue

집합

  1. 중복없는 불변 객체들의 순서없는 유한집합. 인덱싱할 수는 없지만 이터레이팅은 가능하다.
  2. 가변 집합 set과 불변 집합 frozenset이 있다.
  3. a | b 합, a & b 교, a - b 차

매핑

  1. 임의의 인덱스 집합으로 인덱싱되는 객체들의 유한집합. 매핑에서 인덱스 표기법 map[k]는 매핑 map에서 k로 인덱스 되는 객체를 선택한다. 이는 표현식에 사용되거나 del 문의 대상이 될 수 있다.
  2. 내장 매핑형은 dict 하나가 존재하며, 키로 사용할 수 있는 것은 불변 객체뿐 ─ 키의 해시값이 불변할 것을 요구하기 때문 ─ 이다.
  3. isinstance(test_obj, collections.abc.Mapping)를 통해 타입을 검사하면 된다
  4. 기본값 설정 : setdefault() 이용 or defaultdict 사용 or __missing__() 정의
  5. 매핑의 읽기 전용 객체 반환 : MappingProxyType

특별 메서드

특별 메서드를 구현하면 사용자 정의 객체도 내장형 객체처럼 작동하게 된다. 사용자 정의 클래스의 묵시적인 특수 메서드 호출은 메서드가 인스턴스 딕셔너리가 아닌 객체의 형에 정의되어 있을 때만 올바르게 작동함이

특별 메서드를 직접 구현하기보다는 ABC를 상속하는 것이 낫다.

  • collections.abc

    • Iterable, Container, Sized, Callable, Hashable, Iterator, Sequence, MutableSequence, Mapping, MutableMapping, Set, MutableSet, MappingView, ItemsView, Keyiew, ValuesView

  • numbers

    • Number, Complex, Real, Rational, Integral

Basic customization

  • object.__new__(clazz[, ...])
    1. 클래스 clazz의 새 인스턴스를 만들기 위해 호출된다.
    2. 첫 번째 인자는 만들려는 인스턴스의 클래스, 나머지는 생성자에 전달될 것들
    3. __new__()가 clazz 인스턴스를 반환한다면 인스턴스의 __init__() 메서드가 __init__(self[, ...])처럼 호출된다.
    4. 주로 int, str, tuple 등 불변형 클래스의 서브 클래스의 인스턴스 생성을 위해 사용하고, 일반적인 구현은 super().__new__(cls[, ...])를 호출한 뒤에 나머지 작업 수행
  • object.__init__(self[, ...])
    1. 베이스 클래스가 __init__() 메서드를 갖고 있다면, 서브 클래스 __init__() 구현 시 명시적으로 호출해주는 것이 옳다.
    2. __new__() 메서드가 생성하고 __init__() 메서드가 값을 설정하므로, __init__()은 None만을 반환할 수 있다.
  • object.__del__(self)
    1. 베이스 클래스가 __del__() 메서드를 갖고 있다면, 서브 클래스에서 명시적으로 호출해줘야 한다.
    2. del x는 x의 참조 횟수를 감소시키고, x.__del__()은 참조 횟수가 0이 될 때 호출된다.
    3. 인터프리터 종료 시 남아있는 객체들의 __del__() 호출은 보장되지 않는다.
  • object.__repr__(self)
    1. repr() 내장 함수에 의해 호출되며, 객체에 대한 정형적인 문자열 표현을 계산한다.
    2. 객체의 문자열 표현 자체가 객체를 만들 수 있는 표현식이 되는 것이 좋다
    3. __str__()이 없는 경우 대신 호출될 수 있다
  • object.__str__(self)
    1. str(object)와 내장 함수 format(), print()에 의해 호출된다
    2. 읽기 좋은 객체의 문자열 표현을 반환하면 된다
    3. object에 정의된 기본 구현은 object.__repr__()을 호출한다
  • object.__bytes__(self)

    • bytes 생성자에 의해 호출되며, 객체의 바이트열 표현을 반환하면 된다.

  • object.__format__(self, format_spec)

    • format() 내장 함수, 포맷 문자열 리터럴 계산, str.format() 메서드에 의해 호출된다. 정의되지 않은 경우 str(x)가 호출된다.

  • object.__lt__(self, other) + le, eq, ne, gt, ge

    • 각각 <, <=, ==, !=, >, >=로 호출된다.

  • object.__hash__(self)
    1. 내장 함수 hash()와 set, frozenset, dict과 같은 해시형 컬렉션에서 호출된다.
    2. 사용자 정의 클래스는 기본적으로 인스턴스의 고유값을 반환하는 __hash__()와 __eq__()를 갖는다.
    3. __eq__()를 재정의하면 __hash__()는 None으로 설정된다. __hash__()를 재정의할 때, __eq__()를 같이 정의해줘야 컬렉션에서 정상 작동된다.
    4. __eq__()를 재정의하지 않고 __hash__()를 지원하지 않으려는 경우, __hash__ = None 설정을 직접 해줘야 한다.
  • object.__bool__(self)
    1. 논리값 검사와 내장 연산 bool()에서 호출된다.
    2. 재정의하지 않은 경우, __len__()이 정의됐다면 return __len__() != 0
    3. 재정의하지 않았고, __len__()도 없다면 참으로 취급

Customizing attribute access

  1. object.__getattribute__(self, name)
    1. 인스턴스의 속성을 접근할 때 항상 호출된다.
    2. 인스턴스의 속성값을 반환하거나 AttributeError를 일으켜야 한다.
    3. 재정의할 때, 무한 재귀의 발생을 방지하기 위해 object.__getattribute__(self, name)처럼 베이스 클래스의 메서드를 호출해야 한다.
  2. object.__getattr__(self, name)

    3. __getattribute__()가 AttributeError를 일으키거나 __get__()이 AttributeError를 일으킬 때 호출된다.

  3. object.__setattr__(self, name, value)

    4. 속성명(name)에 대한 값(value)을 별개 객체가 아닌, 인스턴스(self)의 속성에 직접 대입하는 경우, object.__setattr__(self, name, value)처럼 베이스 클래스의 메서드를 호출해야 한다.

  4. object.__delattr__(self, name)

    5. del obj.name이 의미가 있는 경우에만 재정의해야 한다.

  5. object.__dir__(self)

    6. 객체에 dir()이 호출될 때 호출된다. 시퀀스를 반환해야 한다.

Implementing Descriptors

다음의 메서드들은, 메서드가 정의된 클래스의 인스턴스가 소유자(owner) 클래스에 등장할 때만 적용된다. 속성(attribute)들은 소유자 클래스의 __dict__의 키로 이용되는 속성을 가리킨다.

  1. object.__get__(self, instance, owner)
    1. 소유자 클래스 속성 접근, 소유자 클래스 인스턴스의 속성 접근시 호출된다.
    2. owner는 항상 소유자 클래스며, 클래스 속성 접근 시 instance는 None.
    3. 속성값을 반환하거나 AttributeError를 일으켜야 한다
  2. object.__set__(self, instance, value)
  3. object.__delete__(self, instance)
  4. object.__set_name__(self, owner, name)

    5. 소유자 클래스 owner가 생성될 때 호출되어 이 디스크립터를 name에 할당한다.

디스크립터 호출

  1. 디스크립터 : __get__(), __set__(), __delete__() 중 하나라도 정의한 클래스
    • Data descriptor : __set__()이나 __delete__() 중 하나라도 정의한 경우

      • 보통 데이터 디스크립터는 __get__()과 __set__()을 모두 정의한다. 이 경우 데이터 디스크립터는 인스턴스의 __dict__에 있는 값보다 우선한다.
      property() 함수는 데이터 디스크립터로 구현되기 때문에, 인스턴스가 property의 동작을 변경할 수 없다. property는 __set__()을 자동 구현하므로, 읽기 전용 속성을 만들기 쉽다.

    • Non-data descriptor : 인스턴스 __dict__보다 우선순위가 낮다

      • 파이썬 메서드(staticmethod(), classmethod() 포함)는 비데이터 디스크립터로 구현되기 때문에, 인스턴스가 메서드를 새로 정의하거나 덮어쓸 수 있다. 따라서 같은 클래스의 다른 인스턴스들과는 다른 동작을 할 수 있다.
      → 모든 사용자 정의 함수는 __get__() 메서드가 정의된 디스크립터다. 사용자 정의 함수가 객체의 메서드로 바인드되고, 바인딩된 메서드 객체는 __call__() 메서드를 가지며, __self__ 속성을 첫 번째 인자로 전달해서 __func__ 속성이 참조하는 원래 함수를 호출한다.

  2. 속성 접근(obj.name) 기본 절차

    3. obj.__dict__['name'] → type(obj).__dict__['name'] → type(obj)의 메타 클래스를 제외한 베이스 클래스 조회

  3. 속성 접근 기본 절차 대신 디스크립터가 호출되는 경우
    1. 직접 호출

      2. name.__get__(obj)

    2. 인스턴스 결합

      3. obj.name → type(obj).__dict__['name'].__get__(obj, type(obj))

    3. 클래스 결합

      4. Clazz.name → Clazz.__dict__['name'].__get__(None, Clazz)

    4. Super 결합

      5. super(Child, obj).method() → obj.__class__.__mro__에서 Child 바로 다음 나오는 베이스 클래스 Parent를 찾은 뒤, Parent.__dict__['method'].__get__(obj, obj.__class__)

object.__slots__

  1. 멤버를 명시적으로 선언하고 __dict__와 __weakref__ 생성을 거부할 수 있다. 이를 통해 공간 절약 + 속성 접근 속도 향상을 기대할 수 있다.
  2. __slots__가 없는 클래스를 상속하면 인스턴스의 __dict__와 __weakref__ 속성은 항상 존재한다
  3. __dict__가 생성되지 않으므로, __slots__에 없는 새 속성을 할당할 수 없다(AttributeError).

    4. 동적으로 새로운 속성을 할당하는 것이 필요하다면 __slots__에 '__dict__'를 추가한다

  4. __weakref__가 생성되지 않으므로, 인스턴스에 대한 약한 참조를 지원하지 않는다.

    5. 약한 참조 지원이 필요하다면 __slots__에 '__weakref__'를 추가한다.

  5. __slots__는 각 변수 이름마다 디스크립터를 만드는 방식으로 클래스 수준에서 구현된다
  6. 부모가 선언한 __slots__는 자식 클래스에도 제공된다

    7. 자식 클래스가 __slots__(새로 추가되는 변수명만 포함)을 정의하지 않으면 __dict__와 __weakref__를 갖게 된다

Customizing class creation

  1. classmethod object.__init_subclass__(cls)

    2. 클래스를 상속하는 경우, 베이스 클래스의 __init_subclass__가 호출된다. 클래스 데코레이터는 직접 적용할 클래스를 지정해줘야 하지만, __init_subclass__는 미래에 정의될 모든 서브 클래스에 적용된다.
    새 클래스에 주어진 키워드 인자들(metaclass 제외)은 부모 클래스의 __init_subclass__로 전달된다. __init_subclass__에서는 필요한 키워드 인자들만 사용하고, 나머지는 베이스 클래스로 전달해야 한다.

  2. metaclass
    1. 참고자료
    2. 기본적으로 클래스는 type(name, bases, dict)을 사용해서 만들어진다.
    3. 메타클래스는 type(name, bases, dict)의 동작을 대신 수행할 수 있는 함수/클래스다. __new__, __init__, __call__을 후킹하여 복잡한 작업을 뒤에서 처리할 수 있게 해준다.
  3. 클래스 및 인스턴스 검사
    1. ABC; Abstract Base Class 도입을 위해 추가되었다

    2. class.__instancecheck__(self, instance)

      3. isinstance(instance, class)에서 호출된다

    3. class.__subclasscheck__(self, subclass)

      4. issubclass(subclass, class)에서 호출된다

    5. classmethod object.__class_getitem__(cls, key)

  5. 콜러블 흉내내기

    1. 매핑 : keys(), values(), items(), get(), clear(), setdefault(), pop(), popitem(), copy(), update()

      2. collections.abc 모듈의 추상 베이스 클래스 MutableMapping을 이용하면 쉽다

    2. 가변 시퀀스 : append(), count(), index(), extend(), insert(), pop(), remove(), reverse(), sort()

    5. 이터레이션을 위해 매핑, 시퀀스 모두 __iter__()를 구현하도록 권장

    메서드 목록

    1. object.__len__(self) : 내장함수 len()에서 호출. __bool__()을 정의하지 않은 경우 이를 이용한다
    2. object.__length_hist__(self) Since 3.4 : operator.length_hint()에서 호출. 추정값을 반환하면 된다
    3. 슬라이싱
      1. object.__getitem__(self, key)

        2. self[key]. 시퀀스 타입의 경우 정수와 슬라이스 객체만 key로 허용. 음수 인덱스를 어떻게 해석할 지를 여기서 구현한다. key가 적절하지 않은 타입이면 TypeError, 시퀀스의 범위 밖이면 IndexError(특히 for 루프가 이에 의존), 매핑의 key가 없으면 KeyError

      2. object.__setitem__(self, key, value)
      3. object.__delitem__(self, key)
    4. object.__missing__(self, key) : dict.__getitem__()이 못찾은 경우 이를 통해 기본값을 반환
    5. object.__iter__(self) : 이터레이터가 필요한 경우 호출된다
    6. object.__reversed__(self)

      7. 역순으로 탐색하는 이터레이터를 반환하면 된다. reversed() 내장함수는 기본적으로 시퀀스 프로토콜 ─ __len__(), __getitem__()을 이용하므로 이보다 효율적인 구현을 제공할 수 있을때만 구현한다

    7. object.__contains__(self, item)

      8. in, not in에서 호출되며, 기본적으로 __iter__()를 이용하고, 없으면 __getitem__()을 이용한다.

  7. 숫자 흉내내기

Coroutine

  1. Generator Coroutine

    2. yield문의 실행 제어 효과를 이용한 비동기 처리 구문

    • generator.close()

      • 제너레이터가 실행을 중단한 yield 표현식이 GeneratorExit 예외를 발생하게 만든다. GeneratorExit 예외를 받으면 제너레이터는 아무런 값도 생성하지 않아야 한다.

    • generator.throw(exc_type[, exc_value[, traceback]])

      • 제너레이터가 중단한 곳의 yield 표현식에 예외를 전달한다. 제너레이터가 예외를 처리하면 제어 흐름이 다음 yield 문까지 진행되고, 생성된 값은 generator.throw() 호출 값이 된다. 제너레이터가 예외를 처리하지 않으면 코루틴은 중단되고 호출자까지 예외가 전파된다.

    • yield from
      • yield from x

        • iter(x)를 호출해서 x의 반복자를 가져온다. StopIteration 예외를 내부적으로 잡아서 자동으로 처리한다.

      • yield from은 가장 바깥쪽 호출자와 가장 안쪽의 하위 제너레이터 사이에 양방향 채널을 열어준다.
        • delegating generator : yield from 표현식을 갖는 제너레이터 함수
        • subgenerator : yield from 표현식이 가져온 제너레이터.
        • caller : delegating generator를 호출하는 코드
  2. Awaitable Object

    3. object.__await__(self) Since 3.5 : 어웨이터블 구현을 위한 이터레이터 반환

  3. Coroutine Object

    4. 코루틴 객체는 어웨이터블 객체다. 코루틴의 실행은 __await__()의 호출 결과를 이터레이트함으로써 제어된다.

    • coroutine.send(value)

      • 코루틴의 실행을 시작하거나 재개. value가 None이면 __await__()의 이터레이터를 전진, None이 아니면 이터레이터의 send() 메서드로 위임

    • coroutine.throw(type[, value[, traceback]])

      • 코루틴에서 예외가 처리되지 않으면 호출자에게 되돌아온다

    • coroutine.close()
  4. Asynchronous Iterator Since 3.5
    • object.__aiter__(self)

      • 비동기 제너레이터 반환

    • object.__anext__(self)

      • 비동기 제너레이터는 여기서 비동기 코드를 호출할 수 있다. 이터레이터의 다음 값을 반환하는 어웨이터블을 반환해야 한다. 이터레이션이 끝났다면 StopAsyncIteration을 발생해야 한다

  5. async with문 Since 3.5
    • object.__aenter__(self)

      • __enter__()와 유사하며, 어웨이터블을 반환한다

    • object.__aexit__(self, exc_type, exc_value, traceback)

      • __exit__()과 유사하며, 어웨이터블을 반환한다
