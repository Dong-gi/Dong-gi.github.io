이클립스 애너테이션 처리 오류

  1. 자료 : https://bugs.eclipse.org/bugs/show_bug.cgi?id=367599
  2. 자료 : https://github.com/palantir/gradle-processors/issues/5
  3. 예외 : DOMA4011, javax.annotation.processing.FilterException: Source file already created
  4. 원인 : 이클립스 배치 모드에서 특정 파일 수(2000)를 초과하면 애너테이션 처리에 버그 발생
  5. 해결 : eclipse.ini vmargs 섹션에 아래처럼 추가
    6. -vmargs -DmaxCompiledUnitsAtOnce=20000

MySQL 시간 기본값 0 설정이 안 될 때

  1. 왜 굳이 now(), NULL, 특정 시각 상수가 아닌 0을 이용하려는지는 모르겠지만...
  2. 원인 : 서버에 NO_ZERO_DATE 옵션이 설정됐기 때문
  3. 옵션 확인
    4. SHOW VARIABLES LIKE 'sql_mode' -- 또는 SELECT @@sql_mode
  4. 해결 1
    5. SET GLOBAL sql_mode=(SELECT REPLACE(@@sql_mode,'NO_ZERO_DATE',''));
  5. 해결 2 : /etc/my.cnf 수정
    6. [mysqld] sql_mode="수정값"

PostgreSQL remaining connection slots are reserved for non-replication superuser connections

  1. 자료 : https://stackoverflow.com/a/32584211/7369515
  2. 원인 : 일반적으로 최대 연결 수 부족
  3. 해결 : /var/lib/pgsql/data/postgresql.conf
    4. max_connections=${적당히_큰_숫자}

도커 컨테이너 에러 종료 후 재시작 불가

  1. 예외 종료된 도커 컨테이너가 아래와 같이 실행되지 않을 때
    2. Error response from daemon: container "83a7208d6b68d3d97be501b47d6a4bb28245b086f25eda819dec93a8d40320a3": already exists
  2. 아래 명령으로 컨테이너의 잔존 메타데이터를 지우면 다시 시작할 수 있다.
    3. docker-containerd-ctr --namespace moby --address /var/run/docker/containerd/docker-containerd.sock c rm [container_id]

    container_id는 전체를 넘겨줘야 하고, address 옵션값은 정확한 경로로 넘겨줘야 한다.
