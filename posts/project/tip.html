SSL 인증서

Certbot

  • 무료 SSL 인증서(3개월) 제공
  • 저장 위치 : /etc/letsencrypt/
  • DNS 레코드를 이용한 인증서 발급

    • ↓shell

    $ sudo certbot certonly -d *.4joy.link --manual --preferred-challenges dns

    DNS 레코드 확인

    ↓shell

    $ nslookup -q=TXT _acme-challenge.4joy.link
  • certbot renew [--cert-name CERTNAME] [options]
    • 만료 30일 미만인 (모든) 인증서 갱신
    • 인증서 이름은 certbot certificates로 확인 가능
    • --pre-hook "service nginx stop" --post-hook "service nginx start"

      • 인증서를 하나라도 갱신해야 하는 경우, 각 갱신 전후에 실행. hook가 실패해도 인증서 갱신은 계속 진행된다

    • --deploy-hook DEPLOY_HOOK

      • 갱신에 성공한 각 경우에만 실행

    • --force-renewal

      • 만료 기간과 무관하게 갱신

오라클 클라우드 > 로드밸런서 > 인증서 추가

  • SSL 인증서로 cert.pem 설정
  • CA 인증서로 fullchain.pem 설정
  • 전용 키에 privkey.pem 설정

Windows

  • 차단한 앱 알림 해제

    • 레지스트리 : HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Notifications\Settings\

  • 확장자 일괄 변경

    • ↓shell

    ren *.html *.txt
  • dir 결과 저장

    • ↓shell

    dir /s /b *.png > png.txt
    • /s : 재귀
    • /b : "디렉터리/파일명"만 출력
    • > : 저장
  • 시작 프로그램 등록
    1. shell:startup 실행

      2. C:\Users\%USER_NAME%\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup

    2. exe 바로가기 붙여넣기
  • 부트매니저 설정
    • SSD에 설치한 윈도우가 BIOS 설정에서는 검색되지 않는 경우에 대해 해결됨 확인
    • 자료 : https://m.blog.naver.com/pspsim/221141734669
      1. msconfig 실행
      2. 부팅 메뉴에서 기본값 설정, 불필요한 부팅 옵션 삭제 등 처리 후 저장
  • SSD 인식 불량 해결
    • 파티션은 활성 상태인데 볼륨은 비활성 상태인 경우에 대해 해결됨 확인
    • 자료 : https://kin.naver.com/qna/detail.nhn?d1id=1&dirId=1010305&docId=291566434&qb=7KeI66y4CSDtlZjrk5zrlJTsiqTtgawg7J247Iud7J20IOyViOuQmOuEpOyalA==&enc=utf8§ion=kin&rank=2&search_sort=0&spq=0

      • ↓shell

      diskpart list disk select disk 1 # 디스크 번호는 list 결과에 따라 적절히 clean create partition primary # 부팅 디스크로 이용하려면 주(primary) 파티션이어야 함 list partition select partition 1 active # 파티션 활성화 format fs=ntfs quick # 빠른 포맷 진행 exit
