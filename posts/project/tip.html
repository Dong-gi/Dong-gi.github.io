메시지 큐 설계

설계 전 제약사항

  • Pub/Sub 구조를 사용한다 : 메시지 생산과 소비는 특정 토픽에 대하여 이루어진다
  • 메시지 소비는 이벤트 풀링 방식으로 진행된다

설계 요소

  • 높은 처리 대역폭을 위해 하나의 토픽에 대한 메시지들을 여러 파티션에 분할하여 저장할 필요가 있다
    • 생산자가 특별히 지정하지 않은 경우 : 임의 파티션에 전달된다
    • 생산자가 파티션을 지정한 경우 : 해당 파티션에 전달된다
    • 신규 메시지를 n개 복제본까지 저장이 완료됐을 때 생산자에게 확인 응답을 전달한다 : n은 처리 속도와 안정성 사이의 trade-off
  • 내결함성을 위해 각 파티션들은 분산 복제되어야 한다

    • 예. 각 파티션에 대해 1개 마스터, 2개 복제본을 사용하는 노드 배치

    노드1노드2노드3노드4
    토픽1/파티션1(마스터)토픽1/파티션1(복제)토픽1/파티션1(복제)
    토픽1/파티션2(마스터)토픽1/파티션2(복제)토픽1/파티션2(복제)

    예. 토픽이 많이 추가되어 노드를 추가해야 한다면? -- 물론 노드를 하나만 추가해야 된다는 법은 없다

    노드1노드2노드3노드4노드5
    토픽1/파티션1(마스터)토픽1/파티션1(복제)토픽1/파티션1(복제)
    토픽1/파티션2(마스터)토픽1/파티션2(복제)토픽1/파티션2(복제)
    토픽2/파티션1(마스터)토픽2/파티션1(복제)토픽2/파티션1(복제)
    토픽2/파티션2(복제)토픽2/파티션2(마스터)토픽2/파티션2(복제)

    예. 노드가 줄었다면?

    • 기존 복제 파티션이 마스터 승격 -- 메시지 큐잉 처리
    • 신규 복제 파티션 추가 -- 마스터로부터 열심히 복제 수행
    • 관리자의 조정이 아니라, 장애 발생으로 인해 노드가 줄었다면 대체 노드가 신규 복제 파티션을 갖는다
    노드1노드2(삭제)노드3노드4
    토픽1/파티션1(마스터)토픽1/파티션1(복제)토픽1/파티션1(복제 시작)
    토픽1/파티션2(복제 시작)토픽1/파티션2(마스터)토픽1/파티션2(복제)

    예. 처리량을 늘리기 위해 파티션을 추가한다면? -- 신규 메시지만 신규 파티션으로 가도록 해야한다

    노드1노드2노드3노드4
    토픽1/파티션1(마스터)토픽1/파티션1(복제)토픽1/파티션1(복제)
    토픽1/파티션2(마스터)토픽1/파티션2(복제)토픽1/파티션2(복제)
    토픽1/파티션3(복제)토픽1/파티션3(마스터)토픽1/파티션3(복제)

    예. 불필요한 파티션을 제거한다면?

    1. 신규 메시지가 삭제 대상 파티션에 가지 않도록
    2. 모든 메시지가 소비되면 해당 파티션 삭제
  • 소비자는 특정 토픽에 대해 구독을 신청하고, 메시지를 가져갈 파티션을 할당받는다
    • 적절한 메시지 소비를 위해, 어떤 파티션이 어떤 소비자에게 할당됐는지 관리하는 코디네이터가 있어야 한다
    • 코디네이터는 소비자 추가만이 아니라, 소비자 제거(의도적이든 사고든)에도 대응해야 한다

      • 즉, 소비자가 할당받은 파티션은 이후 변경될 수 있다

    • 특정 생산자의 메시지를 FIFO로 소비되도록 보장하려면?
      • 생산자는 파티션을 지정하여 메시지를 생산한다
      • 토픽의 파티션에 대한 소비자를 최대 1개로 제한한다
      • 파티션보다 많은 수의 소비자가 존재할 수 없으므로, 높은 처리 대역폭이 필요하다면 미리 파티션을 다수로 나눌 필요가 있다

메시지 전달 보장 레벨

  • 최대 한 번 전달 : 생산자는 메시지 생산에 대한 확인 응답을 받지 않고, 소비자가 메시지를 가져가면 무조건 처리됐다고 간주
  • 최소 한 번 전달 : 생산자는 메시지 생산에 대한 확인 응답을 받고, 소비자가 메시지 소비 완료했다고 브로커에게 전달해야 처리됐다고 간주
    • 생산자는 타임아웃 발생 시 메시지를 다시 생산
    • 메시지 소비가 실제로는 완료됐지만 브로커에게 전달되지 않은 경우, 같은 메시지가 다시 소비
  • 정확히 한 번 전달 : 많은 오버헤드가 추가로 필요

지리정보 색인

균등 격자(even grid)

  • 지도의 위도/경도를 각각 균등하게 나눈다
  • 단순하지만, 각 격자의 정보 밀도가 심하게 차이날 수 있다

    • 예. 사막/대양을 주로 포함하는 격자 vs 도심을 포함하는 격자

지오해시(geohash)

  • 지도 전체에 대해 적용한 균등 격자를, 다시 각 격자에 재귀적으로 적용한다
  • 격자를 재귀적으로 나누면서 고유한 주소를 할당한다

    • 예를 들어, 위도/경도를 각각 2등분하는 경우

    • 1단계 분할 : 주소 0 ~ 3 할당. 격자 크기 = 약 20,000km^2
    • 2단계 분할 : 주소 00 ~ 33 할당. 격자 크기 = 약 10,000km^2
    • 3단계 분할 : 주소 000 ~ 333 할당. 격자 크기 = 약 5,000km^2
  • 두 격자 주소의 접두어가 충분히 일치하다면 두 격자는 인접하다

    • 역은 성립하지 않는다

쿼드트리(quadtree)

  • 격자를 재귀적으로 나누어 지오해시를 구성하는 경우, 사막/대양도 잘게 나뉘어 결과적으로 아무런 정보를 갖지 않는 격자의 수가 크게 늘어나게 된다
  • 쿼드트리에서는 트리 각 노드의 정보가 임계값 이상으로 많은 경우에만 재귀적으로 쪼갠다

구글 S2

  • 지구를 힐베르트 곡선이라는 공간 채움 곡선을 이용하여 1차원으로 색인한다
  • 힐베르트 곡선 상에서 인접한 두 지점은 색인 이후에도 인접한 위치에 있음이 보장된다
