파티션 테이블에 새로운 인덱스 추가하기

부모 테이블에 대해 create index concurrently가 불가능하므로, 아래 절차로 생성

  1. 부모 테이블에 only 옵션으로 create index 실행

    자식(파티션) 테이블에 대해 create index가 재귀적으로 실행되지 않는다. 따라서 전체 index는 INVALID 상태

  2. 각 파티션 테이블에 대해 create index concurrently 실행

    ↓ sql

    create index concurrently idx_xxx_2024 on xxx_2024 (xxx); alter index idx_xxx attach partition idx_xxx_2024;

    모든 파티션 테이블에 대해 인덱스가 연결되면 전체 index는 자동으로 VALID 상태로 변경

OVERLAPS : 날짜 구간 겹침 확인

  • (start1, end1) OVERLAPS (start2, end2)

    [start1, end1) 구간과 [start2, end2) 구간에 겹침이 존재하는지 여부를 반환한다

  • 종료 시각 대신 interval을 이용할 수 있다

  • 구간의 시작 시각과 종료 시각이 같은 경우, 구간은 지정된 시각에 한정된다

그룹별 n개 행만 반환

LATERAL JOIN은 좌항의 테이블을 모두 구성한 뒤, 좌항 테이블 행을 직접 참조하여 우항의 테이블을 구성할 수 있게 해준다

↓ sql

select player_id, t2.play_data from ( select distinct player_id from t_player_play_history ) t1 join lateral ( select play_end_time, play_data from t_player_play_history x where x.player_id = t1.player_id order by play_end_time desc limit 3 ) t2 on true order by player_id, play_end_time desc

한글 종성 구분

  1. 한글은 0xAC00('가')부터 0xD7A3('힣')까지 총 11,172자가 정의
  2. 11,172 = 초성 19 * 중성 21 * 종성 28(종성 없는 경우 포함)EastAsianWidth
  3. 코드 포인트 = (초성 인덱스 * 21 + 중성 인덱스) * 28 + 종성 인덱스 + '가'0xAC00
    • (코드 포인트 - '가'0xAC00) % 28 = 종성 인덱스
    • (코드 포인트 - 종성 인덱스 - '가'0xAC00) / 28 % 21 = 중성 인덱스
    • ((코드 포인트 - 종성 인덱스 - '가'0xAC00) / 28 - 중성 인덱스) / 21 = 초성 인덱스
