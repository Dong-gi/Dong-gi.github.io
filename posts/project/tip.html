PostgreSQL

  • 명령어 목록
  • timezone 변경
    1. postgresql.conf의 log_timezone, timezone 설정 변경
    2. SELECT pg_reload_conf();
    3. alter database db_name set timezone to 'Asia/Seoul';
  • postgres 유저 비밀번호를 모를 때
    1. pg_hba.conf 파일을 비밀번호 없이 접속할 수 있도록 trust로 변경 후 접속(서비스 재시작 필요)
    2. 비밀번호 설정 후 pg_hba.conf 다시 원복
  • 모든 테이블에 대한 권한 부여

    • ↓ sql

      GRANT SELECT ON ALL TABLES IN SCHEMA public TO user;

    • ↓ sql

      GRANT EXECUTE ON ALL FUNCTIONS IN SCHEMA public TO user;
    • 이후 추가되는 테이블에도 적용하려면 기본 권한을 변경

      • ↓ sql

      ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT SELECT ON TABLES TO user;
    • 실제로 적용됐는지 확인

      • ↓ sql

      select * from information_schema.role_table_grants where table_name = 'table_name';
  • dblink 재활용

    • ↓ sql

    SELECT dblink_connect('myconn', 'hostaddr=1.2.3.4 port=5432 dbname=history user=readonly password=password'); (SELECT account_id, last_login_date FROM v_account WHERE account_id IN (SELECT r.account_id FROM dblink('myconn', 'SELECT account_id, sum(amount) as total_amount FROM v_payment_history WHERE status=2 GROUP BY account_id HAVING sum(amount) >= 10000') AS r(account_id BIGINT, total_amount BIGINT)) ) UNION ...
  • 시각 차이 획득

    • ↓ sql

    select abs(extract (epoch from (insert_datetime - '2019-07-04 19:39:55'::timestamp)));
  • 커넥션 정리

    • ↓ sql

    select pg_terminate_backend(pid) from pg_stat_activity where client_addr = '10.0.2.2';
  • AWS Redshift와 드라이버 충돌될 때 jdbc 연결 쿼리에OpenSourceSubProtocolOverride=true추가
  • 테이블 목록 획득

    • ↓ sql

    select * from information_schema.tables
  • 테이블 정보 획득

    • ↓ sql

    select * from information_schema.columns where table_name = '{tableName}'
  • 테이블 기본 키 획득

    • ↓ sql

    SELECT pg_attribute.attname as column FROM pg_index, pg_class, pg_attribute, pg_namespace WHERE pg_class.oid = '{tableName}'::regclass AND indrelid = pg_class.oid AND nspname = 'public' AND pg_class.relnamespace = pg_namespace.oid AND pg_attribute.attrelid = pg_class.oid AND pg_attribute.attnum = any(pg_index.indkey) AND indisprimary;

윈도우

  • 차단한 앱 알림 해제

    • 레지스트리 : HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Notifications\Settings\

  • 확장자 일괄 변경

    • ↓ shell

    ren *.html *.txt
  • dir 결과 저장

    • ↓ shell

    dir /s /b *.png > png.txt
    • /s : 재귀
    • /b : "디렉터리/파일명"만 출력
    • > : 저장
  • 시작 프로그램 등록
    1. shell:startup 실행

      2. C:\Users\%USER_NAME%\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup

    2. exe 바로가기 붙여넣기
  • 부트매니저 설정
    • SSD에 설치한 윈도우가 BIOS 설정에서는 검색되지 않는 경우에 대해 해결됨 확인
    • 자료 : https://m.blog.naver.com/pspsim/221141734669
      1. msconfig 실행
      2. 부팅 메뉴에서 기본값 설정, 불필요한 부팅 옵션 삭제 등 처리 후 저장
  • SSD 인식 불량 해결
    • 파티션은 활성 상태인데 볼륨은 비활성 상태인 경우에 대해 해결됨 확인
    • 자료 : https://kin.naver.com/qna/detail.nhn?d1id=1&dirId=1010305&docId=291566434&qb=7KeI66y4CSDtlZjrk5zrlJTsiqTtgawg7J247Iud7J20IOyViOuQmOuEpOyalA==&enc=utf8§ion=kin&rank=2&search_sort=0&spq=0

      • ↓ shell

      diskpart list disk select disk 1 # 디스크 번호는 list 결과에 따라 적절히 clean create partition primary # 부팅 디스크로 이용하려면 주(primary) 파티션이어야 함 list partition select partition 1 active # 파티션 활성화 format fs=ntfs quick # 빠른 포맷 진행 exit

  • Reference

    • ↓ java

    boolean isEqual(byte[] a, byte[] b) { if (a.length != b.length) return false; for (var i = 0; i < a.length; ++i) if (a[i] != b[i]) return false; return true; }
  • 네트워크 연결 속도가 불안정한 상황에서도 최소 20마이크로초 단위로 서버의 응답 시간 차이를 식별할 수 있다고 한다
